模块总结

发表于2025-07-23|更新于2025-07-23|2.Areas🌐science小论文准备论文复现总结可能会用到的模块
|总字数:1.2k|阅读时长:3分钟|浏览量:

问题总结

  • 在多模态领域中，语义差异限制了特征表达，影响了模型性能（BeFA:Khurana等人，2023；Rinaldi、Russo和Tommasino，2023）
  • 多模式推荐的目标是预测用户偏好，而不是理解多模式内容（CHOSION ）
  • BPR损失只能监督最终的用户评分学习，而不能优化每种特定模态的评分学习（CHOSION POWERec）
  • 粗粒度属性表示项目之间的共性，而细粒度属性是影响用户购买决策的关键因素。由于显式交互特征仅通过模态项目图增强，并通过简单的聚合来表示用户模态，因此它们缺乏与用户行为（GUME）相关的信息

各模型总结

基准Encoder模型选用

融合模态选用

自监督任务选用

LATTICE-2021

LATTICE：GCN + Encoder（user-item + item-item双图） + Lately Fusion（attention加权融合模态）

FREEDOM-2023

FREEDOM：LATTICE + Encoder（item-item冻结） + Encoder（ user-item的边修剪dropout 去噪模块）

PGL-2025

PGL：LATTICE + Encoder（全局感知+局部感知） + structure-based SSL增强（特征dropout）

MENTER-2025

MENTER：FREEDOM + Lately Fusion增强（多层次跨模态对齐 feature-based SSL） + SSL混合增强（特征dropout + 图扰动）

SMORE-2025

SMORE：LATTICE + Lately Fusion（基于频谱） + feature-based SSL（注意力+注入行为信号对比学习）

MGCN-2023

MGCN：LATTICE’（基于ID引导过滤模态噪声）+ Encoder（多视图编码）+ Lately Fusion（ID引导下提取模态偏好权重+ feature-based SSL辅助任务最大化多模态特征和ID特征的互信息）

DSGRec-2025（四区）

DSGRec（4区）：MGCN + Encoder（ID信号+多模态信号）+ structure-based SSL（超边）

DGM-2025

DGM： LATTICE + Encoder（跨模态通用编码器+特定模块编码器）+ 模态生成模块（特征重建+双特征生成+refine graph）

CMDL

MGCN+自注意力（特有+共享）+互信息解耦

  1. 模态解耦表示：明确区分模态共享和特有知识，解决现有方法无法区分的问题
  2. 互信息正则化：通过统计约束确保解耦有效性，解决简单约束不足的问题
  3. 动态组合机制：根据协同信息自适应组合不同表示，解决简单聚合的局限性

​

CHOESION

MGCN’（残差网络）+双阶段融合+BPR注意力

​

DGVAE

CLIP/Sentence-Transformer(多模态预处理) + Res-GCN’(简化残差图卷积网络) + 解耦潜在原型表示模块 + 互信息最大化对齐模块

  1. 可解释性提升：通过将图像等转换为文本描述，使推荐结果可以用自然语言解释
  2. 性能保证：图结构捕捉高阶关系，解耦表示学习更丰富的语义信息
  3. 对齐验证：互信息最大化确保数字交互可以与文本偏好对应

DiffMM-2024

GNN’(基础协同过滤) + 多模态图扩散模块(含前向/反向过程和模态感知信号注入) + 跨模态对比学习框架(含两种对比范式)

BeFA

预训练模型 + 基于行为特征解耦 + 行为引导的特征重构

通过行为信息引导，可以识别并强化真正反映用户偏好的内容特征，同时抑制无关信息，从而解决信息漂移和遗漏问题。

FMMRec

LATTICE’(基准推荐模型) + 公平导向的模态解耦模块 + 关系感知的公平学习模块(公平/不公平关系挖掘 + 角色指示嵌入)

  1. 模态解耦：明确分离多模态内容中的敏感和非敏感信息，控制因果关系
  2. 关系挖掘：利用解耦后的嵌入构建用户关系图，调整因果影响
  3. 对抗学习：确保学习到的表示独立于敏感属性，实现反事实公平

GUIDER

噪声识别 + 正负样本增强 + 知识蒸馏

  1. AMSC：通过模态内相似性和跨模态一致性校准样本，解决了多模态空间中噪声交互难以检测的问题。
  2. DBPR：直接利用校准后的样本优化推荐模型，避免了传统方法在多模态空间中的失效。
  3. OT-based KD：通过教师模型的去噪知识指导学生模型，解决了直接应用去噪损失函数到MMRecs的困难。

GUME

  1. 图增强：通过多模态相似性添加边，长尾物品能获得更多信息传播路径，缓解冷启动问题。
  2. 用户模态增强：通过对比学习整合历史偏好和潜在兴趣，用户模态表示更全面。
  3. 模态对齐：去除噪声后，模态特征更纯净，推荐更准确。

MoDiCF

DDPM + 扩散数据补全 + 反事实多模态推荐

POWERec

基于提示的多模态用户兴趣学习 + 弱模态增强
