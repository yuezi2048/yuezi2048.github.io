4.Pytorch等重要依赖安装

发表于2025-07-06|更新于2025-07-06|2.Areas🌐science小论文准备云服务器运行环境
|总字数:652|阅读时长:2分钟|浏览量:

**摘要：**AI领域的工作，大多是基于TensorFlow或者Pytorch实现的，那么Pytorch的安装就至关重要了，在前一节虚拟环境的基础上，我们首先要确定自身的显卡驱动版本，然后还要确定作者的依赖版本。这样我们才能顺利地搭好Pytorch环境！

前置工作

确定CUDA版本

首先确定CUDA版本

或者

确定项目依赖

查看项目需要的pytorch和python版本（README.md或者requirement.txt）

在该网站中找到对应的安装命令，并选择兼容的CUDA版本

Installing previous versions of PyTorch

创建虚拟环境

这里以2.1.0版本的Pytorch为例，我们接下来完成创建虚拟环境后，并安装pytorch等重要依赖，个人建议全局安装一个通用的环境，python的3.9.7版本比较稳定，推荐，在虚拟环境下安装每个作者对应的环境即可（没错，每复现一个项目就可能需要折腾一下，但折腾以后会方便很多），如果偷懒的话，不按照作者的环境能跑通其实问题也不大，复现结果可能会有点点差别。

注意python和pytorch的依赖关系：【环境搭建】Python、PyTorch与cuda的版本对应表

​

安装

安装Pytorch

安装pytorch依赖，注意需要开启显卡安装才能安装成功，可以先无显卡的时候下载，再开显卡后安装

安装命令地址：https://pytorch.org/get-started/previous-versions/

安装pytorch_geometric

安装pytorch_geometric，我们要手动下载轮子，否则会有问题：https://data.pyg.org/whl/

  • 通过whl对应的cuda和pytorch安装
  • 根据python版本+windows下载所有依赖
  • pip install *.whl

安装其他依赖

省流版快速复现代码

可以提前在记事本改好，直接一键粘贴运行命令。

技巧：安装Pytorch前可以无卡模式执行，等到安装Pytorch那步报错的时候再开显卡（不想省1快钱可以跳过该步）
