连地址法

链地址法（拉链法 链接法）：一个地址处只能存放同义词，不存放非同义词，将同义词组成链表，存在对应的地址处即可

不同情况下的ASL：查找一个数据的比较次数

ASL平均查找长度：查找n个数查找长度和 / n

开放地址法

  • ASL成功查找长度
  • ASL失败查找长度

链地址法

  • ASL成功查找长度

高级搜索树

  • 二叉树 —有序性 —> BST —-平衡性—-> AVL（旋转）

  • AVL树的缺点：

    • （1）不直观（需要计算每个节点的平衡因子）；
    • （2）频繁的插入和删除操作，需要不断的旋转调整，比较复杂 —> 需要大量的调整操作 —> 引申出高级搜索树：红黑树 B树 B+树
      • 自平衡：通过引入一些调整手段，在插入、删除等操作导致失衡后，能够自行调整平衡

  • 多路平衡查找树（B树）

    • 一个节点有多个数据，同时有多个孩子（如外存中存放多个数据块，减少IO次数），一个节点有n-1个数据，就有n个孩子
    • 是绝对平衡的（平衡因子全部是0） + 满足有序性

  • 2-3树（3阶B树）：可以是空树，或者由2-节点 和 3-节点组成，并且绝对平衡：即根节点到任意一个叶子节点的深度相同

    • 2-节点：含有一个数据，两个孩子的节点，并且满足顺序性：左孩子<父亲<右孩子
    • 3-节点：含有两个数据，三个孩子的节点，并且满足顺序性：最左孩子<父亲第一个数据<中间孩子<父亲第二个数据<最右孩子
    • 度为3

  • 2-3-4树（4阶B树）：由2-节点、3-节点、4-节点组成

  • 2-3-4-…m树（m阶B树）

将上述的度取名为阶

对于B树和红黑树：

  • 内部节点：非空节点
  • 外部节点：最下面一层非空节点的空孩子节点 -— 叶子节点（空节点）

红黑树：本质上就是2-3-4树（或者2-3树）

  • 为了进行查找遍历，通过分裂下移，对3/4阶B树转换成BST二叉树
  • 为了后续进行整合，对分裂的节点标记为红色

红黑树的性质

  • （1）具有顺序性
  • （2）每个节点一定是黑色或者红色的
  • （3）根（转换后的BST）和叶子节点（空节点）一定是黑色的
  • （4）不存在两个相连的红色节点（红色节点的父亲和孩子一定是黑色节点，是B树原来的节点）
  • （5）对于每个节点来说，从该节点到其下面任一叶子节点的路径上的黑色节点的数目是相同的

口诀：左根右，根叶黑，不红红，黑路同（记住）

黑高：在一颗红黑树中，从一个节点x出发，到其下一个叶子节点的简单路径所包含的黑色节点的个数（注：不含起点）

  • 结论1：从根节点出发，到叶子节点的最长路径不超过短路径的两倍

    • 最短路径：黑高
    • 最长路径：树的高度（临界情况：该路径上每个非空黑色节点都往下拆分一个红色节点）
    • $h ≤ 2b_h$

  • 推论1：红黑树也是平衡的，一个节点其矮的节点高度为h，另外一个节点高度就不能超过2h + 1（矮的路径h+1，高的路径不超过2(h + 1)，所以高度2(h + 1) - 1 ）

    • 红黑树相对于AVL树，平衡的限制放宽，因此增删后，调整不会太频繁，但是就单次调整来说会复杂很多，适合增删频繁的场景

  • 结论2：有n个内部节点的红黑树，其高度 $h ≤ 2log(n+1)$ —> 计算红黑树操作的复杂度

    • 通过结论1证明

红黑树的应用

  • 编程语言中，大多数自平衡BST库函数，都是基于红黑树实现的（C++ map set，Java TreeSet TreeMap）
  • Linux操作系统的CPU调度，有一个完全公平调度，也是基于红黑树实现的
  • Linux提供的epoll多路复用的底层结构

红黑树

红黑树的操作：查找、插入、删除

  • 查找：就是BST的查找操作
  • 插入操作：在红黑树新插入一个数据，也就是插入一个新节点
    • 先执行BST的插入操作，去找z应该插入的位置，首先满足顺序性
    • 插入后，考虑z的颜色：初始颜色设定为红色（注：如果是空树插入，插入之后要把z变成黑色满足根叶黑性质）
      • z的初始颜色是黑色：一定破坏了黑路同性质，每次插入后都得进行调整
      • z的初始颜色是红色，可能破坏不红红的性质，只有插入后z的父亲节点也是红色才破坏性质，需要调整

*如果破坏了不红红性质，如何进行调整？ —> 染色 + 旋转

  • 染色：红色变黑色，黑色变红色
  • 旋转：LL LR RR RL，和AVL树的旋转一模一样

除了递推分析破坏性质修复，也可以换一种思路理解：看成是一个拆分错误的B树，组合成一个正确的B树

插入操作

假设插入节点z，z初始为红色

  • 首先执行BST的插入操作

  • 具体的调整：

    • 空树插入：z是根节点，z直接变黑色即可

    • 非空树插入：z不是根节点

    • z->parent是黑色，不破坏红黑树的性质，插入结束，不需要调整

    • z->parent是红色，破坏了不红红，需要进行调整

      • z是红色
      • z->parent是红色
      • 爷爷节点g = z->parent->parent不可能是根节点，是黑色
      • z的叔叔节点u：红色/黑色（空节点也是黑的）

    • 情况1：z->parent父亲节点是g的右孩子，叔叔节点是左孩子，此时u=g->left

      • 如果叔叔u是红色，则将z的父亲、叔叔和爷爷变色：父亲和叔叔变黑，爷爷变红，如果爷爷的父亲也是红色或者爷爷是根节点，把爷爷节点看成新的需要调整的节点z，看爷爷节点满足哪种情况，对爷爷不断向上调整即可，直到某个需要调整的节点z父亲是黑色，或者z是根节点就停止

      • 如果叔叔u是黑色

        • 新节点z是其父亲的右孩子，z相对于g来说是RR，父亲，爷爷染色（父亲变黑色，爷爷变红色），以爷爷为中心左旋

        • 新节点z是其父亲的左孩子，z相当于g来说是RL，关于父亲右旋，并交换z指针和p指针，转换成RR的情况，按照RR的方式处理

    • 情况2：z->parent父亲节点是g的左孩子，叔叔节点是右孩子（u = g->right 情况1的对称问题）

      • 如果叔叔u是红色，则将z的父亲、叔叔和爷爷变色：父亲和叔叔变黑，爷爷变红，如果爷爷的父亲也是红色或者爷爷是根节点，把爷爷节点看成新的需要调整的节点z，看爷爷节点满足哪种情况，对爷爷不断向上调整即可，直到某个需要调整的节点z父亲是黑色，或者z是根节点就停止
      • 如果叔叔u是黑色
        • 新节点z是其父亲的左孩子，z相对于g来说是LL，父亲，爷爷染色（父亲变黑色，爷爷变红色），以爷爷为中心右旋
        • 新节点z是其父亲的右孩子，z相当于g来说是LR，关于父亲左旋，并交换z指针和p指针，转换成LL的情况，按照LL的方式处理

删除操作

是否满足红黑树的性质，要看y和x的颜色（y是free掉的节点，x是y的子孩子），根据排序组合有以下情况（注：不会有两个红色的情况）：

  • y红，x黑（不会破坏任何性质，不需要调整）

  • y黑，x红（一定破坏黑路同）

    • 如果y的父亲是红色，那么会破坏不红红，此时只需要把x调整成为黑色就可以（包括y的父亲是根节点的情况）

    • 如果y的父亲是黑色，那么不会破坏其他性质

  • y黑，x黑：复杂情况

对于y黑，x黑的情况，要关注以下节点：

  • 以删除之前的关系来说

    • y节点：黑色
    • x节点：黑色，y是x父亲
    • w节点：红色/黑色，是y的兄弟，x的叔叔（删除之后，w是x的兄弟）
    • p节点：红色/黑色，y的父亲节点，x的爷爷节点（删除之后p是x的父亲节点）

  • y（x）在右，w在左，删除y后，x顶替y的位置，x和w成为兄弟（x是黑）

    • a）w是黑色且w至少有一个红色的孩子RED（p是黑色/红色）

      • a.1）LL：RED节点是w的左孩子，RED相对于p是LL

        • w变成p的颜色，再对p右旋，最后red和p变黑

      • a.2）LR：RED节点是w的右孩子，RED相对于p是LR

        • 思想：LR变成LL，对w进行左旋，然后red变黑，w变红。再交换red和w指针 —> a.1

    • b）w是黑色且w没有红色的孩子（w两个孩子都是黑色，p是黑色/红色）

      • 双黑节点：只是在删除逻辑中存在的，删除黑色的y后，我们认为节点释放了但是颜色留下了，然后黑色的x顶替了y的位置。认为x拥有两层黑色，x就称为双黑节点，双黑节点对黑高的贡献是2
      • 此时目标是让双黑节点x退掉一层黑色，w陪着退掉一层黑色（即w变成红色），为了维持子树的黑高，退的一层黑还给p
        • 如果p是红色，则变成黑，调整结束
        • 如果p为黑节点，p作为新的要调整的节点，直到碰到红色节点或者根节点为止（整体少一个黑高） ——循环向上调整

    • c）w是红色节点（x是黑色，p是黑色，w的孩子也是黑色）

      • 对p右旋，w变黑，p变红，w=p->l ，此时转换成情况a或者情况b的情况

  • y（x）在左，w在右

    • a）w是黑色且w至少有一个红色的孩子RED（p是黑色/红色）
      • a.1）RR：RED节点是w的右孩子，RED相对于p是RR
        • w变成p的颜色，再对p左旋，最后red和p变黑
      • a.2）RL：RED节点是w的左孩子，RED相对于p是RL
      • 思想：RR变成RL，对w进行右旋，然后red变黑，w变红。再交换red和w指针 —> a.1
    • b）w是黑色且w没有红色的孩子（w两个孩子都是黑色，p是黑色/红色）
      • 双黑节点：只是在删除逻辑中存在的，删除黑色的y后，我们认为节点释放了但是颜色留下了，然后黑色的x顶替了y的位置。认为x拥有两层黑色，x就称为双黑节点，双黑节点对黑高的贡献是2
      • 此时目标是让双黑节点x退掉一层黑色，w陪着退掉一层黑色（即w变成红色），为了维持子树的黑高，退的一层黑还给p
        • 如果p是红色，则变成黑，调整结束
        • 如果p为黑节点，p作为新的要调整的节点，直到碰到红色节点或者根节点为止（整体少一个黑高） ——循环向上调整
    • c）w是红色节点（x是黑色，p是黑色，w的孩子也是黑色）
      • 对p左旋，w变黑，p变红，w=p->r ，此时转换成情况a或者情况b的情况

  • 时间复杂度 O(logn)

B树

应用场景

  • B树一个节点中可以存放多个数据，相比于自平衡的二叉树来说，树的高度矮很多
  • 减少磁盘I/O次数，对外存进行分块，一块就是一个外存的节点

B树的性质

  • 每个节点最多m-1个关键字，最多m个孩子节点
  • 每个非根节点最少有ceil(m / 2) - 1个关键字，最少有ceil(m / 2)个孩子节点
  • 根节点中，最少有1个关键字，最少有2两个孩子
  • 一个节点中如果有x个孩子节点，该节点一定有x - 1个关键字
  • 顺序性：集结点内部有多个数据，满足顺序性，节点之间也要满足顺序性
  • 绝对平衡

B树的操作

  • 查找
  • 插入
  • 删除

查找：基于顺序性进行查找，类似于BST，考虑节点中有多个关键字即可。

假设查找k

  • 先让k和根节点中的第一个数据进行比较，如果相等，查找成功
  • 如果不相等，继续和第二个、第三个关键字比较，如果有数据和k相等，则找到
  • 否则，找到第一个大于k的数据就停止，继续向该数据所对应的左孩子中进行查找，如果没有大于k的数据，则在最右边的孩子进行查找，继续上面的过程
  • 直到找到k或者找到空节点就停止

插入：回溯插入法、主动插入法，本质是一样的：查找插入位置–按顺序插入–分裂上传

回溯插入法：在一颗B树中插入关键字k

  • 情况1：空树插入，新建根节点，把k放到根节点中即可
  • 情况2：非空树插入
    • （1）查找插入的位置：先对k执行查找操作，肯定会查找失败，会找到一个空节点，空节点的父亲节点k就是应该插入的节点（终端节点）
    • （2）按顺序插入
    • （3）插入之后，被插入节点关键字个数<= m - 1，插入结束，如果> m - 1，进行分裂该节点（分裂回溯上传）
      • 分裂回溯上传：从该节点数据数组的 s = ceil(m / 2)的位置分开，数据数组分成3部分（1—s-1 ，s ，s + 1—-m）
        • 1 – s -1放到一个节点中，s + 1—m放到一个节点中，此时一个节点被分裂为两个节点，s节点则插入到父亲节点中
        • 对父亲进行判断，关键字个数是否超过上限，如果超过了上限，继续对父亲分裂回溯上传，直到某个父亲节点接受孩子后不超上限位置，或者上传到了根节点，根节点的个数也超过了上限，根节点分裂，新建一个根节点，停止

主动插入法

  • 情况1：空树插入，新建根节点，把k放到根节点中即可
  • 情况2：非空树插入
    • 在查找过程中就把有分裂隐患的节点（关键字 = m-1）提前进行分裂，避免一些节点插入关键字后，避免插入关键字后，关键字个数> m - 1

删除操作：假设删除节点k（兄弟够借则借，否则合并）

  • （1）查找。首先进行查找k的位置，后面的位置往前移
  • （2）删除：最后转换成在终端节点删除数据x
    • 情况1：x是终端节点，按顺序删除即可，执行顺序表的删除
    • 情况2：x是非终端节点，转换为情况1即可，去找k的中序遍历前驱/后继 k1 及其所在的节点x1（后继好找），让k1覆盖k，转换为在x1中k1，此时x1节点一定是终端节点
  • （3）删除关键字之后，如果x中关键字个数 >= ceil(m / 2) - 1，删除结束，否则进行调整：
    • 够借：先去看x相邻的亲兄弟节点够不够借（关键字个数 > ceil(m / 2) - 1），注意借关键字时，要保证顺序性，左兄弟借最大关键字，给父亲，然后父亲的节点对应的关键字给x。同理，右兄弟借最小的关键字
    • 不够借：相邻两个兄弟节点关键字个数 == ceil(m / 2) - 1，则合并一个兄弟节点，合并时要把父亲节点的关键字一起合并，由于父亲节点少了一个关键字，如果小于下限就继续调整，如此循环直到父亲节点符合要求，或者最终合并到根节点为止。（注：根节点的关键字下限是1）

对于m阶B树，$O(log_mn)$

B+树

  • 内部非终端节点的数据域，只存放一条记录的关键字，而不存放数据指针

  • 内部终端节点的数据域，既存放关键字又存放数据指针

  • 内部非终端节点只存在索引的作用，所有关键字和数据都存放在终端节点中，上层节点是下层节点的索引

B+树的优势：

  • 相比于B树更矮胖
  • 支持区间查找（通过终端节点之间指针相互链接）

B+树的性质（m阶，实际上有两个阶m和n，m控制内部非终端节点，n控制内部终端节点）

  • 一个节点中的关键字个数和孩子个数相等
  • 对于非根节点来说，关键字和孩子个数上限是m，下限ceil(m / 2)
  • 对于根节点来说，关键字下限是1，上限是m
  • 终端节点通过指针相连，形成类似链表的结构，支持区间查找
  • 满足B树的性质：顺序性、绝对平衡

操作：查找、插入、删除

查找

  • 单点查找：查找某一个关键字k（基于顺序性）
    • 从根节点开始比较，在节点中找第一个大于等于k的数k1
    • 去k1对应的孩子节点中继续查找，直到找到终端节点为止
    • 对于终端节点中第一个大于等于k的k1，如果等于k==k1，说明找到，否则说明不存在
  • 区间查找
    • 先对左边界数据lk进行单点查找，找到第一个大于等于lk的数
    • 从该数开始在终端节点上遍历查找即可

插入，类似B树

  • 在B+树中插入关键字k

    • 查找插入位置（一定是在终端节点上）
    • 执行顺序表的插入
    • 因为上层节点是下层节点的索引，即上层节点保存了下层节点的最大值，如果插入后导致节点的最大值发生变化，此时需要向上更新索引
    • 判断插入数据的节点关键字个数是否超过上限，如果超过了上限，分裂上传：
      • 一分为二：s = (m + 1) / 2， [1，s]，[s + 1，m + 1] 分别放在两个节点中，再把新建立节点的最大值插入到父亲节点中

  • 为了更加实用，在实现B+树时，每个节点均不保存最右边孩子的索引，一个m阶的B+树关键字个数的上限就是m - 1，孩子个数的上限就是m

删除

在B+树中插入关键字k

  • 查找插入位置（一定是在终端节点上）
  • 判断被删除数据的节点的关键字个数是否低于下限，如果低于下限，就进行调整（兄弟够借则借，不够则合并）
    • 借兄弟时无需经过父亲节点，此时问题转换为兄弟节点删除借的节点
    • 合并时，靠左的节点固定合并到靠右的节点，删掉原先左节点的索引即可，并更新终端节点的链表，此时问题转换为父亲删除对应的关键字问题
  • 如果删除的数据是该节点的最大值，需要向上更新父亲节点中的索引

时间复杂度：logm(n)

总结

  • 基础查找：顺序、折半、分块、哈希
  • 高级搜索树：红黑树、B树、B+树
