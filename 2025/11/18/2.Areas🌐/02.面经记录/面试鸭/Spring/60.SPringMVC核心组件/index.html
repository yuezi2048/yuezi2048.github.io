60.SPringMVC核心组件

发表于2025-11-18|更新于2025-11-18|2.Areas🌐02.面经记录面试鸭Spring
|总字数:1.2k|阅读时长:4分钟|浏览量:

回答重点

  1. DispatcherServlet：Spring MVC 的前端控制器，所有请求都会先由它接收并分发到相应的处理器。它是整个 Spring MVC 流程的入口，也是处理请求的调度中心。
  2. HandlerMapping：根据请求 URL 查找对应的处理器（Controller）。HandlerMapping 用于将客户端的请求映射到具体的控制器方法。
  3. Controller（控制器）：负责处理用户请求的核心组件。Controller 接收请求后，调用业务逻辑处理，返回数据给前端。
  4. ModelAndView：Controller 返回的对象，包含模型数据和视图信息，表示返回给用户的页面或 API 响应。
  5. ViewResolver：用于解析视图名称并找到实际的视图（如 JSP、Thymeleaf 等），然后将处理结果渲染为 HTML 或其他格式，发送给客户端。
  6. HandlerInterceptor（拦截器）：在请求处理的各个阶段拦截 HTTP 请求和响应，可以在控制器执行前后添加自定义逻辑，如日志记录、权限检查等。
  7. HandlerAdapter：负责执行找到的处理器（Controller），将请求交给处理器进行业务逻辑的处理。

核心工作流程：

  1. DispatcherServlet 接收请求并调用 HandlerMapping 查找对应的控制器。
  2. HandlerAdapter 调用具体的 Controller 方法处理请求。
  3. Controller 返回 ModelAndView，包括视图名和模型数据。
  4. ViewResolver 解析视图并渲染结果。
  5. 最终，DispatcherServlet 将处理结果返回给客户端。

扩展知识

DispatcherServlet（前端控制器）

作用

DispatcherServlet 是整个 Spring MVC 应用的前端控制器（Front Controller），它拦截所有的 HTTP 请求，并将请求分发给适当的组件进行处理。

工作原理

  • 当客户端发送请求时，DispatcherServlet 会先接收请求。
  • 然后，它会根据请求 URL 通过 HandlerMapping 找到合适的处理器（Controller）。
  • 处理完请求后，它会根据 ViewResolver 找到对应的视图，将处理结果发送回客户端。

HandlerMapping（处理器映射）

作用

HandlerMapping 负责将请求 URL 映射到具体的处理器（即 Controller）方法。它通过配置的映射规则来查找 Controller 并确定应该调用哪个方法。

常用实现

  • RequestMappingHandlerMapping：处理基于 @RequestMapping 注解的映射。
  • SimpleUrlHandlerMapping：通过简单的 URL 配置映射请求。

Controller（控制器）

作用

Controller 是 MVC 模式中的核心，它接收来自 DispatcherServlet 的请求，处理业务逻辑，并将处理结果返回给视图层。Controller 通过 @RequestMapping 注解映射 URL 路径，并通过 Model 向视图传递数据。

常见用法

  • 使用 @Controller 注解定义控制器。
  • 使用 @RequestMapping 注解指定处理的请求路径。

ModelAndView（模型和视图）

作用

ModelAndView 是控制器返回的对象，包含模型数据和视图名称。模型部分包含业务数据，视图部分指定渲染该数据的视图模板。

使用方式

控制器方法可以直接返回 ModelAndView 或者视图名称，通过 Model 传递数据。

ViewResolver（视图解析器）

作用

ViewResolver 的作用是根据控制器返回的视图名称找到对应的视图文件（如 JSP、Thymeleaf 模板），并将处理结果渲染成 HTML 响应给客户端。

常见实现

  • InternalResourceViewResolver：用于解析 JSP 视图。
  • ThymeleafViewResolver：用于解析 Thymeleaf 模板视图。

HandlerInterceptor（拦截器）

作用

HandlerInterceptor 用于在请求处理的各个阶段（如控制器方法执行前后）进行拦截。它可以用于实现日志记录、权限验证等功能。

常见应用

  • 日志记录：在请求开始时记录日志。
  • 权限检查：在控制器方法执行前检查用户权限。
  • 性能监控：记录请求的执行时间。

示例：

HandlerAdapter（处理器适配器）

作用

HandlerAdapter 是 Spring MVC 中的处理器执行器，它负责调用找到的处理器（Controller）方法。它解耦了 DispatcherServlet 和具体的处理器。

工作原理

DispatcherServlet 通过 HandlerMapping 找到对应的处理器后，HandlerAdapter 会调用该处理器执行相应的逻辑。不同类型的处理器可能有不同的 HandlerAdapter，如处理注解的 RequestMappingHandlerAdapter。
