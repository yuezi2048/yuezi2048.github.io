44.@PathVariable

发表于2025-11-18|更新于2025-11-18|2.Areas🌐02.面经记录Spring
|总字数:1.1k|阅读时长:4分钟|浏览量:

@PathVariable 是Spring MVC用于在URL模板中提取变量值的注解，是为了在处理HTTP请求时，从URL路径中获取变量，绑定到控制器的参数上

如http://example.com/users/123 后端 Java 可以按下面代码接收，适合RESTful风格的URL设计

主要作用

  • 从 URL 中提取路径参数：通过 @PathVariable 注解，Spring 可以将 URL 中的某些动态部分提取出来，并将这些值传递给控制器方法的参数。
  • 动态 URL 解析：适用于 RESTful 风格的 URL 设计，常用于表示资源的唯一标识符（如 /users/{id}），这种设计符合 REST 规范，参数通常是 ID 或其他路径中的动态数据。

和@RequestParam的区别：前者是URL获取动态数据，一般是处理Restful风格URL，后者是处理表单/查询参数

高级用法：嵌套使用、可选属性、正则匹配、自动类型转换、但不像RequestParam支持默认值，但可以通过j接收Optional参数解决

扩展知识

@PathVariable 与 @RequestParam 的区别

@PathVariable 和 @RequestParam 都是用于从 HTTP 请求中提取数据的注解，但它们的用法不同：

  • @PathVariable：从 URL 路径中提取动态数据，常用于 RESTful URL 模式。URL 中的参数通常是路径的一部分，如 /users/{id}。
  • @RequestParam：从请求的查询参数中提取数据，通常用于处理表单数据或查询参数。参数在 ? 之后，格式如 /users?id=1。

RESTful API 的设计

在 RESTful API 设计中，@PathVariable 非常适合用于资源的标识符。例如，常见的 RESTful URL 结构：

  • /users/{userId}/orders/{orderId}：用于嵌套资源，如用户的订单。
  • /products/{productId}：用于访问单个产品资源。

使用 @PathVariable 使得 RESTful API 的 URL 直观易懂，并符合 REST 的语义化设计。
