53.@Conditional

发表于2025-11-18|更新于2025-11-18|2.Areas🌐02.面经记录Spring
|总字数:1.4k|阅读时长:5分钟|浏览量:

Conditional用于有条件地装配 Bean。如可以根据环境（如开发、测试、生产）或特定上下文条件动态装配 Bean，实现动态加载。

主要作用：

  • 有条件地加载 Bean：@Conditional 根据某个条件来决定某个 Bean 是否需要注入到 Spring 容器中。条件可以是操作系统类型、类路径是否存在某个类、某个属性的值等。
  • 实现动态配置：可以根据环境（如开发、测试、生产）或特定上下文条件动态装配 Bean，避免不必要的 Bean 被加载。

使用场景：

  • 在不同的操作系统或环境中注入不同的 Bean。
  • 根据应用程序的配置文件、属性值来动态启用某些功能。
  • 在条件满足时，才装配某些依赖或服务。

扩展知识

@Conditional 的使用方式

@Conditional 注解的实现依赖于 Condition 接口。使用 @Conditional 时，需要定义一个类实现 Condition 接口，并在该类的 matches() 方法中实现条件逻辑。Spring 会根据 matches() 方法的返回值（true 或 false）来决定是否加载对应的 Bean。

Condition 接口

Condition 接口是 Spring 提供的用于自定义条件的核心接口，它包含一个 matches 方法，该方法会接收 ConditionContext 和 AnnotatedTypeMetadata 参数，并返回一个布尔值，决定条件是否满足。

常见的内置条件

Spring 提供了一些常用的条件类，可以直接使用，不需要自己定义。

@ConditionalOnProperty 用于根据配置文件中的某个属性值来决定是否装配 Bean。常用于启用或禁用某些功能模块。

应用场景与实战

根据不同的环境配置装配 Bean

可以通过 @Conditional 注解根据不同的环境配置注册不同的 Bean。例如，可以为开发、测试和生产环境提供不同的数据库连接配置。

通过这样的配置，Spring 可以在不同的环境中自动选择适合的数据库源。

根据操作系统装配 Bean

可以根据操作系统类型来决定是否装配某些平台相关的服务。例如，在 Windows 上加载 Windows 特定的服务，而在 Linux 上加载 Linux 特定的服务。

结合 Spring Boot 自动配置

在 Spring Boot 中，@Conditional 常与自动配置（AutoConfiguration）结合使用。Spring Boot 中的大量自动配置都是通过 @ConditionalOnClass、@ConditionalOnMissingBean 等条件注解实现的。比如在集成不同数据库时，Spring Boot 会自动检查类路径中是否存在相关的 JDBC 驱动类，并根据条件进行相应的自动配置。

其他相关注解

@Profile

@Profile 是 Spring 中另一种条件性注解，允许根据环境（如 dev、test、prod）选择性加载 Bean。@Profile 专注于环境相关的配置，而 @Conditional 提供了更加通用的条件机制。

@ConfigurationProperties

@ConfigurationProperties 允许将外部配置文件中的属性映射到类中，并与 @ConditionalOnProperty 一起使用，根据配置动态注入不同的 Bean。
