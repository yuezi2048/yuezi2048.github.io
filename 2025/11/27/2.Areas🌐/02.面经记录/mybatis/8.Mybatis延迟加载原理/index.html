Mybatis 支持延迟加载（懒加载）

延迟加载（Lazy Loading）允许Mybatis在查询主对象时不用立即加载所有数据。可优化查询性能，减少不必要的数据加载，尤其在查询关联对象或集合时非常有用。

原理：通过 Mybatis 提供的代理对象来替代主对象，当访问具体关联的属性时，才真正触发数据库的查询，特别是在处理大数据集或者复杂对象关系时，对性能提升比较大。

内部维护一个延迟加载器ResultLoaderMap记录哪些属性需要延迟加载，进入代理对象的invoke，检查被调用方法是否是延迟加载属性的getter方法，如果是，并且该属性的值为null，执行sql查询，再setter设置到目标对象的属性中

扩展知识

延迟加载的应用场景

关联对象延迟加载：主对象的关联对象只有在被访问时才加载。例如，一个 User 对象包含 Address 对象，但只有在访问 user.getAddress() 时才查询 Address 数据。

集合延迟加载：主对象的关联集合（如 List 或 Set）只有在访问时才加载。例如，Order 对象关联 OrderItem 列表，只有在 order.getOrderItems() 被调用时，才查询数据库加载列表。

配置 Order 的懒加载，在这个配置中，当首次访问 user.orders 属性时，selectOrdersByUserId 查询会被触发来加载用户的订单。

因此，只有真正访问订单的时候，才会触发订单的查询，这就叫延迟加载，后续如果再次访问订单属性则不会再触发查询，可以直接从代理对象中获取之前缓存的数据。

配置示例

还有懒加载全局配置，在 MyBatis 配置文件（mybatis-config.xml）中，配置如下属性：

  • lazyLoadingEnabled：启用延迟加载。
  • aggressiveLazyLoading：设置为 false 时，只有在访问具体属性时才会触发加载；设置为 true 时，访问任意延迟加载的属性都会加载所有延迟加载的属性。
