5.为什么写个xml再写个DAO接口就能访问数据库

发表于2025-11-27|更新于2025-11-27|2.Areas🌐02.面经记录面试鸭mybatis
|总字数:1k|阅读时长:4分钟|浏览量:

背后的机制：**JDBC ** + 动态代理，解析XML映射，动态生成DAO接口实现类，完成SQL执行，具体过程：

  • 加载配置：启动时，在配置文件（mybatis-config.xml）加载数据库连接信息和 Mapper 映射文件（Mapper.xml）。
    • SQL解析为内部 MappedStatement 对象（ SQL 语句、参数映射规则、返回结果映射规则）
  • 动态代理实现DAO接口：Mybatis对每个DAO接口生成动态代理类MapperProxy，拦截接口方法调用
    • 具体机制就是通过方法名和参数匹配对应的MappedStatement，后续调用JDBC来执行SQL
  • JDBC执行SQL：Mybatis的SqlSession是对JDBC的封装，目标是是使用 PreparedStatement 来完成 SQL 的执行。
    • 通过XML的SQL和DAO接口方法传入的参数，生成完成SQL语句，参数通过占位符绑定到PreparedStatement
    • JDBC执行，得到ResultSet
  • 结果映射：Mybatis将JDBC查询结果，会通过ResultMap 或 resultType 映射为 DAO 接口方法的返回值类型（如 POJO、Map 或 List）。
