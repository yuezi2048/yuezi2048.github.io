66.Spring如何支持国际化

发表于2025-11-22|更新于2025-11-22|2.Areas🌐02.面经记录Spring
|总字数:1.4k|阅读时长:5分钟|浏览量:

国际化解决的思路：根据用户的语言环境Locale 动态选择对应语言的文本，具体在Spring中是 LocaleResolver 和 ResourceBundleMessageSource 实现。

实现国际化的核心步骤：

  • 定义国际化的资源文件： messages.properties （如message_zh.properties），后续由MVC加载
  • 配置解析规则LocaleResolver：确定用户的语言环境
    • 默认是请求头Accept-Language
    • 会话层获取Locale（SessionLocaleResolver）
    • Cookie（CookieLocaleResolver）
  • 配置ResourceBundleMessageSource返回消息：加载国际化资源文件，用于返回语言内容
  • 最后是应用层面：@RequestMapping 或 Thymeleaf 等模板引擎的国际化标签来实现动态内容切换
    • 如多语言的URL路径映射（配合占位符）

LocaleResolver 的不同实现

SessionLocaleResolver

SessionLocaleResolver 基于会话的 Locale 解析器，它将用户的语言环境存储在 HTTP 会话中，并在会话中持续使用该 Locale。

CookieLocaleResolver

CookieLocaleResolver 通过存储在 Cookie 中的语言设置来决定用户的 Locale，它允许用户切换语言并在后续请求中保持相同的 Locale。

AcceptHeaderLocaleResolver 基于 HTTP 请求头 Accept-Language 来解析用户的语言环境。它通过浏览器发送的语言首选项来确定 Locale。

国际化资源文件的组织

国际化资源文件（通常是 .properties 文件）应该根据语言和国家来命名。例如：

  • messages_en.properties：英语资源文件。
  • messages_zh_CN.properties：中文（中国）资源文件。
  • messages_fr_FR.properties：法语（法国）资源文件。

Spring 会根据用户的语言环境自动选择最合适的资源文件。例如，如果用户的语言环境为中文（中国），Spring 会优先选择 messages_zh_CN.properties，如果找不到，则选择 messages_zh.properties。

国际化的默认行为和回退机制

当用户请求的语言环境没有对应的资源文件时，Spring 会使用默认的语言环境资源文件作为回退机制。默认的资源文件通常是没有后缀的 messages.properties 文件。
