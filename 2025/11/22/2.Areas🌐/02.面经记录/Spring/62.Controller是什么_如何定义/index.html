62.Controller是什么_如何定义

发表于2025-11-22|更新于2025-11-22|2.Areas🌐02.面经记录Spring
|总字数:1.2k|阅读时长:4分钟|浏览量:

Controller 是MVC架构的C层，对应控制层：

  • 接收客户端请求（@ModelAttribute / @RequestBody）
  • 调用业务逻辑层Servcie
  • 数据返回给视图层（Model & View或者String） / 返回JSON或XML数据响应（@ResponseEntity）

定义 Controller 的步骤：

1）使用 @Controller 或 @RestController 注解：类上标记 @Controller 注解，表示这是一个控制器，Spring 会将其注册为一个 Spring MVC 的 Bean。如果是用于 RESTful API 的开发，可以使用 @RestController，它是 @Controller 和 @ResponseBody 的组合注解，默认返回 JSON 数据。

2）使用 @RequestMapping 注解映射请求路径：在方法上使用 @RequestMapping 或其衍生注解（如 @GetMapping、@PostMapping 等）来映射 HTTP 请求。

请求：@ModelAttribute、@RequestParam/@RequestBody/@PathVariable

返回格式：ModelAndView，string(重定向/转发)

异常处理：@ExceptionHandler、@ResponseStatus、ResponseEntity

扩展知识

Controller 的核心概念

@Controller 和 @RestController

  • @Controller：用于定义一个传统的 Spring MVC 控制器，它处理客户端请求，返回视图名，通常用于返回 HTML 页面。
  • @RestController：专用于构建 RESTful API 的控制器，等同于 @Controller + @ResponseBody。返回的内容会直接作为 HTTP 响应体，通常用于返回 JSON 或 XML 格式的数据。

示例：

@RequestMapping 和派生注解

@RequestMapping：用于映射 HTTP 请求到指定的处理方法。可以作用在类或方法上，支持配置请求路径、请求方法、请求参数等。

派生注解：

  • @GetMapping：专用于处理 GET 请求。
  • @PostMapping：专用于处理 POST 请求。
  • @PutMapping：专用于处理 PUT 请求。
  • @DeleteMapping：专用于处理 DELETE 请求。

Controller 的高级用法

处理请求参数

  • @RequestParam：用于获取 URL 中的查询参数。
  • @PathVariable：用于获取 URL 路径中的动态参数。
  • @RequestBody：用于将请求体中的 JSON 数据直接映射为对象。
