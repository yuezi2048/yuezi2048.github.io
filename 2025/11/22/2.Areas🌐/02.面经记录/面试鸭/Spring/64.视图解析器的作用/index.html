64.视图解析器的作用

发表于2025-11-22|更新于2025-11-22|2.Areas🌐02.面经记录面试鸭Spring
|总字数:409|阅读时长:1分钟|浏览量:

视图解析器（ViewResolver）是SpirngMVC的接口

作用：逻辑视图名称解析为实际的视图对象（JSP 、Thymeleaf、FreeMarker 模板等）。

具体流程：拿到控制器返回的视图名称后，查找对应的视图View，并将模型Model数据传递给视图，生成最终的 HTML 响应：

  • 接收视图名称：控制器返回一个逻辑视图名称。
  • 视图解析：视图解析器根据逻辑视图名称和配置，解析并找到实际的视图文件。
  • 渲染视图：将模型数据传递给视图对象，由视图对象生成最终的 HTML 响应。

细节：解析器实现类、优先级/默认视图、自定义视图解析器（实现ViewResolver接口，实现解析方法，Bean注册）

扩展知识

1）常见的视图解析器实现：

  • InternalResourceViewResolver：用于解析 JSP 文件视图。
  • ThymeleafViewResolver：用于解析 Thymeleaf 模板视图。
  • FreeMarkerViewResolver：用于解析 FreeMarker 模板视图。

2）视图解析器相关配置：

  • 优先级：可以配置多个视图解析器，通过 order 属性设置解析器的优先级。
  • 默认视图：可以指定默认视图解析器，当找不到匹配的视图名称时使用。

3）如何自定义一个视图解析器

第一步：创建自定义的视图解析器，实现 ViewResolver 接口的 resolveViewName 方法。

第二步：在 Spring 配置文件或类中注册自定义的视图解析器。
