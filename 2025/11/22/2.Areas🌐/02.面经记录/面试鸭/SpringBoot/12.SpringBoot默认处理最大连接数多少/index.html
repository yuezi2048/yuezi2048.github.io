11.SpringBoot默认处理最大连接数多少

发表于2025-11-22|更新于2025-11-22|2.Areas🌐02.面经记录面试鸭SpringBoot
|总字数:402|阅读时长:1分钟|浏览量:
  • tomcat能处理的最大连接数 = 最大连接数 8192 + 等待数 100
  • tomcat能处理的最大请求线程数 = 200，核心线程数10，队列长度无限（因为是IO密集型，线程池策略区别于默认JDK线程池的机制）

拿 Spring Boot 默认的容器 Tomcat 为例，从引入的 spring-boot-autoconfigure 依赖可以看出，Tomcat 默认的最大连接数是 8192 。

但这里要注意！Tomcat 还有一个默认等待数，加上这个才是 SpringBoot 默认的最大连接数。

所以 SpringBoot 默认同时可以处理的最大连接数 = Tomcat 默认的最大连接数 + Tomcat 默认等待数 = 8192 + 100 = 8292。

扩展知识

SpringBoot 默认同时可以处理多少个请求？

默认还是 tomcat 这个 web 容器。

实际上是这取决于 tomcat 的线程池配置。

tomcat 的默认核心线程数是 10，最大线程数 200，队列长度无限。但是它的线程池机制和 JDK 默认线程池不一样，因为主要是 I/O 密集型的任务，所以 tomcat 改造了线程池策略，在核心线程数满了之后，会直接创建线程到最大线程数。

所以，在默认的配置下，同一时刻，可以处理 200 个请求。 其他的请求只能排队了。

可以通过 server.tomcat.max-threads 修改最大最大工作线程数。
