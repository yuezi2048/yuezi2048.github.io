20.SpringBoot事件机制了解

发表于2025-11-25|更新于2025-11-25|2.Areas🌐02.面经记录面试鸭SpringBoot
|总字数:1.5k|阅读时长:4分钟|浏览量:

Spring Boot 的事件机制，本质上是Spring的发布-订阅模式的事件机制，用于各个组件的消息传递和解耦，简化模块化开发和维护。

如通过监听Spring应用上下文的启动和关闭事件，进行相应的初始化、清理操作，而不需要修改Spring源码

作用总结：

  • 解耦：通过事件机制，可以在不同组件之间传递消息，而不需要它们之间有直接的依赖关系，从而提高了代码的可维护性和扩展性。
  • 异步处理：某些事件可以异步处理，从而提高应用程序的响应速度和性能。
  • 状态通知：通过事件机制，可以通知应用程序的不同部分发生了某些特定的状态变化，比如启动完成、环境准备就绪等。

追问：事件的原理是什么？

基于观察者设计模式，首先通过ApplicationEventPublisher发布事件，随后监听器ApplicationListener接收到事件进行处理，如ApplicationEventMulticaster就会将事件广播给所有监听器（支持异步处理）

追问：SpringBoot内置的事件是什么？

  • ApplicationStartingEvent：在运行开始时发布。
  • ApplicationEnvironmentPreparedEvent：在环境信息已准备但上下文还未创建时发布。
  • ApplicationPreparedEvent：在上下文创建完成但未刷新时发布。
  • ApplicationStartedEvent：在上下文刷新并且应用启动完成时发布。
  • ApplicationReadyEvent：在任何相关的代码都已准备好运行时发布。
  • ApplicationFailedEvent：在启动时遇到错误时发布。

注：事件支持和事务集成，同时有不同的策略（事务提交前后/回滚 发布）

扩展知识

事件机制的工作原理

Spring 的事件机制是基于 观察者模式 实现的。在 Spring 中，事件发布者 和 事件监听器 被解耦，发布者发布事件后，事件由 ApplicationEventMulticaster 分发给所有的监听器。事件机制的主要工作流程如下：

  1. 事件发布：事件通过 ApplicationEventPublisher 发布，发布者不需要关心事件的处理细节。
  2. 事件监听：监听器（ApplicationListener）接收到事件后进行处理。Spring 的 ApplicationEventMulticaster 负责将事件广播给所有合适的监听器。
  3. 异步处理：如果监听器使用了 @Async，事件会被异步处理，从而不影响主线程的执行。

自定义事件与内置事件

除了自定义事件，Spring Boot 还提供了多个内置事件，用于监听应用生命周期中的重要节点。常见的内置事件包括：

  • ApplicationStartedEvent：应用启动时触发，在 SpringApplication.run() 开始时发布。
  • ApplicationReadyEvent：应用启动完成且准备好服务时触发。
  • ContextRefreshedEvent：应用上下文刷新时触发。
  • ContextClosedEvent：应用上下文关闭时触发。

异步事件处理的应用场景

在某些情况下，事件处理可能需要消耗较多时间（如发送邮件、调用外部服务等）。为了避免阻塞主线程，Spring 提供了异步事件处理功能。通过 @Async 注解，Spring 会在独立的线程池中异步处理事件，确保主线程的任务流畅执行。

事件机制与事务的集成

Spring 事件机制可以与 事务管理 集成，使得事件的发布行为受事务控制。例如，你可以选择在事务提交成功后才发布事件，避免在事务回滚时发布不必要的事件。

通过使用 @TransactionalEventListener 注解，开发者可以控制事件在事务的哪个阶段进行处理，常见的事务阶段包括：

  • AFTER_COMMIT：事务提交成功后发布事件。
  • AFTER_ROLLBACK：事务回滚后发布事件。
  • BEFORE_COMMIT：事务提交前发布事件。
