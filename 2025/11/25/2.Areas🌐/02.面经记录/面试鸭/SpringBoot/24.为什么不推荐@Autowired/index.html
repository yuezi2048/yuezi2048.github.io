24.为什么不推荐@Autowired

发表于2025-11-25|更新于2025-11-25|2.Areas🌐02.面经记录面试鸭SpringBoot
|总字数:548|阅读时长:1分钟|浏览量:

  • Spring官方推荐构造器注入，而不是字段注入，原因：

    • 可能有空指针异常

    • 单元测试不友好

    • 掩盖循环依赖问题

  • @Autowired 实际上 Spring 提供的，导致了业务代码和框架强绑定，如果更换给其他 IOC 框架则不适用了

    • @Resource 是 JSR-250 提供的，它是 Java 的标准，因此如果非要使用字段注入也应该使用 @Resource。

扩展知识

字段注入的缺点

需要注意空指针问题

例如上述的代码，Spring 启动后会报空指针错误！因为构造方法的执行顺序高于 @Autowired 的顺序。所以在执行这个类的构造方法时，mianshiyaService 还未被注入，因此它的值还是 null，所以报错了。

与 IOC 容器强耦合，对单元测试不友好。

字段注入使得与 Spring 强绑定，耦合的情况其实很多同学已经感受。例如在写单元测试的时候，我们必须使用 Bean 注入，不然就无法实现单测（导致很多同学写的是单测，实际上是集成测试）。

会掩盖循环依赖

  • 578. Spring 如何解决循环依赖？

容易违反单一职责原则

因为字段注入使用起来真的是太方便了，一个 Bean 依赖项逐渐增多，可能已经违反了单一职责。如果采用构造器就不会有这个问题，因为太多的依赖项会使得构造器看起来很臃肿，让开发反思一下当前的类承担的职责是否过多。

构造函数注入的优势

  • 强制依赖注入：构造函数注入确保所有必需的依赖在对象创建时就被注入，防止依赖缺失。
  • 不可变性：通过构造函数注入，所有依赖在对象创建时就被初始化，之后无法再被修改，确保了对象的不可变性。
  • 易于测试：构造函数注入使得编写单元测试更加简单，可以通过构造函数直接传入 mock 对象。
