17.Spring_Actuator

发表于2025-11-25|更新于2025-11-25|2.Areas🌐02.面经记录面试鸭SpringBoot
|总字数:1.3k|阅读时长:4分钟|浏览量:

Spring Boot Actuator 是 Spring Boot 提供的一个监控和管理功能的工具。可以监控：健康情况health、指标metrics、跟踪请求，管理环境参数env等

Actuator 提供了丰富的端点（Endpoints），用于展示和操作应用的内部状态。这些端点可以通过HTTP、JMX等方式进行访问。具有简便性、可扩展性、安全性、易于集成（举例：集成Spring Security配置访问权限，保障一些敏感断点的安全性）

它的优势：

  • 易于集成：开箱即用，简单配置即可使用，可以很容易地与外部监控系统集成，如 Prometheus、Grafana 等。
  • 丰富的端点：提供了大量预定义的端点，如健康检查（/health）、应用信息（/info）、度量指标（/metrics）等。
  • 丰富的信息：提供了关于应用、系统、JVM、内存、线程等丰富的信息。
  • 高度可扩展：可以自定义端点，扩展或修改现有的端点功能。
  • 安全性：支持端点的安全访问控制，可以通过配置限制哪些端点可以公开访问。

扩展知识

Spring Actuator 提供的端点

  • /actuator/health：显示应用的健康信息（默认情况下会检查数据库连接、消息代理等）。
  • /actuator/info：显示应用的自定义信息，如版本、描述等。
  • /actuator/metrics：显示应用的度量指标，如 JVM 内存使用情况、垃圾收集信息、线程信息等。
  • /actuator/loggers：显示和配置日志级别。
  • /actuator/env：显示应用的环境配置信息，包括系统属性、环境变量、配置文件属性等。
  • /actuator/beans：显示 Spring 应用上下文中所有的 Bean 信息。
  • /actuator/threaddump：显示应用的线程转储信息。
  • /actuator/httptrace：显示最近的 HTTP 请求和响应跟踪（默认情况下只存储最近 100 个）。
  • /actuator/mappings：显示所有的 URL 映射路径。

Actuator 的健康检查机制

Actuator 的 /health 端点是用来检查应用健康状况的核心工具。它通过整合多个 HealthIndicator（如数据库、消息队列、缓存系统等）来评估应用的健康状态。

示例：查看数据库健康状态

通过自定义 HealthIndicator，你可以根据应用的具体需求定制健康检查逻辑。

Actuator 中的度量指标

Actuator 的 /metrics 端点能够提供非常丰富的应用运行时指标，帮助开发者了解应用的性能和状态。Spring Boot Actuator 集成了 Micrometer 度量库，支持多种监控系统（如 Prometheus、Graphite、InfluxDB 等）。

常见的指标：

  • jvm.memory.used：当前 JVM 已使用的内存。
  • jvm.threads.live：当前活跃的线程数。
  • http.server.requests：HTTP 请求的统计信息，包括处理时间、请求状态码等。

如何使用自定义指标：

通过自定义度量，开发者可以监控应用中重要的业务指标或性能数据，并将其暴露到 /metrics 端点。

安全性与访问控制

Actuator 的某些端点（如 /env、/loggers 等）可能包含敏感信息，因此通常需要与 Spring Security 集成来限制访问权限。通过配置权限规则，可以确保只有授权用户可以访问这些管理端点。
