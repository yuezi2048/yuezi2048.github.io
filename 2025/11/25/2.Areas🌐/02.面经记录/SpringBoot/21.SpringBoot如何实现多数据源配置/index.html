扩展知识

1. 在配置文件中定义多数据源

首先，在 application.yml 或 application.properties 文件中为不同的数据源定义连接信息。

2. 配置 DataSource、SqlSessionFactory 和 TransactionManager

为每个数据源分别配置 DataSource、SqlSessionFactory 和 TransactionManager，并为主数据源标记 @Primary。

示例：数据源配置类

3. 配置 Mapper 扫描路径

每个数据源需要独立配置对应的 Mapper 扫描路径。使用 @MapperScan 注解指定 Mapper 的包路径，并绑定相应的 SqlSessionFactory。

4. 使用 @Transactional 指定事务管理器

在需要使用多数据源的地方，可以通过 @Transactional 注解指定对应的事务管理器。

示例：指定事务管理器

通过在 @Transactional 注解中指定不同的 transactionManager，可以确保每个数据源的事务管理独立，避免数据源之间的事务冲突。

10.SpringBoot能否用XML配置
虽然 Spring Boot 强调基于 Java 注解和属性文件的配置，但仍然支持传统的 XML 配置。 实现方式： @ImportResource 注解导入 XML 配置文件，让 Spring Boot 应用程序能够识别和使用这些 Bean。 场景：一般是公司在将 Spring 项目迁移到 Spring Boot 时，因为时间或工作量问题，才会在过渡期间混合使用两者。SpringBoot 项目不太会使用 XML 配置。
