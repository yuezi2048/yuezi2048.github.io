2.2.1 静态资源规则和定制化

根据官方的描述，可以得到springboot对静态资源的映射规则：

  • 在类路径下面定义目录
    • resources
    • public
    • static
    • META-INF/resources
  • 访问静态资源路径：项目的根目录 + 静态资源的名称（localhost:8080/a.html）

注：在springboot中，如果项目中存在同名的静态资源和同名的动态资源，我们会优先去访问动态资源，如果动态资源不存在，然后再去访问对应的静态资源，如果静态资源也找不到，则报404

  • 如果我们想自定义静态资源的映射规则（逻辑映射），springboot也给了解决方案，对静态资源的映射路径添加前缀
