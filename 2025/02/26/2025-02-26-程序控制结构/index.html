五、程序控制结构

5.1 顺序控制

细节

  • 顺序控制中的变量需要前向引用

5.2 分支控制

5.2.1 if-else

  • 单分支 if
  • 双分支 If-else
  • 多分支 if-else if -else

细节

  • 每个条件表达式都会走，直到有一个满足的，才会执行后面的代码块
  • 可以没有else，此时if-else没有执行入口
  • 最多只有一个执行入口

关注一个代码

5.2.2 嵌套分支

  • 建议嵌套不要超过3层，否则可读性很差
  • 判断好某个条件，需要进一步判断各种情况下使用

5.2.3 switch

快速入门

  • 表达式对应的是一个值
  • 逐个匹配常量，如果没有，执行default

细节

  • 表达式数据类型与常量要一致或者可以自动转化成常量，如表达式是char，而常量是int
  • 表达式返回值的数据类型必须是：byte、short、int、char、enum、String（double和float是不可以用switch判断的）
  • case必须是常量，不能是变量
  • default可以不写
  • break语句写了以后，执行后就退出，如果没有break，会继续执行后续的语句块（请注意下一步是没有判断的）

技巧：

（1）[0, 100]范围内筛选及格的可以用(int)(score / 60) 判断

（2）当要连续判断多个值且返回相同结果时，可以使用switch穿透的性质

switch和if的比较

  • 具体判断的数值不多且符合switch使用的数据类型的话，建议用switch，更加清晰可读
  • 对于区间判断，建议用if，if使用范围更广

5.3 循环控制

5.3.1 for

四要素：初始化、条件、操作、迭代

细节

  • 循环条件是boolean类型
  • 初始化和迭代可写到别的地方，分号不可省略
  • 初始值和迭代可以用逗号隔开，给出多个语句
  • for(;;) 等价于while(true)

5.3.2 while

  • 同样包含四要素
  • 只是位置与for循环不同
细节
  • 循环条件是返回一个布尔值的表达式
  • while 循环是先判断再执行语句

5.3.3 do while

细节

  • do while也是关键字
  • 先执行再循环，至少执行一次语句块
  • 最后有一个分号

5.4 多重循环控制*

  • 循环再套一层循环，建议不要超过三层循环影响可读性。
  • 内层循环结束一轮后，外层循环完成一趟
  • 注意内存分析法对复杂流程进行分析

5.5 跳转控制

5.5.1 break

需求场景：随机生成 1-100 的一个数， 直到生成了 97 这个数， 看看你一共用了几次?
提示使用 (int)(Math.random() * 100) + 1 （random生成[0, 1)的随机数，使用乘法和加法可指定范围）

  • 使用break终止语句块

break使用标签进行指定语句块的跳转

  • break后加上标签指定
  • 标签命名规则同变量符规则
  • 尽量开发中不用标签
  • 没有指定则跳出最近的循环体

ps：判断字符串相等，建议常量写前面防止空指针异常

5.5.2 continue

  • continue 语句用于结束本次循环， 继续执行下一次循环
  • 同样地，continue也可以使用标签进行跳转

5.5.3 return

  • 方法中跳出所在的方法，在main方法中退出程序

5.6 重要编程思想*

  • 化繁为简：复杂问题拆分成若干简单的子问题
  • 先死后活：先特殊情况的常量 再推广到一般情况下的变量

以输出空心菱形为例说明该思想的作用。
