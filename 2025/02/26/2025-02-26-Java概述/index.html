二、Java概述

2.1 Java特点

2.1.1 Java历史

程序：计算机编写的一系列有序指令的集合

诞生历史

  • Java8、Java11为LTS（长久支持版本），重点学习

  • 现使用Sublime感受Java语言学习，巩固一些基础的代码编写与规范，后续使用idea或Eclipse。

2.1.2 Java语言的特点

  • 面向对象（oop）
  • 健壮性（强类型机制、异常处理、垃圾回收）
  • 跨平台（本质上是编译好的class字节码文件可以通过各个操作系统的JVM运行）
  • 解释性语言（同js,php），区别于C/C++编译性语言，java编译后的代码不能直接被机器执行，需要jvm来解释，而编译性语言可以。

2.1.3 JDK、JRE、JVM*

JDK（Java Development kit 开发工具包）

  • JDK = JRE + Java开发工具（包括java、javac、javadoc、javap等）
  • JDK提供给开发人员进行Java开发，JDK本身包含JRE运行环境

JRE（Java Runtime Environment 运行环境）

  • JRE = JVM + Java核心类库
  • 如果只需运行编译好的class文件，只需JRE

JVM（（Java virtual Machine 虚拟机）

  • 虚拟计算机，执行指令，管理数据、内存、寄存器，包含在JDK和JRE中
  • 屏蔽了底层的平台差别，”一次编译，到处执行”

JDK下载地址

  • 下载后，将JDK目录下的bin目录加入path环境变量中（建议使用JAVA_HOME变量存储JDK路径，从而调用相对路径%JAVAHOME%复用）
  • DOS命令的执行原理是首先在当前目录查找程序是否存在，不在的话查找path环境变量
  • 使用java -version检查是否能正常使用

细节部分：

环境变量对应管理员用户和所有用户的使用权限

2.2 学习方法

  • 以需求为导向，联系之前学过的传统技术，思考改进之处
  • 先学大致的语法原理，快速动手简单的demo，然后再学习技术的细节
  • 细节的学习永无止境

2.3 Java快速入门

2.3.1 Java运行机制*

执行javac -encoding UTF-8 Hello.java后，产生三个class文件

运行不同的类名，会执行不同的入口函数

可以得出的重要结论：

  • Java开发步骤：java –> class –> 结果

  • 一个文件最多一个public类，如果有，则文件必须和该类的名字匹配。
  • Java也允许指定运行非public类的main方法
  • 注意良好的习惯：括号成对、大小写区分、分号等。

2.3.2 转义符

\t ：tab

\n ： 换行

\\，\“，\‘ 防止引号闭合，所以用到转义

\r：回车，注意没有换行

  • System.out.println(“韩顺平教育\rhsp”); 结果是hsp教育

2.3.3 注释

单行多行注释（多行注释不能嵌套）

文档注释

javadoc生成文档注释

javadoc -d d:\\temp -author -version Comemt.java

2.3.4 代码规范*

  • 类和方法用javadoc注释
  • 非javadoc注释给维护者看的，主要告诉他们为什么这样写，注意什么问题，如何修改
  • tab缩进 shift+tab 左移
  • 运算符两边习惯加空格
  • utf-8 编码
  • 行宽不超过80
  • 代码风格：次行风格（建议）或行尾风格

2.4 DOS命令了解

DOS：Disk Operating System 磁盘操作系统

2.4.1 相对路径和绝对路径

  • 从当前文件夹下开始就是相对，从根目录开始就是绝对

2.4.2 常见命令

  • dir
    • dir d:\abc2\test200
  • cd（change directory）
    • cd ..
    • cd \
    • cd d:\abc2\test200
    • cd /D c:
  • tree
  • cls
  • exit
  • md rd（创建删除文件夹） copy del echo（输出文件） type move（剪切）
    • 大致了解，重点关注linux命令
