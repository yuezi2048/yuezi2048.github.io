三、AOP

3.1 转账的事务问题与解决思路

继续上一章节转账案例的进度，当前进度：

  • 通过注解或者XML配置文件的方式配置了IOC
    • 对数据源、queryRunner核心对象、以及DAO层Service层关键对象 进行bean对象管理
  • 实现了具体的业务代码，关键业务是AcoountServiceImpl中的transfer转账业务

当前进度存在的问题：没有实现事务

这会造成一个问题：当业务操作处理过程中，出现了问题，那么就不满足事务的一致性，一个扣钱了，另外一个人却没有加钱，这是非常严重的问题。

造成该问题的根本原因：

  • 没有定义事务；
  • 数据库链接对象不统一，进而造成数据库的操作不在同一个事务中。

解决方案：将数据库的CRUD操作的连接对象统一即可，具体来说三个步骤

  • 通过ThreadLocal将当前线程和数据库连接对象绑定
  • 定义事务的管理器，包含开启事务、提交事务、回滚事务、释放资源的方法
  • 然后具体的执行业务的方法应当修改成如下方式

3.3 代理分析

通过上述步骤，可以解决事务的问题，但是引出一个新问题，在当前进度下，我们将业务代码和事务代码写在了一起，违背了职责单一性的设计原则

在给出解决方案前，给出一个例子：

  • 厂家原本需要完成生产、销售、售后三个服务，但是厂家应该专心于生产，此时出现了违背职责单一性原则
  • 而进行优化后，厂家通过找了代理商，专门进行销售和售后服务，此时商家就可以专心做生产了
  • 从另外一个角度来看：通过动态代理，增强了生产厂家对象的功能

引出解决思路：可通过动态代理的思想解决：

  • 特点：代理对象在程序运行过程中创建
  • 作用：在不修改源码的基础上对方法增强
  • 分类：
    • 基于接口的动态代理——jdk
    • 基于子类的动态代理——cglib

3.3.1 基于接口动态代理 (jdk)

我们要实现基于接口动态代理，可使用JDK官方提供的Proxy类：

  • 创建代理对象前提：被代理类至少要实现一个接口，否则无法使用

  • 创建代理对象：newProxyInstance方法，参数说明

    • ClassLoader：类加载器，用于加载对象字节码，写被代理对象的类加载器，固定写法
    • Class[]：字节码数组，用于让代理对象和被代理对象拥有相同方法（实现同一个接口即可实现），写接口字节码文件，固定写法
    • InvocationHandler：处理器。定义增强的代码，写该接口的实现类，通常是匿名内部类

3.5 AOP

3.5.1 AOP概念

AOP(Aspect oriented Programming) 面向切面编程，是OOP的扩展和延申

  • 主要是通过横切技术，解剖封装对象内部，将影响多个类的公共行为封装到一个可重用模块，这就是切面（aspect）的由来
    • 理解：将与业务无关却可以供业务调用的行为提取封装，减少重用，进而起到在逻辑处理中的各部分之间降低耦合的隔离效果（前面的添加事务就是一个很好的例子）
  • AOP采取横向抽取机制，取代传统纵向继承体系的重复性代码

3.5.2 AspectJ框架

3.5.2.1 AspectJ简介

  • AsPectJ是springAOP正在使用的开发方式，SpringAop底层采用的就是动态代理技术
  • 我们之前手写了一个AOP小demo，体会到动态代理（jdk/cglib）配置过于繁琐且固定，因此spring封装了第三方的AspectJ框架。

3.5.2.2 AOP术语

给出AOP的关键术语

  • JointPoint: 连接点，即有可能被增强（也叫拦截）的方法，CRUD方法都可以成为连接点
  • PointCut: 切点，真正被增强的方法，比如只对save方法增强
  • Advice: 增强或者通知，应用在切点方法上的增强方法，例如权限管理，事务管理的方法
  • Target: 目标类。连接点方法所属类就是目标类，比如这里的UserDao类
  • Introduction: 引介，是 类层面上的增强（实际开发做的比较多的是方法的增强）
  • Proxy: 代理对象，代理对象将增强方法应用在切点上，
  • weaving：织入，描述通知Advice 应用到目标类Target切点方法PointCut 的过程，比如权限校验加入到save方法的过程
  • Aspect：切面类，增强方法所属的类就是切面类

注：通知和切入点的关系是多对多的

对应的XML代码，说明：

  • 首先导入目标类和切面类作为bean对象
  • 接下来通过aop:config标签配置AOP相关
    • 通过aop:pointcut标签配置目标类上的切点pointcut
      • 传入id和expression切点表达式：返回值类型 包名+类名+方法(参数类型)
    • 通过aop:aspect配置切面类上的代理对象aspect
      • 传入ref，即切面类的id
      • 并在代理对象内配置应用到切点上的通知（参数传入method和切点pointcut-ref属性），根据通知类型分为以下标签
        • 前置通知：aop:before，表示目标方法执行之前进行增强
        • 后置通知：aop:after-returning，表示目标方法执行之后进行增强
        • 环绕通知：aop:around，表示目标方法执行前后进行增强
        • 抛出异常通知：aop:after-throwing，表示目标方法执行出现异常时增强
        • 最终通知：aop:after，表示目标方法是否出现异常，最终都会执行
