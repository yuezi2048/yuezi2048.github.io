所以动态规划中每一个状态一定是由上一个状态推导出来的，这一点就区分于贪心，贪心没有状态推导，而是从局部直接选最优的。

DP的五部曲

  1. 确定dp数组（dp table）以及下标的含义
  2. 确定递推公式
  3. dp数组如何初始化
  4. 确定遍历顺序
  5. 举例推导dp数组

找问题的最好方式就是把dp数组打印出来，看看究竟是不是按照自己思路推导的！

做动规的题目，写代码之前一定要把状态转移在dp数组的上具体情况模拟一遍，心中有数，确定最后推出的是想要的结果。

这样才是一个完整的思考过程，而不是一旦代码出问题，就毫无头绪的东改改西改改，最后过不了，或者说是稀里糊涂的过了。

DP的灵魂三问：

  • 这道题目我举例推导状态转移公式了么？
  • 我打印dp数组的日志了么？
  • 打印出来了dp数组和我想的一样么？

基础题目

509. 斐波那契数

dp数组含义：第i个下标对应的斐波那契数

递推公式：dp[i] = dp[i - 1] + dp[i - 2]

初始：dp[0] = dp[1] = 0;

递归顺序：从左往右

模拟： 0 0 1 2 3 5 …

70. 爬楼梯

dp数组：爬到第i个位置的方法数（i >= 1，不应该用0下标，因为不符合dp数组的意义且n范围不在0的范围内 解释起来会比较牵强）

递推公式：dp[i] = dp[i - 1] + dp[i - 2]

初始：dp[1] = dp[2] = 1;

递归顺序：从左往右

模拟：1 1 2 3…

拓展1：如果每次是最多爬m个台阶（0<= m < n）呢 ？—> 完全背包问题

拓展2：因为是通过前面推出来的，所以可以进一步优化空间复杂度到O(1)

746. 使用最小花费爬楼梯

dp数组：到达下标i位置的最小代价为j

递推公式：Math.min(dp[i - 1] + cost[i - 1], dp[i - 2] + cost[i - 2]);

初始 dp[0]=0 dp[1]=0

递归顺序：从左往右

62. 不同路径

dp数组：从dp[1][1]到dp[i][j]的路径数

递推公式： dp[i][j] = dp[i - 1][j] + dp[i][j - 1];

初始 dp[i][0] = 0, dp[0][j] = 0 dp[1][1]=1

递归顺序：i，j正常顺序遍历

引申：暴力解法是dfs暴力搜索向右和向下的方向，巧妙的解法是数论知识求组合数$C_{m+n-2}^{m-1}$（总步数中m-1步是必定往下走的）

优化：通过滚动数组进行状态压缩

1049. 最后一块石头的重量 II

本题关键是分析出要让石头分成重量相同的两堆（尽可能相同），相撞之后剩下的石头就是最小的。

所以要求得在容量为sum / 2的范围内，能装下的最大的数字（value），然后和sum - dp[m]进行作差以后就是最小的

494. 目标和

第一次做的时候判断，单纯看正数的方法数

实际上可以先推导得到正数的范围是 (target + sum) / 2，这样可以优化数组空间，且无需统计，直接返回最终数值即可

求装满背包有几种方法的情况下，公式：dp[j] += dp[j - nums[i]];

假设输入数组为 [1, 1, 1, 1, 1]，目标值为 3， 假设加法的总和为x，那么减法对应的总和就是sum - x，此时 x - (sum - x) = target –> x = (target + sum) / 2：

更新示例

  1. 计算总和 sum = 5。
  2. 计算目标和 targetSum = (5 + 3) / 2 = 4。
  3. 初始化 dp 数组为 [1, 0, 0, 0, 0, 0]。
  4. 逐步更新 dp 数组：
    • 第一次遍历后：dp = [1, 1, 0, 0, 0, 0]
    • 第二次遍历后：dp = [1, 2, 1, 0, 0, 0]
    • 第三次遍历后：dp = [1, 3, 3, 1, 0, 0]
    • 第四次遍历后：dp = [1, 4, 6, 4, 1, 0]
    • 第五次遍历后：dp = [1, 5, 10, 10, 5, 1]
  5. 最终结果为 dp[4] = 5。

讲解完全背包的时候，还会用到这个递推公式！

多重背包

有$N$种物品和一个容量为$V$ 的背包。第i种物品最多有$M_i$件可用，每件耗费的空间是$C_i$ ，价值是$W_i$ 。

求解将哪些物品装入背包可使这些物品的耗费的空间 总和不超过背包容量，且价值总和最大。

理解：01背包的拓展版本，每个物品能用Mi次，看成是有Mi个物品（空间和价值相同）
