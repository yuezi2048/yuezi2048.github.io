55.Object类的方法

发表于2025-04-26|更新于2025-04-26|2.Areas🌐02.面经记录Java基础
|总字数:615|阅读时长:1分钟|浏览量:

具体如下：

  • 作用：比较两个对象是否相等，默认是比较两个对象内存地址
  • 使用：通常重写该方法来比较对象的内容
  • 作用：返回对象哈希码，是对象的整数表示，其用于支持基于哈希的集合HashMap、HashSet等
  • 使用：如果重写了equals方法，那么必须重写hashCode方法，保证相等的对象有相同的hash码
  • 作用：返回对象字符串，默认返回类名+十六进制哈希码
  • 使用：重写该方法，定义更有意义的描述
  • 作用：返回对象的运行时类（Class对象），该方法被final修饰，不能被重写
  • 使用：获取对象的类信息，常用于反射
  • 作用：唤醒对象的监视器上等待的进程，该方法必须在同步块/同步方法中调用
  • 使用：多线程场景下线程间的通信与协调
  • 作用：唤醒对象的监视器上等待的所有进程，该方法必须在同步块/同步方法中调用
  • 使用：与notify类似，唤醒所有的线程，用于处理多个线程之间的协作
  • 作用：使线程等待，直到其他线程调用notify()或者notifyAll()方法，需要在同步块或者同步方法中调用
  • 使用：线程间的通信，线程会等待被唤醒或超时唤醒
  • 作用：使线程等待，直到指定的时间到期或者被唤醒，超时后会自动被唤醒
  • 使用：带有超时功能的等待机制
  • 作用：使线程等待，直到指定的时间和纳秒数到期或者被唤醒，超时后会自动被唤醒
  • 使用：更加精细的超时功能的等待机制，精确到纳秒
  • 作用：创建并返回当前对象的副本，默认实现是浅拷贝
  • 使用：通常会重写方法实现深拷贝，保证克隆对象的完整性
  • 作用：垃圾回收器确定不存在对象的引用时调用，用于资源释放等清理工作
  • 使用：（不建议使用），依赖垃圾回收器，会有不确定的性能问题。推荐try-with-resources和AutoCloseable接口来进行资源管理

Java字节码 是Java编译器将源代码编译后生成的中间表示形式。 介于Java源代码和JVM执行的机器码之间 Java字节码会交给JVM解释或者即时编译（JIT）成机器码执行 拓展 字节码结构 Java字节码是与平台无关的指令集，存储在.class文件中，每个.class文件包含类的定义信息、字段、方法以及方法对应的字节码指令 字节码指令集 包括加载、存储、算数运算、类型转换、对象操作、控制流等。常见的包括aload, iload, astore, iadd, if_icmpgt 执行过程 JVM通过解释器逐条执行字节码。或者通过JIT编译器将热点字节码即时编译为机器码，提高执行效率 反射和动态代理 Java的反射API可以在运行时动态生成或修改字节码，从而创建代理对象或者实现动态方法的调用 字节码增强 框架 Hiberrnate、Spring AOP等Java框架使用字节码增强技术，通过修改类的字节码来实现功能增强，包括ASM、Javassist、CGLIB等
