36.JDK9为什么将String的char数组改为byte数组
为了节省内存空间，提高内存利用率。 JDK9之前，String类是基于char[]实现，内部是UTF-16编码，每个字符使用2个字节。但是如果一些字符只需要1个字节（如Latin-1字符），那么就会空间浪费。 JDK9之后做了优化，采用byte数组优化，ASCII字符串等单字节字符都可以通过byte[]存储，减小空间占用 并引入coder变量标识编码方式（Latin-1或UTF-16），如果字符串只有Latin-1的字符（如ASCII），那么就单字节编码，否则UTF-16，保证兼容性的同时，能减少空间占用 拓展 Latin1是单字节编码，ISO-8859-1的别名，其编码范围是0x00-0xFF，共256个，收录了西欧语言等文字符号，ASCI编码只用到7位，是0x00-0x7f Latin1使用实际上是比较广泛的，如MYSQL8之前（MySQL5.6）数据表默认就是Latin1
