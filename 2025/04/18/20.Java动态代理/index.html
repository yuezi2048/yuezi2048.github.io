20.Java动态代理

发表于2025-04-18|更新于2025-04-18|面试Java基础
|总字数:799|阅读时长:2分钟|浏览量:

动态代理：Java运行时创建代理对象的机制，不需要在编译阶段确定。通过代理模式提供了一种机制，实现了在不修改目标对象的基础上，对其行为进行增强 / 修改。

  • 代理：可理解为对调用目标的包装，在调用前进行的一些逻辑处理，实现职责单一性的原则
  • 静态代理：相对于动态代理的运行时生成代理类，静态代理则是预先就编码好了代理类

动态代理用途

  • 简化代码：减少重复代码，特别是横切关注点（事务管理、日志、权限等）的方面
  • 增加灵活性：使代码更具灵活性和扩展性，因为其运行时生成对象的动态特性
  • 实现AOP：动态代理是实现面向切面编程AOP的基础，在方法调用前后可以插入额外的逻辑

实现动态代理

  • jdk基于接口动态代理：只对接口进行代理，不对类代理
  • CGLIB代理：通过字节码动态生成目标类的子类实现代理，支持对类（非接口）进行代理i
