十四、集合

14.1 Collection

14.1.1 为什么要有Collection集合

数组有不足的地方：

  • 长度必须在定义的时候指定，不可更改
  • 数组只能存储同一类型的数据
  • 使用数组进行添加或者删除，效率低
    • 需要创建一个新的数组对象，然后遍历赋值，再把引用指向新的数组（繁琐）

而集合具有如下特点：

  • 动态保存任何对象
  • 提供了操作对象的方法：add、remove、set、get
  • 使用集合添加，删除元素代码简洁

14.1.2 集合的框架体系（背）

  • Collection（单列集合）

  • Map（双列集合）

因为接口不能实例化，需要通过实现子类List或者Set实现，这里先用ArrayList进行演示…

  • Collection实现子类可以存放多个Object元素
    • list: 可以存放重复元素（有序集合）
    • set: 不允许存放重复元素（存放顺序有些有序有些无序，具体细节后面再聊）
  • 常用方法
    • add、remove、contains、size、isEmpty、clear
    • 多元素操作：addAll、containsAll、removeAll

14.3 List接口实现类——ArrayList & Vector

14.3.1 快速入门

  • Permits all Elements, including null —— ArrayList允许加入null，并且支持多个null
  • ArrayList底层是数组存储
  • ArrayList基本等同Vector，一点不同：ArrayList线程不安全(执行效率高)，多线程不建议用ArrayLIst

14.3.2 *底层源码剖析(1.5倍)

先说结论

  • ArrayList中维护的是transient Object[] elementDate (transient表示短暂的，表示该属性不会被序列化)
  • 创建ArrayList对象时，如果是无参构造器，初始化elementDate容量为0
    • 扩容机制：第一次扩容为10，再次扩容扩容为原来的1.5倍 (0->10->15->22->33…)
  • 如果使用的是指定大小的构造器ArrayList(8)，初始化elementDate容量为size
    • 扩容机制：如果长度不够了，直接扩容为原来的1.5倍**(8->12->18->27->40…)**

代码debug最后一次添加后的效果。最后一共30个元素，容量为40

14.3.3 Vector底层源码剖析(2倍)

  • 底层也是一个数组 protected Object[] elementData
  • Vector线程同步，线程安全，因为他的操作方法有synchronized

14.3.4 ArrayList Vs Vector

  • ArrayList效率相比Vector高，但Vector保证线程安全
  • ArrayList和Vector无参都是默认10
    • ArrayList是1.5倍扩容，而Vector是2倍扩容

14.4 List接口实现类——LinkedList

14.4.1 LinkedList源码剖析

  • LinkedList底层实现的是双向链表
    • first last指向首结点和尾节点
    • 每个结点(Node对象)维护了prev、next、item属性
    • 元素的添加和删除相对数组效率较高，但get，set方法因为不能通过索引访问，需要遍历一遍链表 那效率就比较低
  • 符合list的特征，可以添加多个重复元素，包括null
  • 线程不安全

输出时的内存情况

14.4.2 ArrayList Vs LinkedList

  • 增删多–>LinkedList
  • 改查多–>ArrayList（场景更常见）
  • 当然增删和改查可以分成不同的模块用不同的数据结构实现，要看具体业务

14.6 Set接口实现类——HashSet

14.6.1 快速入门

  • HashSet实现了Set接口
    • 可以存放null，但只能有一个null
    • 不保证存入和取出顺序一致
    • 不能有重复元素
  • HashSet本质上是HashMap（见源码分析）

14.6.2 添加相同元素案例分析

14.6.3 *源码剖析

结论：

  • HashSet 底层是 HashMap
  • 添加一个元素时，先得到hash值，会转成->索引值
  • 找到存储数据表table，看这个索引位置是否已经存放的有元素
    • 如果没有，直接加入
    • 如果有,调用 equals 比较，如果相同，就放弃添加，如果不相同，则添加到最后
  • 在Java8中,如果一条链表的元素个数到达 TREEIFY THRESHOLD(默认是8)，并且table的元素个数>=MIN TREEIFY CAPACITY(默认64),就会进行树化(红黑树)

补充：hashMap的数据结构示意图

hashMap结构解读：

  • hashSet维护了table表，由HashMap$Node组成的数组表示
  • 提供了关于数组+链表+红黑树的关键属性
    • 元素个数size, 门限值threshold, 加载因子loadFactor，修改计数 modCount
  • 提供获取K-V的关键属性
    • entrySet， 键keySet，值values(null)

14.6.4 细节1——扩容和树化机制

源码其实已经分析过了，这里只需总结验证即可

  • 有上限值capacity和门限值threshold（capacity * loadFactor），如果长度到达门限值，则会进行两倍扩容
    • 第一次添加开始后的变化情况： 16(12)->32(24)->64(48)…….括号里面是门限值
  • 如果链表的元素长度大于8，并且table的长度大于等于64，则进行树化，否则采用数组扩容机制

扩容debug

当size到达门限值12，再次添加后，扩容为32(24)的长度

树化debug

  • 当size大小到达8以后，此时table长度为16，不满足树化条件，因此table长度扩容为32，加入该元素
  • 当size大小到达9以后，此时table长度为32，不满足树化条件，因此table长度扩容为64，加入该元素
  • 当size大小到达10以后，此时table长度为64，满足树化条件，则对第1个索引的链表做红黑树树化处理，加入该元素后size为11

  • 注意细节：因为是&&，所以他会优先看hash值，再看相等

14.6.6 细节3——hash值修改导致的bug

  • 执行remove(p1)以后，会首先查找下标，因为p1的hash值变了，导致remove根据hash值查找到的索引错误，删除失败
  • 加上一个CC后，他的hash值和原来的p1又不在一个索引，所以能正常插入
  • 加上一个AA后，他的hash值虽然和o1是一个索引上了，但是根据hash和equals都不同，因此会认定为不是同一个对象，所以会正常插入到链表的尾部

14.7 Set接口实现类——LinkedHashMap

本质其实和HashMap一致，主要是引入了数组+双向链表维护了元素的次序问题，扩容机制和hashSet相同，简单的说明其特点即可。

  • HashSet本质上是LinkedHashMap（见源码分析，维护的是数组+双向链表）
  • LinkedHashMap实现了Set接口那一套：不能有重复元素
  • 虽然是根据hash值来决定对应的存放位置，但LinkedHashMap不同的是使用双向链表维护了元素的次序
    • 其他Set是不保证取出顺序 和 插入顺序的一致性

给出双向链表的示意图

可以得到LinkedHashSet的关键结构，这里就说一下重要的点，细节见源码

  • table同样是HashMap$Node类型，这个是和hashMap一致的
  • 不同的是，存储链表结点的类型是LinkedHashMap$Entry，他继承了HashMap.Node这个静态内部类
    • 从源码可以看到，为了维护一个双向链表，LinkedHashMap$Entry类 扩展了before after head tail等等属性，插入的时候也会有相应的双向链表操作

14.9 Map接口实现类——HashMap

14.9.1 HashMap特性

  • Map的实现类：HashMap HashTable Properties
  • HashMap使用频率最高
  • 拥有Map的一般特性
    • 使用key-value存储数据，key不能重复，value可以重复（对null同样生效）
    • 使用相同的key，value会覆盖替换
  • 和HashSet一样，不保证映射的顺序，底层是以hash表的方式存储的（jdk8底层：数组+链表+红黑树）
  • HashMap没有实现同步，线程不安全，方法没有做同步互斥处理，没有synchronized

14.9.2 源码分析

追源码前，先说结论：机制同HashSet(HashSet就是将value设置为null，本质上用的也是HashMap)

  • 底层维护的是Node[] table，默认为null(注：HashMap$Node implements Map$Entry，Map$Entry提供getKey getValue方法)
  • 创建对象时 加载因子loadFactor=0.75
  • 添加key-val时，通过key的哈希值得到table的索引
    • 如果没元素，直接添加，否则判断该元素key与待添加的key是否相等
    • 如果相等，就替换val，否则进一步对链表/红黑树的后续结点 逐个进行判断
    • 如果没有找到，就加入新元素(容量不够需扩容resize)
  • 初始，capacity=16 threshold=12，后续->32(24)->64(48)…
  • java8开始，一条链表元素超过TREEIFY_THRESHOLD(8) 并且table长度>=MIN_TREEFY_CAPACITY(64) 就树化为红黑树
    • 剪枝：红黑树回退为链表

14.9.3 细节——扩容和树化机制

机制同HashSet

  • 扩容：扩容初始为16，门限为容量的0.75倍，元数数量size到达门限值后，则扩容为原来的两倍
  • 树化：当链表长度大于8时，并且table长度大于等于64时，才进行树化，否则进行扩容

14.12 如何选择集合实现类（背）

首先要看我们的场景是单列（一组对象）还是双列（键值对）

  • 单列：Collection接口
    • 允许重复：List实现类
      • 增删多：LinkedList（底层是双向链表）
      • 改查多：ArrayList（底层是数组）
    • 不允许重复：Set实现类
      • 无序：HashSet（底层就是HashMap，JDK8以后维护的是数组+链表+红黑树）
      • 插入和取出顺序一致：LinkedHashSet（底层是数组和双向链表）
      • 排序：TreeSet
  • 双列：Map接口
    • 键无序：HashMap（JDK8以后维护的是数组+链表+红黑树）
    • 键插入和取出顺序一致：LinkedHashMap
    • 读取文件：Properties
    • 键排序：TreeMap

    • Object max(Collection, Comparator):根据 Comparator 指定的顺序,返回给定集合中的最大元素

    • Object min(Collection)

    • int frequency(Collection，Object):返回指定集合中指定元素的出现次数
