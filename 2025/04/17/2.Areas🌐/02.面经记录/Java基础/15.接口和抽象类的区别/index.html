15.接口和抽象类的区别

发表于2025-04-17|更新于2025-04-17|2.Areas🌐02.面经记录Java基础
|总字数:546|阅读时长:1分钟|浏览量:

可以从他们设计的初衷上追溯。

  • 接口是自上而下的，我们基于类的行为约束定义了接口方法，一些类需要实现这些行为才需要定义这个接口

  • 抽象类是自下而上的，很多可重用的代码的公共逻辑封装为一个抽象类，减少代码冗余

这里的自上而下就是先约定接口规范，再写不同的实现类，自下而上是先有不同的类，再定义一个抽象的类

其他的一些细节区别：

  • 方法实现：
    • 接口方法默认是public和abstract的（java8以后的默认方法和静态方法除外）
    • 抽象类是可以定义abstract方法也可以定义具体实现的方法，允许子类继承 / 重用抽象类的方法实现
  • 构造函数 / 成员变量：
    • 接口没有构造函数，并且接口的变量是public static final ，即常量
    • 抽象类可以有构造函数，成员变量也可以被任何修饰符修饰，可以不是常量
  • 多继承：接口可以有多个实现，但是抽象类只能单继承
    • Java8前可以的原因：保证每个方法都没有实现，全部交给实现类定义，不会存在菱形继承的问题
    • Java8之后定义默认方法可以多个实现的原因：对实现类实现多个类的相同方法时做了约束，必须重写方法

拓展：

  • Java8：接口引入static和default方法，让接口可以提供一些具体的实现并且不会影响已经实现接口的类
  • Java9：引入private方法，便于接口内部逻辑复用
  • Java14：引入sealed接口（只能在某些子类使用），增强接口的功能

实际案例：

JavaEE 中Filter（GOF设计模式责任链的典型案例）接口中init和destroy方法定义了default方法，这使得我们实现该接口时并不是必须实现这两个方法
