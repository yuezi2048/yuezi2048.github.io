3.Exception和error

发表于2025-04-14|更新于2025-04-14|2.Areas🌐02.面经记录Java基础
|总字数:355|阅读时长:1分钟|浏览量:

Exception和Errror都是Throwable的子类，来表示程序运行时发生的异常，总体来看，Exception表示可处理异常，Error表示不可恢复错误

  • Exception: 一般是逻辑或外部环境异常，可通过代码进行恢复，包括IOException、SQLException、NullPointerException、IndexOutOfBoundsException等
    • 具体可以分为编译时异常Check Exception和运行时异常UnCheck Exception
      • 编译时异常在编译时必须显式处理，比如使用try-catch，如IOException
      • 运行时异常，不需显示捕获，继承RuntimeException
  • **Error： ** 通常是JVM层级的，无法通过代码恢复，比如内存耗尽OutOfMemoryError、栈溢出StatckOverflowError等，Error不应该被捕获或处理，因为一旦出现程序就无法运行

拓展：

  • 尽量捕获特定异常，团队开发的时候大家清楚
  • 捕获到的异常需要记录，否则后续bug不知道是为什么出现的。
    • 在实际开发时不要用e.printStackTrace()，将详细信息输出到日志系统中，便于后续排查
  • 处理异常尽量早处理，避免堆栈叠的太多
  • try-catch范围能小则小，会影响JVM对代码的优化
  • 能用条件判断解决的，就不要上异常，因为CPU分支预测的优化，异常往往是比条件语句低效的
  • final部分不要处理返回值，会覆盖try部分的返回值逻辑，出现未知错误

2.不可变类
不可变类：对象创建后，字段所有属性都无法被修改的类。保证对象在整个生命周期保持不变 细节： 类被final修饰，防止被子类继承 所有字段被final和private修饰，保证初始化后不可修改字段 使用构造函数初始化所有字段 不提供任何修改字段的方法（setter） 类中如果包含可变对象的引用，要确保这些引用在对象外部无法被修改（例如getter可以返回对象的副本来保护可变对象） 常见不可变类：String、Integer、BigDecimal、LocalDate等 优势： 保证线程安全 安全地被缓存和共享。例如String的字符串常量池 防止状态不一致 缺点：性能问题。每次更新必须创建一个新对象（如String的频繁拼接场景） 拓展： String本质是一个被final修饰的char数组，表示数组指向的地址是不可变的 在此基础上使用private修饰，保证外部无法对value直接进行修改（如果一定要修改，只能返回新对象）
