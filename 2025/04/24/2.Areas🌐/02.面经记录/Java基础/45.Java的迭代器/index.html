45.Java的迭代器

发表于2025-04-24|更新于2025-04-24|2.Areas🌐02.面经记录Java基础
|总字数:616|阅读时长:2分钟|浏览量:

迭代器(Iterator)是Java集合中遍历元素的接口，允许开发者依次访问集合中的每一个元素，无需关心集合的具体实现，即提供了统一的方式来遍历List、Set等集合，通常配合Collection类接口使用

核心方法

  • hasNext()：为true表示还有元素，false表示遍历完毕，退出while循环
  • next()：返回集合的下一个元素，如果没有会抛出NoSuchElementException
  • remove()：移除最近一次通过next()方法返回的元素，必须在next()调用后使用。不是所有实现支持该操作，如果不支持，抛出UnsupportedOperationException

作用

  • 对不同的类型集合用统一的方式遍历元素，避免操作索引，提高可读性和可维护性
  • 支持遍历中动态修改集合内容（如remove方法，for-each是不允许的）

拓展

Iterator对比for-each

关系：for-each是Iterator的简化形式，底层还是Iterator实现

for-each只能遍历集合，不能执行删除等操作，如果要修改集合内容，需要用Iterator

原因：fail-fast机制。Iterator遍历集合，会检测集合是否发生非迭代器自身的修改（如List::add, List::remove等方法），如果监测到修改，抛出ConcurrentModificationException。所以修改集合时，应使用Iterator的remove方法而不是直接修改集合

it的缺陷：

  • 只能单向向后遍历，不能向前遍历
  • remove方法只能删除最近一次方法获取的元素，灵活性有限

所以出现了例如ListIterator等子类来完善

ListIterator

（Iterator子接口，专门操作List类型的接口）

在原有基础上，支持双向遍历和元素修改，核心方法如下

  • hasPrevious()：是否有上一个元素
  • previous()：上一个元素
  • set(E e)：当前元素替换为指定元素
  • add(E e)：当前迭代位置之前插入元素
