目标

  • 建立一个统一管理用户的系统（集中管理的可以分离出来，如支付宝登录）
  • 理清做项目思路

做项目的流程

  • 需求分析
  • 设计（概要设计、详细设计）
  • 技术选型（理由）
  • 初始化/引入相关的技术
  • 写Demo（先跑起来）
  • 实现业务逻辑
  • 测试（单元测试）
    • 最左原则：尽早发现，尽早解决
  • 代码提交 / 代码评审
  • 部署 ——> 发布

需求分析

  • 登录 / 注册
  • 用户管理（管理员）
    • 查询、修改
  • 用户校验（仅星球用户）

技术选型

前端：三件套 + React + 组件库 Ant Design + Umi + Ant Desin Pro（现成的管理系统）
后端：

  • spring（依赖注入，管理java对象，集成其他内容）
  • springmvc（web框架，提供接口访问，restful接口等）
  • mybatis（持久层框架，jdbc的封装）
  • mybatis-plus（mybatis增强，不用写sql就能crud）
  • springboot（不用管理spring配置 整合框架，快速启动 / 集成项目，机箱的一键启动）
  • mysql（redis）

上线：服务器 / 容器（平台）
