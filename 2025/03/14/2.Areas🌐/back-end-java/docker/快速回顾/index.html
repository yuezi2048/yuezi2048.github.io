Docker容器技术命令速查表（基础篇）

一、Docker基础命令

1. 安装与启动

CentOS7安装Docker
yum install -y docker-ce docker-ce-cli containerd.io
systemctl start docker
docker version

配置镜像加速器
tee /etc/docker/daemon.json <<-‘EOF’
{
“registry-mirrors”: [“https://2j0viskq.mirror.aliyuncs.com"]
}
EOF
systemctl daemon-reload && systemctl restart docker

2. 镜像操作

列出镜像
docker images [-a] [-q]

搜索镜像
docker search [IMAGE_NAME] –limit [N]

拉取镜像
docker pull [IMAGE_NAME]:[TAG]

删除镜像
docker rmi [-f] [IMAGE_ID]

查看镜像空间占用
docker system df

3. 容器操作

启动容器
docker run [OPTIONS] [IMAGE] [COMMAND]
常用参数：-it（交互模式）、-d（后台运行）、–name（指定名称）、-p（端口映射）

查看容器
docker ps [-a] [-l] [-n=N] [-q]

进入容器
docker exec -it [CONTAINER_ID] /bin/bash

启动/停止/重启容器
docker start|stop|restart [CONTAINER_ID]

删除容器
docker rm [-f] [CONTAINER_ID]

关键概念解释

镜像（Image）：Docker的只读模板，包含运行环境和应用程序，类似虚拟机镜像但更轻量。
容器（Container）：镜像的运行实例，独立的隔离环境，可启动、停止和删除。
数据卷（Volume）：宿主机与容器间共享的目录，用于数据持久化，容器删除后数据仍保留。
仓库（Repository）：存储镜像的地方，分为Docker Hub（公共）和私有仓库。

二、Docker高级命令（复杂安装与集群部分）

1. 复杂安装

1.1 MySQL主从复制

主库配置
docker run -p 3307:3306 -v /mydata/mysql-master:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=root -d mysql:5.7

从库配置
docker run -p 3308:3306 -v /mydata/mysql-slave:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=root -d mysql:5.7

主从同步
CHANGE MASTER TO MASTER_HOST=’主库IP’, MASTER_USER=’slave’, MASTER_PASSWORD=’123456’, MASTER_LOG_FILE=’日志名’, MASTER_LOG_POS=位置;

1.2 Redis集群

运行Redis节点
docker run -d –name redis-node-1 -p 7000:7000 redis:latest redis-server –port 7000 –cluster-enabled yes –cluster-config-file nodes.conf –cluster-node-timeout 5000

创建集群
redis-cli –cluster create 127.0.0.1:7000 127.0.0.1:7001 127.0.0.1:7002 127.0.0.1:7003 127.0.0.1:7004 127.0.0.1:7005 –cluster-replicas 1

2. Swarm模式

2.1 集群管理

初始化管理节点
docker swarm init –advertise-addr [IP]

查看加入令牌
docker swarm join-token [worker|manager]

添加工作节点
docker swarm join –token [TOKEN] [MANAGER_IP]:2377

2.2 服务部署

创建服务
docker service create –replicas 3 -p 80:80 –name web nginx

查看服务列表
docker service ls

扩展服务副本数
docker service scale web=5

删除服务
docker service rm web

关键概念解释

MySQL主从复制：通过配置主库和从库，实现数据的自动同步，提高数据可用性和读写性能。
Redis集群：将多个Redis节点组成一个集群，通过哈希槽分区实现数据的分布式存储和高可用。
Swarm模式：Docker内置的集群管理和编排工具，用于在多个Docker节点上部署和管理服务。

三、DockerFile与网络配置

1. DockerFile常用指令

FROM centos:7 # 指定基础镜像
MAINTAINER krisswen # 设置作者信息
RUN yum -y install vim # 执行命令安装软件
COPY ./app /app # 复制文件到镜像
CMD [“/bin/bash”] # 容器启动时执行的命令
ENTRYPOINT [“java”, “-jar”, “/app.jar”] # 容器入口命令
EXPOSE 8080 # 声明容器暴露的端口
WORKDIR /app # 设置工作目录
ENV JAVA_HOME /usr/local/java # 设置环境变量

2. 网络配置命令

创建自定义网络
docker network create [NETWORK_NAME]

查看网络详情
docker network inspect [NETWORK_NAME]

连接容器到网络
docker network connect [NETWORK_NAME] [CONTAINER_ID]

容器网络模式
docker run –network host [IMAGE] # host模式
docker run –network none [IMAGE] # none模式
docker run –network container:[CONTAINER_NAME] [IMAGE] # container模式

关键概念解释

DockerFile：用于构建Docker镜像的脚本文件，通过指令定义镜像的构建步骤和配置。
网络模式：Docker提供的不同网络配置方式，包括bridge（默认网桥）、host（共享宿主机网络）、none（无网络）和container（共享其他容器网络）。
自定义网络：通过docker network create创建的用户自定义网络，支持容器间通过服务名直接通信。

四、Docker Compose、可视化与监控

1. Docker Compose

安装
curl -L https://get.daocloud.io/docker/compose/releases/download/1.25.1/docker-compose-`uname -s-uname -m` -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

常用命令
docker-compose up [-d] # 启动服务（后台运行）
docker-compose down # 停止并删除服务
docker-compose exec [SERVICE] /bin/bash # 进入容器
docker-compose logs [SERVICE] # 查看日志

示例配置（docker-compose.yml）
version: ‘3’
services:
web:
image: nginx
ports:
- “80:80”
db:
image: mysql
environment:
MYSQL_ROOT_PASSWORD: root

2. 可视化工具Portainer

安装
docker run -d -p 9000:9000 –name portainer –restart=always -v /var/run/docker.sock:/var/run/docker.sock portainer/portainer

3. 监控方案（CIG）

启动监控服务
docker-compose -f cig-docker-compose.yml up -d

访问服务
CAdvisor：http://IP:8080
InfluxDB：http://IP:8083
Grafana：http://IP:3000（默认账号admin/admin）

关键概念解释

Docker Compose：通过YAML文件定义多容器应用，实现一键部署和管理。
Portainer：Docker可视化管理工具，提供图形界面操作容器、镜像和集群。
CIG监控：由CAdvisor（监控数据采集）、InfluxDB（时序数据库存储）、Grafana（数据可视化）组成的监控方案。
Swarm模式：Docker内置的容器编排工具，支持多节点集群管理和服务调度。

五、补充遗漏命令与关键概念

1. 遗漏的重要命令

1.1 系统与镜像相关

docker info # 查看Docker系统信息
docker events # 实时监控Docker事件
docker build –no-cache # 强制不使用缓存构建镜像
docker history [IMAGE] # 查看镜像构建历史

1.2 数据卷管理

docker volume create [VOLUME_NAME] # 创建数据卷
docker volume inspect [VOLUME_NAME] # 查看数据卷详情
docker volume prune # 删除未使用的数据卷

1.3 网络增强

docker network prune # 删除未使用的网络
docker network connect [NETWORK] [CONTAINER] # 连接容器到网络
docker network disconnect [NETWORK] [CONTAINER] # 断开容器与网络的连接

1.4 Swarm高级

docker service update –image [NEW_IMAGE] [SERVICE] # 更新服务镜像（滚动更新）
docker stack deploy -c docker-compose.yml [STACK_NAME] # 部署Swarm栈
docker stack ps [STACK_NAME] # 查看栈服务状态

2. 遗漏的关键概念

2.1 虚悬镜像（Dangling Images）

  • 特征：镜像名称和标签均为<none>
  • 成因：构建新镜像时未指定标签或旧镜像被覆盖
  • 清理：docker image prune -a

2.2 数据卷容器（Data Volume Container）

  • 作用：专门用于存储数据的容器，为其他容器提供数据共享
  • 示例：
    docker create -v /data –name data_container alpine
    docker run –volumes-from data_container myapp

2.3 联合文件系统（UnionFS）

  • 核心特性：分层叠加、写时复制（Copy-On-Write）
  • 优势：镜像轻量化、快速启动、节省存储空间

2.4 Raft协议

  • 作用：Swarm集群中实现分布式一致性，确保多Manager节点状态同步
  • 特点：强一致性、领导选举、日志复制

2.5 服务发现（Service Discovery）

  • 机制：自定义网络中容器可通过服务名直接通信
  • 示例：容器内通过ping mysql访问MySQL服务（需在同一网络）

2.6 多阶段构建（Multi-Stage Build）

  • 作用：减少最终镜像体积，仅保留运行时所需文件
  • 示例：
    FROM golang:alpine AS builder
    WORKDIR /app
    COPY . .
    RUN go build -o app

FROM alpine
COPY –from=builder /app/app /app
CMD [“/app”]

3. 命令使用场景补充

3.1 容器健康检查

docker run –health-cmd=”curl -f http://localhost:80/health || exit 1” –health-interval=10s myapp

3.2 临时容器执行命令

docker run –rm alpine ping 192.168.1.1

3.3 镜像导入导出

docker save -o myimage.tar myimage:tag # 导出镜像
docker load -i myimage.tar # 导入镜像

4. 总结

以上补充命令和概念完善了Docker操作的全流程覆盖，包括管理管理、数据持久化、网络优化、高级操作等。建议结合实际场景练习，重点掌握：

  1. 数据卷与网络配置的生产级实践
  2. Swarm集群的更新与服务发现
  3. 多阶段构建优化镜像体积
  4. 虚悬镜像与资源清理策略
