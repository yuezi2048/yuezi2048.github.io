61.Write-Ahead_Logging(WAL)技术优势

发表于2025-10-30|更新于2025-10-30|2.Areas🌐02.面经记录MySQL
|总字数:468|阅读时长:1分钟|浏览量:

WAL（Write-Ahead Logging）是事务日志管理技术，核心点：**保证在修改真正数据前，先将修改记录写入日志。**这样及时系统崩溃，也能通过日志恢复数据，保证数据一致性和持久性。具体流程：

  • 事务开始时，数据库所有修改会先记录到日志文件（存储数据变更，用于恢复数据），而不是数据库文件
  • 日志写入磁盘后，才将修改应用到数据库文件中

查询数据到内存 -> 记录undo log -> 记录redo log(预提交) -> 更新内存 -> 记录binlog -> 记录redolog(已提交)

任意一步出问题，都可以回滚

扩展知识

WAL 优点

  • 保证数据一致性：在事务提交之前，变更首先记录到日志中，在系统崩溃后，数据库可以通过重做日志中的操作来恢复到崩溃前的状态，确保数据一致性和持久性。
  • 性能提升：把数据的随机写转化成日志的顺序写，提高了整体性能。

Redo Log

在 MySQL InnoDB 存储引擎中，重做日志（Redo Log）就是 WAL 的实现，用于保证事务的持久性和崩溃恢复能力。

InnoDB 重做日志的工作机制如下：

1）当一个事务开始时，所有对数据库的修改首先记录到重做日志缓冲区中。

2）重做日志缓冲区的数据会周期性地刷新到磁盘上的重做日志文件（ib_logfile0 和 ib_logfile1）。

3）当事务提交时，InnoDB 确保重做日志已写入磁盘，然后将数据页的修改写入数据文件。

4）如果系统崩溃，InnoDB 会在启动时通过重做日志重新应用所有未完成的事务，以恢复数据库到一致状态。
