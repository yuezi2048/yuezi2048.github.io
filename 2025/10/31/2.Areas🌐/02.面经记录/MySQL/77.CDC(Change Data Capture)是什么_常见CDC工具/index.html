77.CDC(Change Data Capture)是什么_常见CDC工具

发表于2025-10-31|更新于2025-10-31|2.Areas🌐02.面经记录MySQL
|总字数:415|阅读时长:1分钟|浏览量:

CDC（Change Data Capture），中文叫变更数据捕获。

它的核心作用是：实时捕获数据库中数据的变化（INSERT、UPDATE、DELETE）事件，并将这些变更同步到下游系统，而不需要全量扫描数据库，提升效率。

通俗讲，就是：当数据库里的数据一有变动，比如新增一条订单、修改了用户信息，CDC 工具能立刻感知到，并把这份变化推送给下游，比如 Kafka、Elasticsearch、数仓等。

常见的 CDC 工具：

工具 特点
Debezium 开源、社区活跃，基于 Kafka Connect，可监听 MySQL、PostgreSQL、MongoDB 等数据库的 binlog。
Canal 阿里开源，专注 MySQL 和 Oracle 的 binlog 同步，常用于国内业务系统。
Maxwell 轻量级 CDC 工具，支持将 MySQL 的 binlog 输出为 JSON。
DataX / Flink CDC 阿里系的数据同步框架，Flink CDC 支持流式计算场景。

数据同步方案视频。

扩展知识

CDC 的主流方案

主流方案都是基于数据库日志来实现变更数据捕获，大多数 CDC 工具（Debezium、Canal）都属于这种。

它直接解析数据库的 binlog / WAL / redo log，实时捕获变化。

优点是性能高、不影响主库，能精确还原每次变更。

大致流程：

除此之外，还有两种方案：

  • 基于时间戳/版本号比对：定期扫描表，通过时间字段（如 update_time）检测变更
  • 基于触发器：通过数据库触发器，在数据变动时写入一张变更记录表。
