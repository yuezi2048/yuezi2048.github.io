伙伴匹配系统：找到志同道合的伙伴，移动端H5网页（尽量兼容PC）

需求分析

  • 标签：每个用户有标签，标签分类
    • 学习方向：java / c++
    • 状态：工作 / 大学
  • 主动搜索：允许用户根据标签搜索其他用户
    • Redis 缓存（本地缓存）
  • 组队（替补机制）
    • 创建队伍
    • 加入队伍
    • 根据标签查询队伍
    • 邀请其他人
  • 允许用户修改标签
  • 推荐
    • 相似度计算算法 + 本地分布式计算

技术栈

前端

  • vue3开发框架（提高页面开发效率）
  • Vant UI（基于Vue移动端组件库）（React版Zent）
  • Vite打包（快）
  • Nginx单机部署

后端

  • java + springboot
  • SpringMVC + Mybatis + MyBatis Plus
  • MySQL数据库
  • Redis缓存
  • Swagger + Knife4j 接口文档

第一期计划

  • 前端项目初始化
  • 前端主页 + 组件概览
  • 数据库表设计
    • 标签表
    • 用户表
  • 开发后端：根据标签搜索用户
  • 开发前端：根据标签搜索用户

6.回表
回表是使用二级索引作为条件查询后，由于二级索引只存储了索引字段的值和主键值，无法得到其他数据。 而为了得到其他数据，需要根据主键去聚簇索引找实际的数据行，这个过程就是回表。 拓展 请注意：回表操作会带来随机I/O的开销，因为根据id主键索引查询的适合，id是不连续的。 （如这里age为20的人，age是有序的，但id是无序的，此时去主键索引查询就会出现大量的随机I/O，相比于顺序I/O效率就低） 因此，代码里尽量不要用select *操作，引发不必要的回表。
