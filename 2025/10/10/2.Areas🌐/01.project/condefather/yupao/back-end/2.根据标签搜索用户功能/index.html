开发后端接口

搜索标签实现

  • 允许用户传入多个标签，多个标签都存在才搜索出来 and
    • like ‘%java%’ and like ‘%C++%’
    • mysql的json字段
  • 允许用户传入多个标签，有任何一个标签就能搜索出来 or
    • like ‘%java%’ or like ‘%C++%’

两种方式：

  • SQL查询(实现简单，可以通过拆分查询进一步优化)
  • 内存查询（灵活，可以通过并发进一步优化）

用户中心提供用户鉴权的功能。

快捷键：ctrl + shift + 减号，折叠，加号展开

测试

sql查询

内存查询

解析JSON字符串：json转换成对象（反序列化）

java序列化库：fastjson（alibaba 快 漏洞多）、gson（google 相对安全）、jackson、kryo

使用gson：https://mvnrepository.com/search?q=gson，复制到pom.xml

注：如果后续想要并发查询，可以通过并行流parallelStream()来实现

但是使用的是公共线程池，问题就是如果查询数据时间很长，就没有资源给其他线程分配了

refer：parallell stream陷阱

用到了Java8新特性：steam/parallel stream 、optional

性能测试

打开调试的sql语句

数据量大的时候，验证效果会更加明显

  • 如果有时候1方法快，有时候2方法快，我们可以枚举业务场景，去根据不同的场景选择查询方式
  • 如果参数不可分析，数据库连接足够，内存足够，并发同时查询，谁先返回用谁
  • 还可以SQL查询和内存计划结合，先用SQL过滤掉部分tag

在查询前后添加记录时间，打印相差时间测试（注意考虑到sql连接，可以先进行一次空查询建立连接）

c.y.yupao.service.impl.UserServiceImpl : sql query time = 25

c.y.yupao.service.impl.UserServiceImpl : memory query time = 29

发现数据量少时区别不大，在大的场景下应当分析业务参数场景，也可以同时兼顾。

编写Controller

controller最好再判断一下，避免service那边露了，双重验证保险

前后端联调

小技巧：可以在resources下写一个banner.txt，这样运行的时候可以看到自己的介绍信息。

问题：发现参数为空时报错，因此使用@RequestParam保证自动映射参数，并封装一个required=false，让我们自己处理请求参数，避免大量的错误信息暴露

这样子传入就可以对应上了。让前端对接这个接口即可（整合axios）

在axios可以定义配置连接，参照axios官方文档，也可以加上请求拦截器，全局响应拦截器等功能

发现出现了跨域问题，要进行配置。可以在controller直接@CrossOrigin暴力全部允许（还是最好限制一下origins），配置类也ok

前端测试效果（user-card-list 标签似乎需要loading属性，加上后就可以顺利加载），具体细节后续学前端的时候再分析。
