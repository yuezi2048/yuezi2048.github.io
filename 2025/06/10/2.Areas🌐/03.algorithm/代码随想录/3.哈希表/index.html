三、哈希法

3.1 理论基础

哈希表

  • 哈希表：key是关键字，作为哈希表的索引下标，访问对应的元素value，在算法中常常用来空间换时间

当我们需要查询一个元素是否出现过，或者一个元素是否在集合里的时候，就要第一时间想到哈希法

  • 哈希函数：映射方式，通常会有取模操作

  • 哈希碰撞：如果映射到了一个位置，那么就出现了碰撞，解决方案：

    • 拉链法

    • 线性探测法

哈希存储结构

集合 java对应类 底层实现 是否有序 数值是否可以重复 能否更改数值 查询效率 增删效率
std::set TreeSet 红黑树 有序 否 否 O(log n) O(log n)
std::multiset Map<String, Integer> 红黑树 有序 是 否 O(logn) O(logn)
std::unordered_set HashSet 哈希表 无序 否 否 O(1) O(1)
映射 java对应类 底层实现 是否有序 数值是否可以重复 能否更改数值 查询效率 增删效率
std::map TreeMap 红黑树 key有序 key不可重复 key不可修改 O(log n) O(log n)
std::multimap Map<String, List<String>> 红黑树 key有序 key可重复 key不可修改 O(log n) O(log n)
std::unordered_map HashMap 哈希表 key无序 key不可重复 key不可修改 O(1) O(1)

242. 有效的字母异位词

给定两个字符串 s 和 t ，编写一个函数来判断 t 是否是 s 的 字母异位词。

相关题目

  • 383.赎金信(opens new window)

    • 本题使用map的空间消耗要比数组大一些的，因为map要维护红黑树或者哈希表，而且还要做哈希函数，是费时的！数据量大的话就能体现出来差别了。 所以数组更加简单直接有效！

  • 49.字母异位词分组(opens new window)

    • 涉及到的算法：字符哈希 + 数组集合
    • 学到了一个基于hash合并的思路：

      • 将满足条件的基于Hash Key（字母异位词满足同个字符哈希），放在一个集合中（List）

        • 对于字母异位词的key，也可以暴力地通过排序来表示，但是该操作时间复杂度会有一个mlogm的级别（m是每个字符串的最大长度），而使用哈希表后，单个操作只有O(1)的时间复杂度（1=26）

      • 最终展现出来的方式是：返回map中所有的值（map.values）

  • 438.找到字符串中所有字母异位词

    • 涉及到的算法：字符哈希+滑动窗口（注意利用滑动窗口的特性来应用双指针）
    • 优化：因为涉及比较，可以利用该点，来比较滑动窗口和比较字符串中不同字符数量的差

3.4 unordered_map

两数之和

1. 两数之和

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

为什么想到哈希表

快速判断一个元素是否出现集合里的时候，就要考虑哈希法了

为什么不用数组和Set，想到用Map

  • 数组的大小是受限制的，而且如果元素很少，而哈希值太大会造成内存空间的浪费。
  • set是一个集合，里面放的元素只能是一个key
  • 因此只有map可以了，这道题目中并不需要key有序，选择std::unordered_map 效率更高

map存什么，k和v是什么

map是为了存储已经访问过的元素，key是元素，value是下标

四数相加Ⅱ

454. 四数相加 II

给你四个整数数组 nums1、nums2、nums3 和 nums4 ，数组长度都是 n ，请你计算有多少个元组 (i, j, k, l) 能满足：

  • 0 <= i, j, k, l < n
  • nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0

这题区别于**0015.三数之和 (opens new window)，0018.四数之和 (opens new window)**，为什么可以使用哈希法？

本题是四个独立的数组，不用考虑重复的四个元素相加等于0的情况（需要对相同方案去重，使用哈希会很麻烦）

所以依旧是可以通过哈希来记录元素在集合中的出现次数。

三数之和（去重）

15. 三数之和

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

**注意：**答案中不可以包含重复的三元组。

四数之和

18. 四数之和

给你一个由 n 个整数组成的数组 nums ，和一个目标值 target 。请你找出并返回满足下述全部条件且不重复的四元组 [nums[a], nums[b], nums[c], nums[d]] （若两个四元组元素一一对应，则认为两个四元组重复）：

  • 0 <= a, b, c, d < n
  • a、b、c 和 d 互不相同
  • nums[a] + nums[b] + nums[c] + nums[d] == target

你可以按 任意顺序 返回答案 。
