八、回溯

8.1 理论基础

回溯理解

回溯法，一般可以解决如下几种问题（组合无序，排列有序）：

  • 组合问题：N个数里面按一定规则找出k个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个N个数的集合里有多少符合条件的子集
  • 排列问题：N个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N皇后，解数独等等

回溯法可理解为树形结构：

  • 集合的大小就构成了树的宽度，递归的深度就构成了树的深度
  • 有递归那么一定有终止条件，那么必然就是高度有限的n叉树

回溯模板

  • 终止条件：对应搜到了叶子节点

  • 搜索遍历：

    • 横向遍历集合区间，对应该节点的子孩子（此图中集合大小和孩子的数量相等）
    • 纵向递归，对应dfs

同集合组合问题剪枝技巧：当k=4时，在第三层的情况下（path.size() = 2)，还需要2个数，那么就是宽度的最后1个数是不必要重复遍历的，即第三层的时候不需要遍历4，否则[1,2,3,4]会和[1,2,4,3]重复，所以在遍历不重复的数字的组合问题时，起点遍历可以遍历到n - (k - path,size()) + 1就停止了

重复元素组合去重（数层去重）

40. 组合总和 II

这里是进行数层去重，在某种意义上其实就是剪枝。

我们首先通过used数组判断如果前一个数没有访问过，并且存在相同的数，来进行去重的，避免该例子的[1, 2]重复

注意if判断的时机！递归前后的状态要清楚！

  • 这里是对数层去重，意思是在for循环中去掉，使用continue操作，而不是在进入递归后在中止条件里才判断（在取第二个1的时候进入递归，startIndex=2）
    • 树枝遍历到相同元素的情况：candidates[i] = candidate[i - 1] && used[i - 1] = true
    • 数层遍历到相同元素的情况：candidates[i] = candidate[i - 1] && used[i - 1] = false
  • 同样地，根据单调性，for循环中只要累加大于target，就可以中止本次数层的遍历了

补充：在理解了数层遍历的基础上，那么在for循环中就可以根据startIndex的起始遍历下标，直接横向跳到下一个不同元素的数层开始递归了，即：

if (i > startIndex && candidates[i] == candidates[i - 1]) continue

通过startIndex来表示used数组的意思，直接for循环跳到与前一个元素不相等的元素上。

补充：如果要是全排列的话，每次要从0开始遍历，为了跳过已入栈的元素，需要使用used。

总结

回溯模板

回溯解决的问题

  • 组合问题：N个数里面按一定规则找出k个数的集合
  • 排列问题：N个数按一定规则全排列，有几种排列方式
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个N个数的集合里有多少符合条件的子集
  • 棋盘问题：N皇后，解数独等等

引申出的问题

组合问题

  • 什么场景下要加startIndex：
    • 组合中单个集合的问题，而在多个集合中间取元素不用加startIndex
    • 而在排列问题中涉及到顺序性，startIndex就没有特别的意义了
  • 剪枝的方法
    • 单集合（n个元素）固定取k个元素问题：数层遍历到n - (k - path.size()) + 1停止
    • 最优性剪枝：没必要往后遍历（例如元素总和已经满足最大）
  • 解集去重
    • 树层去重：
      • 元素具备单调特性：used[i - 1] == false && candidates[i] == candidate[i - 1] 或 i > startIndex && candidates[i] == candidates[i - 1]) continue
      • 跨元素去重：uset.contains(candidates[i]) && candidates[i] == candidate[i - 1]（数组Hash）
    • 树枝去重：
      • 元素具备单调特性：used[i - 1] == false && candidates[i] == candidate[i - 1]

切割问题

以子串回文为例，需要理解下面的问题：

  • 切割问题：看成是每个截取的子串看成是组合问题中的数
  • 切割线：startIndex对应切割的下标
  • 中止条件：一直到切割的右字串为空为止。
  • 在递归中截取子串：通过数层的i来动态截取该层对应的子串，s.substring(startIndex, i + 1)，注意i + 1的重复问题
  • 判断子串回文：是否能想到s[0] == s[n-1]且s[1:n-1]是回文字串的最优子结构

子集问题

  • 区别于之前收集的叶子节点，子集问题是收集所有节点的过程
  • 在一些特定问题中，例如非递增子序列，是一定要排序的（排列问题不用）

排列问题

  • 每层都是从0开始搜索而不是startIndex
  • 需要used数组记录path里都放了哪些元素了

棋盘问题

  • 难点是在于整个搜索过程中，每一个节点应当是去判断（棋盘）了
  • 特别地，对于二维递归问题，需要遍历（row, col）坐标下放入num值是否合法
  • 以及如果是返回唯一路径，我们一般是返回值是boolean，来控制找到的第一个答案向上返回

回溯综合

  • 选择合适的容器存储，{source, target}的可重复机票：Map<String, Map<String, Integer>>
