三、变量

3.1 变量入门

  • 为什么要变量
    • 一个程序就是一个世界，每个元素需要由变量来表示
  • 变量是什么
    • 变量组成三要素：类型 + 名称 + 值
    • 类型 - int，名称 - a，值 - 1
  • 变量原理（上述代码为例）：变量名a指向新开辟的4字节内存，内存值为1，b变量同理
    • 变量相当于内存中一个数据存储空间，变量名就是访问这个空间的索引

变量demo

细节

  • 一个变量占用着一块内存区域，不同的变量占用空间是不同的
  • 定义的时候要定义，变量名和数据类型，先声明，再使用。（变量 = 变量名 + 值 + 数据类型 三要素）
  • 该数据在同一范围内是可以变化的
  • 在同一作用域内不能重名

'+'号的连接方法，看运算符两边变量类型

  • 两边是数值型，加法
  • 有一方是字符串，拼接
  • 从左到右运算，多项相加需要关注的

3.2 八种数据类型*

基本数据类型

  • 数值型：【byte short int long】 ，对应1248字节
  • 布尔型【boolean】 1 字符型【char】 2 （有中文） 浮点【float double】 4 8

引用数据类型：【类 接口 数组】

细节

3.2.1 数值型

数值型

数值型的实际范围，与二进制补码等知识有关。

  • 每个操作系统对数值型定义的字段长度是固定的，不受OS影响，保证可移植性
  • 常量默认int，long型常量后面要加L，一般在int范围之外大数情况下用long
    • int n2 = 1L 不对，违背了自动类型转换中低精度向高精度转的原则，后续介绍不要急。
  • 1 byte = 8 bit，bit是最小单位，以byte n1 = 3和short n2 = 3变量为例

3.2.2 浮点型

float 和 double 对应4字节和8字节的存储空间

  • 浮点数构成：浮点数 = 符号位 + 指数位 + 尾数位
  • 尾数位可能丢失，造成精度损失 => 后续判断时要用一个误差范围判断

细节

  • 同样不受OS影响，长度固定，可移植性
  • 常量默认double，声明float变量，要在常数后面加F
  • 浮点数表示有三点：
    • 一定要有小数点，例如123.0f
    • 小数点前可以没有数字，.123默认为0.123
    • 科学计数法5.12e2 或 5.12E-2
  • double精度比float高，通常情况下用double
    • 比如2.1234567851 和2.1234567851F 最后显示的小数点位数是不一样的
  • 在运算后造成精度损失的问题
    • 8.1 / 3 的结果是趋于2.7的小数而不是本身
    • 如果要检验 2.7 是否是8.1/3的结果，判断时要进行一个精度误差判断（Math.abs）的操作

查询API文档

在线网站或本地文档

API提供了大量应用程序编程接口，里面提供了大量的类、方法等实现了很多功能。

介绍API的结构：包 => 接口、类、异常 => 继续细分字段、方法、构造器等

介绍两种查询的方法

  • 一步步查
  • 直接检索对应的类、接口名

3.2.3 字符型

字符char 占两个字节，一个字符是可以存放一个汉字的，多个字符是字符串String。

使用细节

  • 单引号括字符
    • char c = ‘凌’；
  • 字符允许使用转义字符
    • char c = ‘\t’；
  • 字符本质是整数，输出的时候实际输出的是UniCode码整数对应的字符，存储时将整数的二进制存入内存，读取同理
    • UniCode码转换
    • int a = (int)c; // 输出字符对应的整数
  • 本质上是整数，因此字符可以进行数字运算
    • char c = ‘a’ + 1; // 输出b

常用编码类型

接下来介绍码值和数字的对应规则

  • ASCII，1个字节表示128个字符，理论上可以表示256个字符但实际只用了128个
  • UniCode，字符和汉字统一2字节，浪费空间但表示范围大，是ASCII的拓展，即兼容ASCII
  • utf-8 可表示汉字，字符1字节，汉字3字节，可变大小的编码，节省了空间
  • gbk 可表示汉字，字符1字节，汉字2字节
  • gb2312 可表示汉字
  • big5 繁体中文 台湾、香港

3.2.4 布尔类型

  • 取值只有true和false，没有null，与C/C++不一样的是不能用1 0 代替
  • 占1字节
  • 常用于逻辑运算
    • if条件控制语句
    • do-while while for 循环控制语句

3.3 自动类型转换

自动类型转换：精度小的自动转换成精度大的，转换路线有两条

细节

  • 多项数据混合运算，自动将所有类型转为最高精度的计算
  • 精度大自动转化为精度小的会报错
  • byte、short与char之间不会进行自动类型转化
  • byte，short与char之间三者可以进行运算，但自动会转化成int类型
    • 注意自己与自己本身的运算也会自动转换成int类型
  • boolean不参与自动类型转换

3.4 强制类型转换

容量大的转换成容量小的，既然是强制，那么势必会可能有如下问题

  • 精度损失（高精度浮点强制转换成低精度 double -> float）
  • 溢出（高范围整数强制转换成低精度整数 long -> int）

int x = (int)(10 * 1.5)

细节

  • 强转符号只对最近的数有效，如需整体强转，需要加上括号
  • 注意char类型虽然本质上是整型，但是由于精度问题，int到char需要强转
    • char c3 = (int)n1
  • byte、short和char类型运算，运算后一律自动转化成int

3.5 基本数据类型与String类型转换

基本数据类型 => String 加上空字符串即可

String => 基本数据类型 调用对应类包的parseXXX方法即可

补充：转化成字符，只需取出第一个字符即可 ===》 char c = S.charAt(0)

细节

  • 任何数据类型转化成String没啥问题，但String转化成基本数据类型可能会抛出异常，程序终止

    • 例如”hello”在调用Integer.parseInt(“hello”)就会抛出一个Exception
