4.1 算数运算符

细节

  • / 号在整数中不保留小数

  • Java中，a % b 等价于 a - (int) a / b * b

    • 10 % -3 = 10 - 10 / (-3) * (-3) = 10 - 9 = 1

  • 重点关注两道面试题（关键点：++是一种自增运算，=是一种赋值运算，自增的本质是需要临时变量实现的，之后再赋值操作）

    • 使用内存分析法进行分析，每一步操作对应一个内存中变量值的修改

编程解决实际问题的思路

  • （1）需求分析
  • （2）思路分析
  • （3）代码实现

4.2 关系运算符

  • 常在if条件判断或循环结构出现

细节

  • 关系运算符的结果是boolean类型，即返回值为true或false
  • 关系运算符组成的表达式，叫做关系表达式
  • 关系运算符== 严格区分于=

使用内存分析法进行程序分析题的练习

4.3 逻辑运算符

  • 用于连接多个关系表达式，得到的结果是boolean类型
  • 重点关注短路与&& 和 逻辑与&，以及短路或|| 和 逻辑或 | 的区别
  • !是非，^是异或

  • 与：全真为真，或：有真为真，异或；不同为真
  • 短路与和逻辑与返回的boolean结果相同

细节

  • 短路与&& 和 逻辑与&
    • 对于&&短路与而言， 如果第一个条件为 false ,后面的条件不再判断
    • 对于&逻辑与而言， 如果第一个条件为 false ,后面的条件仍然会判断
    • 一般短路与&&效率高，开发常用
  • 短路或|| 和 逻辑或 |
    • 短路或||： 如果第一个条件为 true，则第二个条件不会判断， 最终结果为 true， 效率高
    • 逻辑或|： 不管第一个条件是否为 true， 第二个条件都要判断， 效率低

练习

  • 异或和取反
    • boolean b = (10 > 1) ^ ( 3 > 5); // T
    • System.out.println(!(60 > 20)); //F

4.4 赋值运算符

  • 将运算后的值赋给一个变量，+= ， -= ， *= ， /= ， %= 等

细节

  • 赋值顺序是从右往左
  • a += 3 等价于 a = a + 3
  • 赋值 = 严格区分==
  • 复合赋值运算符包含强制类型转换

4.5 三元运算符

条件表达式 ? 表达式 1: 表达式 2;

比如找出三个数最大的值，可以用三元运算符实现，后续可以用排序实现

细节

  • 表达式1和表达式2需要能赋给变量（或自动类型转换）
  • 可转换为if else

4.6 运算符优先级

  • 只有单目运算符（++ ~ ！）、 赋值运算符是从右向左运算的

  • 括号 => 单目运算符 => 算数运算符 => 位运算符 => 关系运算符 =>逻辑运算符 => 三元运算符 => 赋值运算符

测试

4.7 标识符命名规则与规范

规则（必须遵守）

  • 标识符由26个字母、数字0-9、下划线_和美元$ 组成
  • 数字不能开头
  • 关键字和保留字不可重名，可包含于其。
  • 严格区分大小写，长度不限制
  • 不能包含空格

规范

  • 包名全小写com.hsp.crm
  • 类名，接口名用大驼峰 TankShotGame .java
  • 变量名 方法名 用小驼峰（驼峰法）userName，getPassword()
  • 常量名 全部大写，下划线连接 TAX_RATE
  • 其他规范见规范文档

关键字已经被用作特殊用途，不能用了，他们都是小写字母组成的

保留字：现有 Java 版本尚未使用， 但以后版本可能会作为关键字使用。 自己命名标识符时要避免使用这些保留字

包括： byValue、 cast、 future、 generic、 inner、 operator、 outer、 rest、 var 、 goto 、 const

4.9 位运算*

4.9.1 进制转换基础

二进制： 0,1 ， 满 2 进 1.以 0b 或 0B 开头。
十进制： 0-9 ， 满 10 进 1。
八进制： 0-7 ， 满 8 进 1. 以数字 0 开头表示。
十六进制： 0-9 及 A(10)-F(15)， 满 16 进 1. 以 0x 或 0X 开头表示。 此处的 A-F 不区分大小写

第一组：
\1) 二进制转十进制
\2) 八进制转十进制
\3) 十六进制转十进制

每一位乘以对应进制的权值 再相加

第二组：
\1) 十进制转二进制
\2) 十进制转八进制
\3) 十进制转十六进制

除X取余法，直到商为0时停止

第三组
\1) 二进制转八进制
\2) 二进制转十六进制

3位/4位为一组直接转

ob11(3)010(2)101(5) => 0325

ob1101(D)0101(5) = 0xD5

第四组

\1) 八进制转二进制
\2) 十六进制转二进制

3逆过程，填充对应3/4位

02(010)3(011)7(111) = 0b1001111

0x2(0010)3(0011)B(1011) = 0b001000111011

4.9.2 原码、反码、补码基础*

需要背下来

  • 二进制最高位是符号位，0是正，1是负（1倒过来就是负号）
  • 正数的原码、反码、补码都一样（三码合一）
  • 负数的反码 = 原码符号位不变，其他位取反
  • 负数的补码 = 负数的反码 + 1，负数的反码 = 负数的补码 - 1
  • 0 的反码，补码都是0
  • Java没有无符号数，每个数都是有符号的
  • 计算机运算的时候，使用补码运算
  • 最终的运算结果是看它的原码

4.9.3 位运算符

  • & | ^ ~：按位与、按位或、按位异或、按位取反

  • >> << >>> （高位补符号位，低位补0）

    • 算数右移>> ：相当于 /2 ，低位溢出,符号位不变,并用符号位补溢出的高位
    • 算数左移<<：相当于*2，符号位不变,低位补 0
    • 逻辑右移（无符号右移）低位溢出， 高位补 0
    • 没有逻辑左移
