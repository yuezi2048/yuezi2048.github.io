一、排序

1.1 快速排序

基本思想：我们是用到了划分的思想

  • 首先我们要确定分界的点
  • 我们让x划分成两个部分，使得所有左边的区间全部小于等于x，右边的区间全部大于等于x
  • 递归地处理两段，让左边 右边分别排好序，就完成了排序。

关键是如何划分，我们给出三个方案：

划分的暴力方法，时间一样，但是代价是：划分两个O(n)的数组空间

  • a数组存储比x小的，b数组存储比x大的，然后合并在一起作为一次划分。

双指针

一次划分的过程

  • 指针i 从左到右遍历，指针j 从右到左遍历
  • 遍历指针i，当其指向的元素比x大的时候，停
    • 遍历指针j，档期指向的元素比x小的时候，停，然后交换，swap(q[i], q[j]) i++，j–
  • 当i和j相遇的时候，一次划分完成。

原理：我保证了指针i的前面元素比x小，指针j后面的元素比x大，从而完成了划分。

我们会发现存在许多边界问题 比如等于的情况，建议：背模板

注：当递归里面写i的时候，不能写x = q[l]，递归里写j的时候，不能写x = q[r]，否则会无限递归（样例：1 2）

细节问题：这里递归用j的话，后面取中间值不用向上取整了，用i需要向上取整 否则最后剩两个数的时候 就会”交换了个寂寞“ 无限递归。

tips:这是最后两个元素无限递归的情况解释

我们只要记住一个版本就可以了，建议使用j 轴的选取 要么直接/2 要么取L 这样边界问题就没有了。

那么用i的话 就注意要取右边，要么向上取整 要么取R 这个段问题同二分问题类似

1.2 归并排序

注：快排分界点随机，归并分界点是中点，快排关键是如何划分，归并关键是如何合二为一，但是解决思路都是双指针。

这里我们是先递归再排序（归并）

  • 确定分界点： (l + r) / 2

  • 左右递归排序

  • 归并两个有序序列合二为一

关键是第三步，同样是双指针的思想，扫描左和右，时间复杂度是O(n)

我们比较两个指针哪个小，小的拿过来，往后挪一位，接着比较两个指针。

一个指针指完了，后面的放进来即可。

为什么说归并排序最好最坏都是nlogn，相比于快排最坏是n方

快排平均又问为什么说是O(nlog2n)，主要是看划分多少次，他期望是O(log2n) 划分n次，就是这么来的

log2n层，每层O(n)

二、二分

2.1 整数二分

关键是一个边界问题。

二分本质不是单调性，他的本质是边界

我们根据某种性质将一个区间一分为二，绿色满足 红色不满足，我们就可以查找到红色或者绿色边界 对应的就是两个模板

实际我们不需要考虑是红色点还是绿色点

只需要写完check函数，考虑如何根据check函数来更新我们的区间

如果是l=mid r = mid - 1，我们需要向上取整，否则r =mid，l = mid + 1，则不需要

找红色右端和绿色左端的两种模板

最终代码模板

最后退出的时候 l = r 所以返回谁都是一样的。

我们考虑第一种并且只有两个元素的情况（L = R - 1）

  • 此时mid = L，那如果是true，L不变，陷入了死循环。
  • 解决方法是mid = (l + r + 1) / 2

给出一个例子

比如我们要找某个数的下标区间范围（第4个到第5个）

如果说找下界，那么就是找大于等于x最左边的数，满足≥x的条件的话，那么更新区间为[L, mid] 即R = mid

顺势就可以写l的范围了。

如果说找上界，那么就是找小于等于x最右边的数，满足≤x的条件的话，那么更新区间为[mid, r] ，即L =mid

顺势就可以写R的范围了，注意L=mid的情况下，mid要向上取整

注意：二分一定保证有解，无解的情况是根据题目来的，比如在二分下，我们找不到x的数 题目那种情形下就是无解了。

2.2 实数二分

思路是一样的，只不过我们不需要处理边界问题，我们只需要根据浮点数区间的精确度来判断最终结果

比如说平方根问题

满足条件的话，就更新区间为[l, mid] ，即R = mid，否则L = mid，不需要处理边界了，最后输出l 输出r都可以

经验值：保留4位小数，就写1e-6，保留5位，就写1e-7，保留6位，就写1e-8

第二种写法是直接迭代100次，相当于对区间长度除以了2^100次方。
