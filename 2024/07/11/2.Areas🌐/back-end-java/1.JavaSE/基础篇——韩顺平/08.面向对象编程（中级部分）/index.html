8.3 访问修饰符

8.3.1 为什么要有访问修饰符(访问权限)

java提供了四种访问修饰符：public、protected、无修饰、private，为了控制方法和属性(成员变量)以及类的访问权限（范围）

  • 公开级别：public修饰，跨包允许访问
  • 受保护级别：protected修饰，对子类和同一个包的类允许访问
  • 默认级别：没有修饰符，对同一个包的类允许访问
  • 私有级别；private修饰，仅对类本身可访问

8.3.2 细节——成员变量和类修饰符使用

  • 修饰符可以修饰属性、成员方法、类
  • 类只能被public和默认修饰
  • 成员方法和属性的访问规则一致

8.4 面向对象三大特征——封装

8.4.1 为什么要封装(授权+隐藏细节)

封装是将我们抽象出来的属性和方法封装在一个类中。数据则会保护在内部，程序其他部分只有通过调用被授权的方法才能对数据进行操作

相较于我们传统的直接使用类对象访问属性的方法，封装有以下好处

  • 隐藏实现细节
    • 比如要链接数据库，我们主函数只需要引用该对象的方法，然后传入参数即可
  • 可以对数据进行验证，保证安全性和健壮性
    • 在实现具体业务前加入授权判断等操作（构造器直接调用get set方法即可）

8.4.2 封装快速入门

使用get set方法来实现封装即可，属性使用private属性，这样在set方法中就可以加入我们的权限判断并且隐藏了具体的实现细节

8.5.3 细节——继承10个使用规则

  • 子类继承父类所有的属性和方法

    • 但是请注意父类的私有属性和方法 子类是不能直接访问的，可通过父类的公共方法去访问

  • 子类必须调用父类的构造器，完成父类的初始化

  • **【关键】**当创建子类对象后，不管使用子类的哪个构造器，默认调用的都是父类的无参构造

    • 如果父类没有提供无参构造，必须在子类中通过super显式调用父类的构造器 从而完成【父类构造器的初始化】

  • 如果希望指定调用父类的构造器，可以使用super(参数列表)显式调用

  • super使用的时候，必须放在构造器的第一行（并且只能在构造器中用super）

  • super()和this()都是只能放在构造器的第一行，因此他们也不会在一个构造器中共存

  • java中所有的类都是Object类的子类，Object是所有类的基类

  • **【关键】**父类构造器调用不能只局限于直接父类，子类构造器的调用会一直追溯到Object的基类

  • 子类最多只能继承一个父类，单继承机制

    • 如果想让A类同时继承B类和C类，那么可以让B继承C，A继承B

  • 继承不能滥用，A继承B必须满足 B is-a A 的关系

8.5.4 继承的本质*

继承的本质就是建立查找关系

  • 在一个地址信息中从父类到子类加载所有的成员
  • 在进行访问的时候，从子类开始逐个向上查找（私有的无法访问，要通过其他可访问的方法来访问）
  • 子类如果没有找到，就往父类找有没有，一直到Object类
  • 没有找到就报异常

8.6 super关键字

8.6.1 为什么有super（访问父类成员）

在继承的基础上，通过super我们可以在子类访问到父类（不要局限在一个父类里面）的成员（属性、构造器、方法）

  • 调用父类构造器（父类的属性由父类初始化，子类的属性由子类初始化）
    • 如果没有写，会自动的调用父类的默认构造器super()
  • 子类和父类的成员重名时，使用super可以访问到父类的成员，如果没有重名，super this 直接访问的效果是一样的
  • super不局限在直接父类，如果说爷爷类有同名的成员，可以用super来访问爷爷的成员，当然如果父亲有这个，秉持就近原则和权限访问原则

8.6.2 快速入门

super 代表父类的引用， 用于访问父类的属性、 方法、 构造器

  • super.属性名
    • 不能访问父类私有的属性
  • super.方法名(参数列表)
    • 不能访问父类私有的方法
  • super(参数列表)
    • 只能放在构造器的第一句且只有这一句

特别注意：如果第一句没写 隐藏着一句super(),见 B b2 = new B(); 语句的执行结果

8.6.3 细节——super和this对比

根据super和this访问的对象，可以从访问属性、调用方法、调用构造器来说明

8.7 方法重写/覆盖/override

8.7.1 为什么要重写（扩展父类方法）

  • 在实际业务中，如果子类在父类的方法上予以拓展，为了避免命名上的重复，因此需要重写技术，即实现在子类中编写与父类相同的方法名，方法参数，返回类型的方法

8.7.2 快速入门

请注意：

  • 返回类型可以是父类的子类，但不能是父类返回类型的父类

  • 修饰符也可以在原来的基础上扩大，但不能缩小

8.7.3 细节——重写使用事项

  • 子类重写方法，要保证父类方法的形参列表、方法名称完全一样
    • 形参要顺序、个数、类型全部相同
  • 子类返回类型要和父类返回类型要一样，或者是父类返回类型的子类

  • 子类不能缩小父类方法的访问权限(访问权限可以在父类的基础上 扩大 或 相同)

8.8.2 快速入门*

方法或对象具有多种形态。 是面向对象的第三大特征， 多态是建立在封装和继承基础之上的。

方法的多态: 重/载和重写

对象的多态（重要）

  • 一个对象的编译类型可以和运行类型不一致
  • 编译类型是在定义对象的时候就已经确定了，不可以改变
  • 运行类型是可以变化的
  • 编译类型是 定义对象时 ‘=’的左边，运行类型是’=’的右边

8.2.3 细节——多态使用事项

  • 使用多态的前提是两个对象（类）之间是继承关系
  • 多态向上转型
    • 本质：父类的引用指向子类的对象
    • 父类类型 引用名 = new 子类类型(); 【如 Animal animal = new Cat();】
    • 编译类型在左边，运行类型在右边
      • 引用可以调用父类的所有成员（遵守访问权限）
      • 不能调用子类的特有成员
      • 最终运行效果看子类的具体实现
  • 多态向下转型
    • 子类类型 引用名 = (子类类型) 父类引用;
      • 如Cat cat = (Cat) animal;
    • 只能强转 父类的引用，不能强转父类的对象
    • 父类的引用必须指向的是当前目标子类 类型的对象
    • 向下转型后，可以调用子类的特有成员

进一步说明：

  • 属性没有重写之说！属性的值看编译类型
  • instanceOf 比较操作符， 用于判断对象的运行类型是否为 XX 类型或 XX 类型的子类型
    • 注意 instanceOf看的是 运行类型

快速过一下向上转型和向下转型

8.2.4 细节——动态绑定机制**

  • 调用对象的方法时，方法和该对象的内存地址/运行类型绑定
  • 调用对象的属性时，没有动态绑定机制，哪里声明，哪里调用

8.9 Object类

8.9.1 equals方法

==和equals对比

  • == 作为一个基本运算符，既可以判断基本类型，也可以判断引用类型
    • 基本类型判断的是值是否相同，如int i = 10; double d = 10.0;
    • 引用类型判断的是地址是否相等，即判断是否为同一个对象
      • 注意，只能判断同一个引用类型，不同的类型会编译报错，如”hello” == new java.sql.date(1111)
  • equals是Object类中的一个方法，只能判断引用类型
    • 默认判断的和==一样，判断地址是否相等
    • 我们往往重写该方法，判断内容是否相等（见Integer类和String类的equals源码）

检验

子类重写equals方法

应用实例: 判断两个 Person 对象的内容是否相等， 如果两个 Person 对象的各个属性值都一样， 则返回 true， 反之 false

8.9.2 hashCode方法

  • hashCode可以提高使用哈希结构的容器的效率
  • 两个引用，如果指向同一个对象，hash值一定相等
  • 两个引用，如果指向不同的对象，hash值是不一样的
  • 哈希值根据地址号得出，但数值不等同于地址
  • 后面在集合中，会根据需要重写Object类的hashCode()方法
  • 案例演示（aa1和aa3是同一个对象空间，aa2不同）

8.9.3 toString方法

  • 默认返回：全类名 + @ + 哈希值的十六进制 【见Object类的toString方法】
    • 子类往往重写toString方法，返回对象的属性信息
  • 重写toString方法 打印对象或拼接对象时，会自动调用该对象的toString形式
    • 直接输出一个对象时， toString 方法会被默认的调用, 比如 System.out.println(monster)； 就会默认调用monster.toString()

8.9.4 finalize方法

  • finalize怎么调用：对象被回收后，系统会自动调用finalize方法。子类可重写该方法，做一些释放资源的操作
  • 什么时候被回收：当一个对象没有任何引用时，被jvm当作垃圾对象，根据``垃圾回收机制会销毁该对象，而在销毁对象前就是会调用finalize方法`
  • 什么时候调用垃圾回收：由系统自行决定（根据自己的GC算法），我们也可以用System.gc()来主动触发垃圾回收
    • 注意：即使我们主动触发了GC，也是不会立刻触发的，类比喊酒店阿姨来打扫的行为，但是她未必回来

注：实际开发中很少用到此方法，当作面试题即可。

8.10 断点调试

8.10.1 为什么要断点调试(debug)

  • 我们开发项目遇到错误时，需要看源码执行过程来一步步找执行过程，这时候就需要断点调试帮助我们定位到错误的地方

    • 调试前或者运行到该语句前，可以在一行语句下断点，运行到这一行就会停住
    • 我们可以【一步一步往下跳过走——跳到下一个断点(resume )—— 跳入到该函数体】，一步步分析变量值
    • 如果运行到出错的语句，程序会停下，报错
    • 断点调试是我们必备的技能，也是在看java源码执行过程的重要技能，帮助我们提高编程水平

  • **注意：**我们在断点调试过程中，程序是运行状态，因此对象是以运行类型来执行的
