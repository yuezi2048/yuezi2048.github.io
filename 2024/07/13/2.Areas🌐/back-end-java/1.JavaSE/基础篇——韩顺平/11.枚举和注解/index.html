十一、枚举和注解

11.1 枚举(Enumeration)

11.1.1 为什么要枚举（定义固定对象）

  • 现在有一个需求，需要调用春夏秋冬四个对象。
  • 按照原来的方法，应该是定义一个类，然后创建四个对象
  • 但存在两个比较大的问题
    • 第一是对象数量的问题：用户可以任意创建季节的对象，不能体现出只有四个季节。
    • 第二是对象固定的问题：创建季节的对象后，用户可以任意的对里面的属性进行修改，即只读属性
  • 因此衍生出通过枚举类 来 定义 数量固定且对象不能被修改 的若干对象。
    • 枚举是一组常量的集合
    • 枚举类是特殊的一种类，只包含有限的特定对象

ps：这里生成类的时候加入了一些信息，在设置里修改一下模板就可以了

11.1.2 自定义枚举

  • 构造器私有化，因为类的属性要求只读
  • 本地创建多个枚举对象， 枚举对象全部用final static修饰，底层优化
  • 对外暴露枚举对象，枚举对象名全部大写，遵循常量规范
  • 不提供setXxx方法，只提供get方法，从而只读

11.2 enum关键字

11.2.1 快速入门

使用 enum 来实现前面的枚举案例， 体会和自定义枚举的区别

  • （1）使用enum替换class类
  • （2）对public static final的常量 直接修改为常量名(形参列表)
  • （3）多个常量之间 用逗号分隔，用分号结尾
  • （4）定义常量写在最前面

11.2.2 细节——enum类的定义

  • enum关键字设计枚举类的时候，会默认继承Enum类
    • 通过反编译的结果，可以看出来枚举类是继承了Enum类，并且由final修饰

  • public static final Season2 SPRING = new Season2(“春天”, “温暖”); –> SPRING(“春天”, “温暖”)
  • 构造器是必须要知道调用哪个构造器
  • 默认无参构造器可以省略形参列表和括号，因为是默认的构造器
  • 当有多个枚举对象时， 使用,间隔， 最后有一个分号结尾
  • 枚举对象必须放在枚举类的行首

11.2.3 细节——enum类实现接口

  • 我们已经知道，枚举类隐式继承了Enum类，那么就不能继承其他类了
  • 但是是可以实现多个接口的

总结

  • 至此，我们知道了使用enum关键字定义枚举类后，他会隐式地继承Enum类
  • 在测试的时候，我们打印该方法的toString()方法时，发现是直接打印了枚举对象的名字，思考动态绑定机制，怀疑是其父类Enum重写了Object类的toString()方法。
  • 接下来，我们跳转到Enum类，就有文章可以做了。

11.3 enum类常用方法

  • toString： 返回当前对象名，子类可重写该方法
  • name：返回当前对象名(常量名)，子类不可重写
  • ordinal：返回当前对象位置号，默认从0开始
  • values：返回枚举类所有常量
  • valueOf：将字符串转化为枚举对象，要求字符串必须是自己的常量名，否则报异常
  • compareTo：比较两个枚举常量，比较的是两者的编号ordinal

11.4 注解(Annotation)

11.4.1 为什么要注解（补充信息）

  • 注解(Annotation)也叫元数据(Metadata)，修饰包、类、方法、属性、构造器、局部变量等信息
  • 注解和注释一样，不影响程序逻辑，但是注解可以被编译执行，是嵌入代码的补充信息
    • 正是由于该特性，Override注解就是可以让编译器检查是否重写了父类的方法
  • JavaSE中，注解一般是标记过时功能(Deprecated)，忽略警告(Suppress Warnings)等功能
    • 主要的还是在Java EE中，比如配置应用程序的切面来代替旧版的冗余代码 XML配置等

11.4.2 快速入门

使用 Annotation 时要在其前面增加 @ 符号, 并把该 Annotation 当成一个修饰符使用。 用于修饰它支持的程序元素
三个基本的 Annotation:

  • @Override: 限定某个方法， 是重写父类方法, 该注解只能用于方法
  • @Deprecated: 用于表示某个程序元素(类, 方法等)已过时
  • @SuppressWarnings: 抑制编译器警告

@Override

  • （1）源码中发现Override注解的类型是@interface,表示这是一个【注解类】（jdk5.0后加入）
  • （2）Override只能修饰方法，类，属性等等都不行
  • （3）修饰注解的注解，如@Target就是元注解，后面会详细地说明，了解能看懂源码就行。
  • （4）@Override的价值：如果我写了@Override注解，编译器就会检查该方法是否真的重写了父类的方法,则编译通过，

@Deprecated

  • （1）如果用Deprecated表示这个类过时了，不推荐使用
  • （2）可以修饰构造方法、字段属性、局部变量、方法、包、参数、类
  • （3） @Deprecated 价值：版本升级的过渡使用，新旧版本的兼容

@SuppressWarnings({“xxx”…})

  • unchecked 是忽略没有检查的警告
  • rawtypes 是忽略没有指定泛型的警告(传参时没有指定泛型的警告错误)
  • unused 是忽略没有使用某个变量的警告错误

后面使用的时候照着编译器报错填进去元素即可，或者省事的话写一个all就能all in了

11.5 元注解

JDK 的元 Annotation 用于修饰其他 Annotation ，我们学习这个主要能知道源码就行，开发一般很少用。

元注解包括四个方面

  • Retention //指定注解的作用范围， 三种 SOURCE,CLASS,RUNTIME
  • Target // 指定注解可以在哪些地方使用
  • Documented //指定该注解是否会在 javadoc 体现
  • Inherited //子类会继承父类注解

11.5.1 @Retention

用于指定该 Annotation 可以保留多长时间, @Rentention 包含一个 RetentionPolicy类型的成员变量, 使用 @Rentention 时必须为该 value 成员变量指定值:

  1. RetentionPolicy.SOURCE: 编译器使用后， 直接丢弃这种策略的注释

  2. RetentionPolicy.CLASS: 编译器将把注解记录在 class 文件中. 当运行 Java 程序时, JVM 不会保留注解。 这是默认值

  3. RetentionPolicy.RUNTIME:编译器将把注解记录在 class 文件中. 当运行 Java 程序时, JVM 会保留注解. 程序可以通过反射获取该注解
