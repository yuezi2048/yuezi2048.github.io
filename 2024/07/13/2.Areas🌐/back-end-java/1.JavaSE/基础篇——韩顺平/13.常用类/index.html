十三、常用类

13.1 包装类

13.1.1 为什么要包装类（接口 方法）

针对于八种基本数据类型对应的引用类型——包装类

有了包装类，我们就可以调用类的方法

  • 实现了Comparable接口，就可以进行大小的比较
  • 实现了Serializable接口，就可以串行化数据，后续可以通过文件流，网络传输传输数据了

13.1.2 快速入门

包装类和基本数据类型转换

以int <-> Integer为例

  • 在jdk5之前（不包括jdk5）都是手动装箱和手动拆箱的
    • 手动装箱：（1）new Integer(n1) （2）Integer.valueOf(n1)
    • 手动拆箱：integer.intValue();
  • 在jdk5之后，采用自动装箱和自动拆箱的方式
    • 自动装箱：Integer integer2 = n2
    • 自动拆箱：n2 = integer2
    • 自动装箱底层还是使用的是valueOf()的方法，自动拆箱底层用的是intValue()方法

包装类和String类型转换

  • Integer -> String：（1）n1 + “” ; （2）i.toString(); （3）String.valueOf(i);
  • String -> Integer：（1）Integer.parseInt(str); （2）new Integer(str)

包装类和Character类常用方法

13.1.3 细节——Integer源码分析

  • Integer.valueOf底层
    • [-128, 127]之间返回IntegerCache[]的引用

13.2 String类

13.2.1 快速入门

String类本质和构造方法

  • String类对象保存的是多个字符拼接在一起的字符序列，双括号括起，如”hello world”
    • 底层实际上用了**private final char value[]**来存储，由于final修饰的是引用类型，那么引用指向的地址空间是固定的，但地址空间里面的值可修改
  • 字符使用UniCode编码（不区分汉字和字母，一律一个字符占2字节）
  • 常用构造器
    • new String();
    • new String(String original);
    • new String(char[] a);
    • new String(char[] a, int startIndex, int count);
  • 可以看到String在实现了Serializable和Comparable接口的基础上，还实现了CharSequence接口

创建String对象的两种方式和区别

先说创建String对象的两种方式

可以发现：

  • 第一种是直接指向常量池空间
  • 第二种是在堆中创建了一个String类，然后String类的value属性指向了常量池空间

给出intern的解释（最终返回的是常量池的引用）

  • 如果常量池包含等于该String对象的字符串(equals方法)，则返回常量池的字符串
  • 否则，把String对象添加到池中，返回字符串的引用

12.2.2 细节1——value的final特性

  • String类是final类，表示该序列不可变
    • 解读字符串不可变：一个字符串对象一旦被分配了，内容就不可以直接在地址空间改变

12.2.3 细节2——变量/常量相加后内存分布

  • 铭记一个重要规则：常量相加，看池。变量相加，看堆
    • 常量相加：底层会自动优化为一个常量
    • 变量相加：底层会创建StringBuilder对象（在堆中），随后在该对象内加入变量元素

13.3 StringBuffer类

13.3.1 为什么要StringBuffer类(可变长)

  • 对于String对象的更新，我们都需要重新开辟一块空间，因此修改效率低
    • String: private final char value[];
  • StringBuffer保存的是字符串变量，而里面的值可以被修改，每次更新不全需要更新地址，效率较高
    • StringBuffer: **char value[] ** // 放在堆中

13.3.2 快速入门

  • java.lang.StringBuffer实际上也是一组可变的字符序列，可以对字符串内容进行增删
  • 之所以修改效率比String高，本质是因为StringBuffer是可变长度的，到达临界点会自动扩容
  • StringBuffer是一个容器：public final class StringBuffer extends AbstractStringBuilder implements `Serializable, CharSequence
  • StringBuffer也有一个value，但该变量不被final修饰
  • StringBuffer类被final修饰，不可被继承
  • StringBuffer类和String类方法很多相似

13.3.3 String和StringBuffer互换

  • String -> StringBuffer：（1）new StringBuffer(str); （2）new StringBuffer().append(str)
  • StringBuffer -> String：（1） stringBuffer.toString()；（2） new String(stringBuffer)

13.3.4 StringBuffer常见方法

  • 增：append(str)
  • 删：delete(start, end)
  • 改：replace(start, end, str)
  • 查：indexOf()
  • 插入：insert(index, str)
  • 长度：length()

应用——价格隔3位加”,”

  • 使用StringBuffer构造器来从Scanner读入的String转为StringBuffer以便操作
  • 使用indexOf查找整数结束的位置，使用insert进行插入”,”

13.3.5 细节——StringBuffer源码分析

得出两个重要的扩容结论

  • 调用构造器后，会在原来基础的长度上加16
  • ensureCapacityInternal()方法会判断当前长度是否满了，满了的话就使用Arrays.copyOf方法
    • 使用newCapacity(minimuncapcity) 根据当前容量得出扩容后的容量为原来的两倍再加1

得出两个关于null的重要结论

  • StringBuffer会在原来长度基础上加16构建字符序列，他直接获取字符串长度，传入null直接报空指针异常
  • StringBuffer.append(null) 调用了appendNull方法，该方法具体操作就是在原来基础上直接加上”null”字符序列

13.4 StringBuilder类

13.4.1 为什么要StringBuilder(线程安全-速率)

  • 用于字符串缓冲区被单个线程使用的时候，效率比StringBuffer快
    • StringBuilder主要是append和insert方法，重载了这些方法以接收任意类型的数据
  • 但是在多个线程使用时，StringBuffer有互斥机制，保证线程安全，而StringBuilder不保证线程安全，区别也就是在这里了

13.4.2 快速入门

方法同StringBuffer

13.4.3 String vs StringBuffer vs StringBuilder

  • StringBuilder和StringBuffer类似，都是可变的字符序列，方法一样（区别就是线程安全和效率的问题）
    • 而String是不可变的字符序列，效率低，但是复用率高
    • 比如s+=”b”操作的时候，原先的字符串对象会被舍弃，产生一个新的字符串，多次执行的话，会导致大量副本留存在内存中，降低了性能

13.9 日期类

13.9.1 第一代日期类Date

  • Date类：精确到毫秒，表示特定的时间

    • 无参是当前时间
    • 亦可带一个毫秒的参数

  • SimpleDateFormat类：解析格式（日期<->文本）

    • 解析成文本：format
    • 解析成日期：parse（注意可能报异常ParseException）

13.9.2 第二代日期类Calendar

  • Canlendar是一个抽象类，获取日历字段的年月日时分秒（月份从0开始）提供了特定的方法
    • public abstract class Calendar implements Serializable, Cloneable, Comparable<Calendar>
    • 抽象类是不能实例化的，可以调用getInstance()静态方法获取
    • HOUR是十二小时进制的时，HOUR_OF_DAY是24小时进制
    • 没有提供输出方法，需要自行组合

13.9.3 第三代日期类

前两代的缺陷

  • jdk1.0引入java.util.Date类，但自从jdk1.1引入Calendar类后，Date大多数方法被弃用了

  • 而Calendar又存在下面的问题：

    • 可变性：日期和时间应当是不可变的
    • 偏移性：年份从1900开始，而月份是从0开始
    • 格式化：格式化只能对Date有用，Calendar只能自行格式化
    • 线程安全：他们都不是线程安全的，不能处理闰秒（隔两天会多1秒）

  • 给出LocalDateTime、LocalDate、LocaTime三个类

    • 使用now()方法获取当前日期的对象

    • getMonth返回的是英文，getMonthValue()返回具体的值

    • 对象可以使用plusYears，minusHours来对时间进行操作

  • 提供DateTimeFormatter格式化（初始化时使用DateTimeFormatter.ofPattern自定义格式）

    • format解析 日期–>文本

  • LocalDateTime其他方法（具体查阅java.time.LocalDateTime）

    • MonthDay：检查重复事件
    • 判断闰年
    • 增加日期的某个部分
    • 使用plus minus计算时间前后的日期

19.9.4 Instant时间戳

可以和Date互转

  • （1）Instant –> Date：Date.from(Instant.now())
  • （2）Date –> Instant：date.toInstant()
