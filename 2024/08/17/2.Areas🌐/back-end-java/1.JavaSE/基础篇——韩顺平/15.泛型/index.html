十五、泛型

15.1 为什么要泛型

  • 如果要在ArrayList中添加三个Dog对象，用传统的多态实现，会有两个问题
    • 每次调用Dog对象方法都需要向下转型一次
    • 如果不小心加入了Cat等其他对象，可能会出现异常

引出泛型解上述问题，有如下好处

  • 编译时，会检查决元素的类型，提高了安全性
  • 减少了元素转换次数，提高了效率
  • 没有编译警告

15.2 快速入门

泛型：广泛的类型，用一个标识符<E>或者其他可以表示Integer、String、Dog等等类型

  • 泛型也叫参数化类型，jdk5.0以后出现的，用以解决数据类型的安全性问题
    • 如果在编译时没有发生警告，那么运行时就不会产生ClassCastException
    • 同时也让代码更加简洁、健壮
  • 类声明和实例化的时候指定具体类型
  • 泛型的作用：在类声明的时候通过一个标识符来表示某个属性的类型，或者某个方法的类型，或者是参数类型
    • 一句话：可以接收数据类型的一种数据类型

15.4 细节——泛型的使用

  • 标识符E T这些都只能表示引用类型
    • 比如List<int> list2 = new ArrayList<>() 就不对
  • 给定泛型的类型后，可以传入该类型或者子类
  • 定义的时候泛型可以简写，编译器会自动识别
    • List<Integer> list2 = new ArrayList<>()
  • 如果直接写List list2 = new ArrayList() ,默认给的泛型<E>就是Object类

12.6 自定义泛型

12.6.1 自定义泛型类

  • 普通成员可以用泛型（属性方法）
  • 创建对象如果没有指明泛型，默认为Object
  • 泛型类的类型，必须在创建对象之后确定
    • 使用泛型的数组，不能初始化( 因为数组在new的时候不能确定T的类型，无法在内存开空间）
    • 静态方法不能使用泛型（静态方法是在累加载阶段，此时对象没有创建）

12.6.2 自定义泛型接口

  • 和泛型类相同之处
    • 接口中，静态成员不能用泛型
    • 没有指定类型，为Object类
  • 而泛型接口的类型，是由继承接口和实现接口两种方式确定的

12.6.3 自定义泛型方法

  • 泛型方法除了可以定义在泛型类中，也可以定义在普通类中
  • 泛型方法被调用 的时候，才确定泛型类型
  • 注意区分泛型的使用
    • public <T, R> void test() { } ：自定义泛型方法，调用方法的时候确定
    • public void test(E e) { }：泛型的使用，创建对象的时候确定

12.7 泛型的继承和通配符

  • 泛型不具备继承性
  • <?>表示接收任意类型的泛型
  • <? extends A>表示接收A及其子类：规定了泛型的下限
  • <? super A>表示接收A及其父类：规定了泛型的上限

12.8 JUnit单元测试

12.8.1 为什么要JUnit单元测试

  • 一个类有多个代码，为了测试，需要写入main方法
  • 多个功能的方法，需要频繁切换，来回注释等操作
  • 如果能单运行一个方法，测试一个方法的功能就方便很多 –> Unit
