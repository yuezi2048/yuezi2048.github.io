23.2 反射机制

23.2.1 反射概念

  • 反射允许程序在runTime时借助Reflection API取得和操作任何类的内部信息（成员变量field、构造器constructor，方法Method等）

    • 设计模式和底层框架会用到

  • 类加载后，堆中会加载一个class对象（一个类只有一个class对象），这个对象就映射了类的完整结构信息

    • 比如Person类会有一个Class对象，他包含的类里的所有结构

23.2.2 *反射机制原理图

java程序的三个阶段

  • 编译阶段，将我们所写的代码javac编译后得到Cat.class字节码文件
  • 类加载阶段，通过类加载器ClassLoader将字节码映射为一个 Class类对象（存放在堆中）
  • 运行阶段，（1）new一个Cat对象的时候，对象可以知道自己的Class类对象；（2）得到Class类对象后，可以进行创建对象、操作属性等操作

23.2.3 反射快速入门

反射主要可以做如下操作：

  • 类与对象
    • 运行时判断任意一个对象的类
    • 运行时构造任意一个类的对象
  • 类的成员和方法
    • 运行时得到任意一个类的成员变量和方法
    • 运行时调用任意一个类的成员变量和方法
  • 动态代理

相关API

  • java.lang.Class：表示一个类。类加载以后在堆中的对象
  • java.lang.reflect.Field：表示类的成员变量，生成的对象指向一个类的某个字段
  • java.lang.reflect.Constructor：表示类的构造方法，生成的对象指向一个类的某个构造方法
  • java.lang.reflect.Method：表示类的方法。生成的对象指向一个类的某个方法

Reflection01

23.2.4 反射优缺点

  • 优点：动态地创建和使用对象（框架底层核心），使用灵活
  • 缺点：反射是解释执行，执行速度较慢
    • 优化：调用方法成员时关闭访问检查setAccessible(false)

23.3 Class类

23.3.1 Class类概念

  • Class类框架
    • Class类也是类，继承Object类
    • Class类由系统创建，不是new出来的
  • 类的实例
    • 每一个类的实例对应一份Class类对象，因为类加载一次
    • 每个类的实例会知道自己对应哪份Class类对象
  • Class内存结构
    • Class类对象存放在堆中
    • 类的字节码二进制数据（元数据）存放在方法区中，指向我们堆中的Class类

23.3.2 Class类常用方法

Class类可以get到

  • Class类对象——类对象运行类型
  • 包名
  • 全类名
  • 获取class类对象实例
  • 获得和修改 字段、构造器、方法（getDeclaredFields()可以获取所有属性）

23.3.3 获取Class类对象的6种方法

  • 第一种：已知类的全类名，读取全路径，使用Class静态方法（常见于配置文件）
    • Class<?> cls = Class.forname(“java.lang.Cat”);
  • 第二种：已知具体的类，通过类的Class获取，该方式最安全且效率最高（多用于参数传递，比如反射得到对应的构造器对象）
    • Class<Cat> cls = Cat.class;
  • 第三种：已知具体的对象实例，调用该实例的getClass()获取（实际上获取到运行类型）
    • Class<? extends Cat> cls = cat.getClass();
  • 第四种：对象实例使用classLoader
    • ClassLoader classLoader = cat.getClass().getClassLoader();
    • Class<?> cls4 = classLoader.loadClass(classAllPath);
  • 第五种：基本数据类型使用.class获取
    • Class<Integer> cls = int.class;
  • 第六种：包装类使用.TYPE
    • Class<Integer> cls = Integer.TYPE;

23.3.4 哪些数据类型有Class类对象

  • 外部类，成员内部类，静态内部类，局部内部类，匿名内部类
  • interface 接口
  • 数组
  • enum 枚举
  • annotation 注解
  • 基本数据类型
  • void

23.4 类加载

23.4.1 什么时候类加载

  • 静态加载
    • 创建对象（new）
    • 子类被加载，父类也被加载
    • 调用类的静态成员
  • 动态加载
    • 反射

23.4.2 *类加载三个阶段

java运行有三个阶段

  • 加载 Loading

    • 将字节码从不同的数据源（class文件 jar包 网络编程）转化为二进制字节流载入到内存中，生成java.lang.Class对象

  • 连接 Linking

    • 验证 verification

      • 为了确保Class文件的字节流符合当前jvm虚拟机要求，并且不危害虚拟机安全
      • 包括：（1）文件格式验证(是否以魔数oxcafebabe开头)；（2）元数据验证、字节码验证、符号引用验证
      • 可以使用Xverify:none关闭大部分类验证，缩短虚拟机类加载的时间

    • 准备 Preparation

      • jvm在此阶段加载静态变量，分配内存并初始化（默认初始值 0 0L null false等）变量的内存在方法区中分配
      • static final int n3底层做了优化，n3在准备阶段会直接赋值为常量

    • 解析 Resolution

      • 符号引用转为直接引用的过程（地址）
      • 理解：两个地方相对距离–>两个地方在地球上的实际经纬度

  • 初始化 Initialization

    • 到初始化阶段才是真正执行程序员定义的java代码，初始化执行的是<clinit>的过程
    • clinit
      • 按语句在源文件中出现的次序，依次自动收集所有静态变量的赋值和静态代码块的语句，并进行合并
      • 虚拟机保证<clinit>方法在多线程环境下正确加锁，同步。多个线程初始化一个类，只有一个线程会执行，其他线程阻塞等待
