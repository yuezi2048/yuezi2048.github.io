一、高精度算法

高精度具体可以做到1e6位数的加减以及对小于1w的数乘除

注：本质是用数组操作，因为我们运算一般是从低位开始算的，所以加减乘一般都是要倒序存，即下标从低位开始存。

1.1 高精度加法

不难发现每次运算都会有进位0或者1

算法思路

  • 倒序存储到数组A B
  • 同时遍历两个数组，t = t+A[i]+B[i] t % 10加到后面，进位t 过更新为 t / 10
  • 最后的进位如果是1 再加进去
  • 倒序将数组输出C就是答案

1.2 高精度减法

不难发现每位运算的时候要借位0或者1，当借位是1的时候，被减数加10。

算法思路

  • 同样地，倒序存入数组
  • A ≥ B 的话 就算 A - B 否则 算 B - A，这里引入cmp函数，思路是先看长度，再从高位逐个比较各数字
  • 同样地 同时遍历两个数组，如果A - B - 借位 大于等于0 则 该位结果就是该非负数，否则 计算该位结果需要加上十，并设置借位为1
    • 我们可以将两种情况合并，如果t大于0 ==> t ，如果t小于0 ==> t+10 合并起来就是 (t + 10) % 10
  • 注意前导0问题，长度大于1的情况下，逐个剔除高位0即可
  • 将结果数组C倒序输出

请注意：此算法要求确保A ≥ B 且 A B 为正数，所以需要加一个判断函数cmp(A, B)，不满足就把A B倒过来算 B - A。同理A B负数则需要额外讨论

1.3 高精度乘法(大乘小)

我们看成是加法的推广，具体体现在进位上，注：结果数要加上进位，进位依旧是默认为0。

然后这边我们注意一下乘数是整体乘即可，也别忘了最后的那一个进位t，这里是放在了for循环一起了，遍历A用条件来判断了。

思考：为什么每次整除10就是下一位的进位？根本原因是十进制的组成就是111 - 1*1 + 10*1 + 100 * 1

即：每当我整除10，那么就是下一位的那一位的实际进位

1.4 高精度除法(大除小)

注意：之前我们都是从最低位开始算的，除法用高位开始可能简单一点，但是为了统一 同时使用，这里也采用了从低位开始存储的模板。

  • 关键步骤是根据被除数逐个求余数
    • 从高位开始逐个与除数相除，余数r（初始0） 拉下来
    • 拼上后面一位的被除数A[i]，那么最终的商C第一位就是r / b
    • 余数r 更新为r % b，如此循环 直到最后一位。

细节问题：

  • 因为是倒序存储的，我们又需要从高位开始，所以负负得正，div函数也要倒过来进行处理（输入问题）
  • 我们得到的商由于输出倒序输出，所以也需要倒序处理（输出问题）
  • 注意前导0问题

二、前缀和 差分

2.1 前缀和

两个问题弄明白

  • Si怎么求（递推）
    • s[0] = 0 , for (int i = 1; i <= n; i ++ ) s[i] = s[i - 1] + a[i]
  • Si的作用
    • 快速求出原数组一段数[l, r]的和S = S[r] - S[l - 1]

细节问题：下标建议从1开始，为了给0下标腾出边界s[0] = 0，好处就是少了一个特判。

譬如，我们要求【1，10】的前缀和S = S[10] - S[0] = S[10] ，在这样一种特殊情况下，这样我们就与原来的公式统一了

预处理O(n)，查询O(1)

现在我们要求【l, r】的和

2.2 二维前缀和

是前缀和的推广，用于求出一个二维矩阵的和，在一维的基础上 ，关键是对这个公式的理解。

用到了所谓的容斥原理，借一个左上角的数字然后又还过来了。

注意：

  • 算部分和的时候，要分清起点和终点的位置，要减去1的地方永远是起点，因为和里要包含起点的位置。

  • 我们让下标从1开始，这样有一个好处：减1之后如果到了边界的话就不需要考虑边界问题了，取而代之的是0处下标全部为0

2.3 差分

是前缀和的逆运算，即构造一个数组，使得B数组的前缀和是a[i]，b是a的差分，a是b的前缀和

我们可以通过O(n)的时间来将差分数组B得到前缀和数组A，但我们通过差分数组只需要O(1)的时间复杂度

我们想象一种场景：

  • 比如我们需要将数组A的一个区间[L,R]内每个数值加上c，按照原来的循环想法是O(n)的复杂度
  • 而我们如果转换到其差分数组上。我们只需要使差分数组B[L] += c，这样一来A在区间[L, ∞]上的数会全部自动加上c
  • 我们再对差分数组B[R + 1] -= c，这样依赖[R + 1, ∞]的数全部自动减去c
  • 这样一来，在前缀和A数组中，[R + 1, ∞]的，没有任何变化，而我们实现了区间[L, R]的数组加上了c。

请注意，差分不用想怎么构造，因为我可以立马告诉你一个差分构造的方式

  • 我假定a数组初始全部是0，那么差分数组b也全部是0
  • 我们可以看成是n次插入操作，即第一次将原数组[1, 1]区间内的元素 加上 a1，第二次将原数组[2, 2]区间内的元素 加上 a2…
  • 差分里面只需要做一个操作，第一个数 + c ，最后一个数后面一位减去c

2.4 二维差分

通过原二维矩阵a 生成差分二维矩阵b，从而满足原矩阵是差分矩阵的前缀和

我们可以类比一维差分和二维前缀和的容斥原理，得到差分b数组对应的四个操作，这里就注意一下横竖都要减去一个c，然后右下角补回来即可 同样是O(1)的时间复杂度。

构造的方法同样是：让每个元素为0，然后插入长宽为1的子矩阵（使用我们的固定四步操作），最后求我们原来a的值的话，直接用b的前缀和即可。
