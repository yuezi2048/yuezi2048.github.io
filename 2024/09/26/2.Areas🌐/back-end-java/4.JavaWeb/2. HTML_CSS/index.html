1.2 HTML概念

HTML

  • HTML(Hyper Test Markup Language 超文本标记语言)，搭建网站结构，显示相关元素
  • HTML5

HTML5 技术结合了 HTML4.01 的相关标准并革新，符合现代网络发展要求，在 2008 年正式发布。HTML5 由不同的技术构成，其在互联网中得到了非常广泛的应用，提供更多增强网络应用的标准机。

与传统的技术相比，HTML5 的语法特征更加明显，并且结合了 SVG 的内容。这些内容在网页中使用可以更加便捷地处理多媒体内容，而且 HTML5中还结合了其他元素，对原有的功能进行调整和修改，进行标准化工作。

HTML5 在 2012 年已形成了稳定的版本。2014年10月28日，W3C发布了HTML5的最终版。

超文本

  • 在普通文本只能展示字符的基础上，进一步可以展示其他网页、图片、视频、音频等多媒体资源，让网页元素更丰富
  • 实现：通过HTML中的标签
  • 本身是文本，但呈现的效果超过了文本，这就是超文本名字的来由

标记语言

  • 区别于Java编程语言，标记语言是由一系列标签组成(如XML HTML)，没有常变量、流程控制、异常、I/O等功能
  • 每个标签都有固定的含义和页面展示效果

给出标签相关的概念

  • 双标签

    • 如p标签来定义一个段落，<p>叫开始标签，</p>叫结束标签。

    • 开始标签和结束标签之间的部分叫文本标签体，也简称：标签体。

  • 单标签

  • 属性（href=”xxx” 是属性，href属性名，xxx内容就是属性值）

3. CSS使用

CSS 层叠样式表(英文全称：(Cascading Style Sheets) 能够对网页中元素位置的排版进行像素级精确控制，支持几乎所有的字体字号样式，拥有对网页对象和模型样式编辑的能力 ,简单来说,美化页面

3.1 CSS三大引入link

  • 行内式：在元素的开始标签加入style属性，语法 样式名:样式值; 样式名:样式值;
    • 缺点
      • HTML和CSS混合，阅读难度和维护成本高
      • 仅对当前元素有效，重复度高，复用度低
  • 内嵌式：在head标签中使用style标签定义CSS样式
    • CSS样式的作用范围控制要依赖选择器(见3.2)
    • CSS的样式代码中注释的方式为 /* */
    • 内嵌式虽然对样式代码做了抽取,但是CSS代码仍然在html文件中
    • 内嵌样式仅仅能作用于当前文件,代码复用度还是不够,不利于网站风格统一

连接式/外部样式表

  • 可以在项目单独创建css样式文件,专门用于存放CSS样式代码
  • 在head标签中,通过link标签引入外部CSS样式即可
  • 说明
    • CSS样式代码从html文件中剥离,利于代码的维护
    • CSS样式文件可以被多个不同的html引入,利于网站风格统一

3.2 CSS三大选择器xxx {}

1. 元素选择器

  • 根据标签名确定样式的作用范围
  • 语法为 元素名 {}
  • 缺点(太多了且标签单一)
    • 样式只能作用到同名标签上,其他标签不可用
    • 相同的标签未必需要相同的样式,会造成样式的作用范围太大

2.id选择器

  • 根据元素id属性的值确定样式的作用范围
  • 语法为 #id值 {}
  • 缺点（只能一个）
    • id属性的值在页面上具有唯一性,所有id选择器也只能影响一个元素的样式
    • 因为id属性值不够灵活,所以使用该选择器的情况较少

3.class选择器

  • 根据元素class属性的值确定样式的作用范围
  • 语法为 .class值 {}
  • 优点（一个元素可以多个样式，一个样式可以多个元素）
    • class属性值可以有一个,也可以有多个,多个不同的标签也可以是使用相同的class值
    • 多个选择器的样式可以在同一个元素上进行叠加
    • 因为class选择器非常灵活,所以在CSS中,使用该选择器的情况较多

3.3 CSS浮动float

CSS浮动(Float)，可以使得元素脱离文档流，按照指定的方向（左/右）移动，直到外边缘碰到框或者另外一个浮动框为止

  • 最初设计Float是为了解决文字环绕图片的问题，浮动后图片一定不会被文字挡住
  • 文档流：文档中可显示对象在排列时所占用的位置/空间
    • 脱离文档流的意思就是在页面中不占用位置了

浮动原理

以框1浮动为例

  • 当框1向右浮动时，脱离文档流并向右移动，框1的右边缘碰到了整个框的右边缘停止，框2和框3顺势向上补位

  • 当框1向左浮动时，脱离文档流并向左移动，框1的左边缘碰到了整个框的左边缘停止，框2和框3顺势向上补位（注意此时框1已经脱离了文档流）

    • 实际效果：框1将框2覆盖住，框2在视图中消失

  • 如果将3个元素都向左浮动，框1碰到整个框的左边缘停止（框2和框3顺势向上），框2碰到框1的右边缘停止（框3顺势向上），框3碰到框2的右边缘停止

  • 注意此时在3个元素向左浮动的情况下，如果包含框太窄，无法容纳水平排列的三个浮动元素，那么其它浮动块向下移动，直到有足够的空间
  • 如果浮动元素的高度不同，那么当它们向下移动时可能被其它浮动元素“卡住”

浮动的样式名:float

3.4 CSS四大定位location

position 属性指定了元素的定位类型。

  • position属性定义元素布局的定位，任何元素都可以定位。
    • 相对relative元素会在正常流中从默认位置偏移
    • 绝对absolute/固定fix元素会生成块级框，脱离文档流
  • 元素可以使用的顶部top，底部bottom，左侧left和右侧right属性定位，但前提是需要用position定义

  • 静态定位static
    • 不设置的时候的默认值就是static，静态定位，没有定位，元素出现在该出现的位置
    • 块级元素垂直排列，行内元素水平排列

  • 绝对定位absolute
    • absolute ,通过 top left right bottom 指定元素在页面上的固定位置
    • 定位后元素会让出原来位置,其他元素可以占用（脱离了文档流）

  • 相对定位relative
    • relative 相对于自己原来的位置进行地位
    • 定位后保留原来的站位,其他元素不会移动到该位置

  • 固定定位fixed
    • fixed 失踪在浏览器窗口固定位置,不会随着页面的上下移动而移动
    • 元素定位后会让出原来的位置,其他元素可以占用（脱离了文档流）

3.5 盒子模型box model

  • 盒子模型"box model"，本质上是一个盒子，封装周围的HTML元素

  • 从外到内组成：边距（margin）、边框（border）、填充（padding）、和实际内容（content）

说明：

  • Margin(外边距) - 清除边框外的区域，外边距是透明的。
  • Border(边框) - 围绕在内边距和内容外的边框。
  • Padding(内边距) - 清除内容周围的区域，内边距是透明的。
  • Content(内容) - 盒子的内容，显示文本和图像

以margin为例

  • 四个参数写法：margin: top, right, bottom, left
  • 两个参数写法：margin: top/bottom, left/right
    • 居中写法：margin: 0px auto;
  • 一个参数写法：margin: top/bottom/left/right

注：padding没有auto

  • 内边距默认不会侵占原有空间的容量，因此效果是往下塌了一段
    • 因为我既要保证文字的内容是100*100又要保证内边距，所以只能延长

Session1.1 为什么要Session HTTP是无状态协议(Stateless)，即HTTP协议本身不会保存请求和响应的通信状态 例如，在客户端请求——服务端回应 的C/S响应模式下，服务器并不会记录该请求来自于哪个浏览器，即客户端的状态 生活例子：张三下馆子吃了一次，第二次再来吃，老板在有状态的情况下知道张三上次吃的啥，在无状态的情况下则不知道，需要张三重新点 引出使用Cookie配合Session解决HTTP协议中请求和响应的持久化问题 Cookie记录少量数据，通过响应头set-cookie来向客户端响应用户要保留的信息 Session记录更多数据，通过HttpSession对象保存和客户端相关的信息 Session的传递配合Cookie一起使用 生活例子：张三去银行办业务，银行给张三开户（Session），并发了银行卡（Cookie），后续张三通过Cookie给银行，银行就会通过Cookie查询到张三的Session 1.2 Cookie1.2.1...
