十九、IO流

19.1 文件

19.1.1 文件流

  • 文件是保存数据的地方，例如word，txt，声音，视频
  • 文件流：文件是通过流的方式进行操作的（举例：人的胃和杯子里的水）
    • 流：在数据源（文件）和程序（内存）之间的路径，根据两个方向可以分为输入流和输出流

19.1.2 创建文件对象

  • 依据路径创建
    • new File(String pathName)
  • 父目录+子路径创建
    • new File(File parent, String child)
    • new File(String parent, String child)

19.1.3 文件方法

文件相关

  • 获取文件信息
    • getName
    • getAbsolutePath
    • getParent
    • length
  • 判断文件
    • exists
    • isFile
    • isDirectory

目录相关

  • mkdir() 创建一级目录
  • mkdirs() 创建多级目录
  • delete() 删除空目录或文件
    • 注意删除文件时，文件需要存在，用file.exists()判断
  • 细节：上述方法返回布尔值，所以需要用if嵌套

19.2 IO流体系框架

  • I/O：Input/Output所写，用于数据传输（读写文件、网络通讯等）
  • Java中，对于数据的I/O 通过流（Stream）的方式进行
  • java.io提供了各种流的接口和类，用以输入输出不同类型的数据（流的理解：外卖小哥从物流中心送外卖的例子）
    • Input：读取外部数据（磁盘等存储数据）到程序（内存）中
    • Output：程序（内存）的数据输出到磁盘等设备

流的分类

  • 根据操作的数据类型分为：字节流Stream（传输二进制）和字符流Reader/Writer（传输文本）
  • 根据流的方向分为：输入流Input和输出流Output
  • 根据流的角色分为：节点流、处理流/包装流

19.3 File_I/O

19.3.1 FileInputStream

  • FileInputStream构造方法：传入文件对象/路径

  • 从文件流中读取字节，-1表示读取结束

    • int read()：从文件流读取一个字节，返回该字节

    • int read(byte[])：从文件流读取多个字节，返回字节的长度

      • 读取后使用new String(byte[], offset, length)的形式表示
      • 细节：可能会没有读满8个字节，没有读满的部分还会保留上一次读取的内容（因此在后续写入的时候需要考虑长度）

  • 细节：注意，文件流需要关闭

19.3.2 FileOutputStream

  • FileOutputStream构造方法：传入文件对象/路径

    • 注：可以在第二个参数append加入true，表示追加文件
    • fileOutputStream = new FileOutputStream(filePath, true); // 追加

  • 写入字节

    • void write(int) 表示写入单个字节
    • void write(byte[]) 写入多个字节
      • String->byte[]：使用str.getByte()方法
      • 若需要偏移量和长度的要求：write(byte[], offset, length) 自定义多个字节

19.3.3 应用——图像copy

19.3.4 FileReader

  • 创建FileReader对象：传入一个文件/文件路径
  • 读取文件的方法
    • int read()：读取单个字符，返回该字符，读到末尾返回-1
    • int read(char[])：读取多个字符，返回读到的字符数，读到末尾返回-1
      • char[] -> String：new String(char[])
      • 自定义char[] -> new String(char[], off, len)

19.3.5 FileWriter

  • 创建FileWriter对象：传入一个文件/文件路径，第二个参数加入true表示追加模式
  • 写入文件的方法
    • void write(int)：写入单个字符
    • void write(char[])：写入字符数组
    • void write(char[], off, len)：自定义写入字符数组
    • void write(String)：写入字符串
    • void write(String, off, len)：自定义写入字符串
      • String -> char[] ：str.toCharArray()
  • 特别注意：FileWriter必须关闭（close）或刷新（flush），否则还保留在内存中，无法写入到文件！
    • close()等价于close()+flush()

19.4 Buffered_I/O

19.4.1 节点流 vs 处理流

节点流： 程序从特定数据源（文件等）中来读取数据（FileReader / FileWriter）

  • 节点流是低级流，直接与数据源相连

处理流（包装流）：在已存在的流的基础上，提供进一步功能

  • 处理流包装了节点流，消除了不同节点流之间的差异，提供了更为方便的方式来输入输出
  • 采用修饰器设计模式，不会与数据源直接相连（后续会模拟该设计模式）
  • 好处：（1）性能提高，采用缓冲提高效率；（2）提供便携的方法输入输出大量数据；（3）关闭流只需关闭外层流即可，无需关注内部的节点流

19.4.2 模拟修饰器设计模式

首先明确Reader的子类继承关系

模拟父类Reader

模拟节点流BufferedReader

  • 在BufferedReader类中，可提供便携的方法输入输出大量数据，或者通过缓冲技术char[]等方式来增加read的效率

模拟Reader类的两个子类FileReader和StringReader（他们代表节点流）

19.5 Object_I/O

19.5.1 序列化/反序列化

  • 需求：
    • 要保存int num = 100，new Dog(“阿黄”, 3)（包含数据类型和数据），并且后续能从文件中恢复数据类型和数据
  • 对应序列化和反序列化
    • 序列化：保存数据时，保存数据类型和数据的值
    • 反序列化：恢复数据的数据类型和数据的值
  • 注意：要让对象支持序列化/反序列化，必须保证该类可序列化，两种方法
    • Serializable接口：无方法实现
    • Externalizable接口： 需要实现方法

19.5.2 ObjectOutputStream 序列化

使用ObjectOutputStream对象的writeXxx方法即可传入我们的数据类型和数据内容

19.5.3 ObjectInputStream 反序列化

注意细节：

  • static和transistent成员不能被序列化
  • 成员的类型也需要可序列化
  • 可以提供serialVersionUID作为序列化的版本号，提高兼容性
  • 反序列化和序列化的顺序要保持一致
