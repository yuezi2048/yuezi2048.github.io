6. 排序

六、排序

（默认升序）

  • 就地排序、非就地排序 –> 空间复杂度
  • 内部排序、外部排序（只能拿一部分数据在内存中排，其他的存在外存中）—> 基本上是内部排序
  • 稳定性：重复数据的相对位置如果不发生变化，那么就是稳定交换
  • 时间复杂度

排序算法包括：

  • 插入：直接插入排序、shell排序（缩小增量排序）
  • 交换：冒泡排序、快速排序
  • 选择：简单选择排序、堆排序（堆）
  • 归并排序：2-路归并
  • 计数排序：桶排序、基数排序

约定

  • 上课案例：a[1] - a[n]
  • 数组分成两部分：乱序区、有序区
    • 初始：整个数组是乱序区
    • 乱序区不断缩小，有序区不断增大
    • 最终：整个数组是有序区

6.1 插入

6.1.1 插入排序

  • 初始：a[1]是有序区、a[2]-a[n]是乱序区
  • 每次把乱序区的一个数据插入到有序区中的合适位置，有序区长度+1，乱序区长度-1，执行n-1次为止

如何把数据插入到有序区的合适位置，即对于a[1]~a[i-1]的有序区，如何将a[i]加入到有序区中

  • 找位置：a[i]从后往前比，直到遇到第一个小于等于a[i]的数a[k]，则 k+1 则是a[i]应该插入的位置

评价：

  • 数据量越少，排序越快
  • 数据越有序，排序越快，最好的情况可以是O(n)时间复杂度

一些鸡肋的优化：

  • 找位置的查找过程，替换成折半查找，折半插入排序
  • 插入的移动过程，基于循环数组，2-路插入排序
    • 比a[0]大的往后移，比a[0]小的往前插，移动能少一半，但是数量级没有变
  • 代码结构优化：查找和插入的过程合并，但是时间复杂度是没有变化

有用的优化：希尔排序（缩小增量排序）

6.1.2 shell排序

我们可以通过分组来发挥插入排序的优势（基本有序+数据量少） —> 减小增量排序

  • 增量d

    • 组数，即将n个数据分成d组，每组n/d个数据
    • 分组策略：

  • 缩小：例如 d = n / 2. d = d / 2…. d = 1

  • 获得不同的增量序列：n/2 n/4, n/8, … 1 —–> 希尔增量序列，希尔排序

代码：

6.2 交换

基于交换的排序：冒泡排序、快速排序

  • 交换：排序过程中，不断比较两个关键字，根据比较的结果进行关键字位置的交换，直到排序完成为止

6.2.1 冒泡排序

  • n个数据进行排序，排n-1躺，数组分为有序区和乱序区，初始全部都是乱序区
  • 在每一趟中，从前往后不断比较相邻的两个数据，如果a[i] > a[i + 1]，交换两数，最终将乱序中最大的数交换到最后的位置上，有序区++

鸡肋的优化：标记一趟是否发生交换来判断前面的乱序区是否已经有序。

6.2.2 快速排序

  • 找一个基准数：把比基准数小的数，放到基准数的前面，比基准数大的数放到基准数的后面，然后从基准数的位置分成前后两部分，再分别堆前后两部分分别进行快速排序
  • 基准数的选择？假设对于a[l] ~ a[r]的数进行排序
    • 选第一个数作基准数 a[l]（为例）
    • 选最后一个数做基准数 a[r]
    • 选中间位置的数的基准数 mid = (l + r) / 2，a[mid]
  • 如何实现把小的放基准数前面，大的放在基准数后面：挖空法（本例演示）、左右指针法、前后指针法（讲义）

6.3 选择

6.3.1 选择排序

  • n个数排序，分为有序区和乱序区
    • 执行n -1 躺排序，第i躺排序时，乱序区为 i ~ n
    • 每趟排序，在乱序区中找到最小的数据，放到乱序区的第一个位置即可

6.3.2 堆排序

用堆对选择排序进行优化

  • 堆（Heap）：一种基于完全二叉树的数据结构，根据约束条件的不同，分为两类（查找最小值的优化）
    • 大顶堆：根结点的值大于其所有孩子结点的值，对于每一颗子树都满足该约束
    • 小顶堆：根节点的值小于其所有孩子结点的值，对于每一颗子树都满足该约束

以小顶堆为例：建堆（完全二叉树通过数组来存储，父子关系可以通过下标体现）

  • （1）自初始化建堆：直接认为存数的数组就是堆，但不满足小顶堆约束，从最小的子树开始调整，以a[n / 2] 为根的子树开始倒着遍历每颗子树即可
    • 自上而下调整：从根结点p开始调整，让p和左右两个孩子比较，如果孩子比p小，则p和最小的孩子交换，再以 交换的孩子做新的调整节点p，直到p比两个孩子都小，或p为叶子节点为止
  • （2）插入建堆：认为是空堆，不断插入数据，进行调整
    • 先把数据插入到堆的最后，此时该数据是叶子节点，向上进行调整，直到调整到根节点，或者满足小顶堆的约束为止
  • （3）删除：每次删除堆顶元素，把最后一个位置的数据放到堆顶，从堆顶向下调整

堆排序：

  • 先对待排序建堆（升序建小顶堆），堆顶就是最小值
  • 循环n次：输出堆顶（或将堆顶放到有序数组中），执行堆的删除操作

注：升序也可以建大顶堆，删除的时候是将最大的数与最后一个元素交换，就地排序

  • 因此想要升序排列，既可以是小顶堆，也可以是大顶堆（原地算法，建议）

堆的应用：

  • 堆排序
  • 优先队列的优化
  • 图算法 迪杰斯特拉算法优化，寻找最大最小值通过构建堆优化为O(logn)

6.4 归并排序

  • 归并排序：分治（分而治之）
  • 2-路归并
    • 如果一个序列，从中间位置分开，如果其前后两部分分别有序（和最终结果的顺序一样），那么只需要将前后两部分重新合并为整体有序
    • 两两划分：不断地将序列分成两半，直到不能分割为止（每个序列只有一个数据时停止）
    • 两两合并：不断地将前后两个有序序列合并为一个整体有序的序列

6.5 计数

6.5.1 计数排序

  • n个非负整数，统计每个数的个数，和这n个数的最大值maxx
  • 遍历cnt[0, maxx]，诶个输出
  • 如果要保证稳定性， 引入前缀和sumcnt[]，含义：1. 记录最后一个i的位置；2. 0~i一共有？个元素
    • 然后在a[]数组倒着遍历，遍历到的数放到sumcnt的位置，然后sucnt - 1，保证最后一个放到的是最后一个位置，从而保证稳定性
  • 缺点：
    • 以空间换时间，存在较大的空间浪费空间
    • 只能对非负整数排序（小数需要乘以10^{最多小数位数}，负数需要加上minn偏移）

6.5.2 基数排序

  • 计数排序基础上，对每一位分别排序，从而得到有序序列

6.5.3 桶排序

  • 按照某种规则（例如a[i] / 10）分成若干个组（桶），对每组进行排序
