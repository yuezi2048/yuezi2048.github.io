一、基础语法

1.1 编译过程

程序为什么要被编译器编译之后才能运行？

  • 机器识别的只有机器语言（由二进制的0和1组成）
  • 因此我们必须将高级语言翻译为能让机器识别的机器语言程序

转换成机器语言的过程是怎么样的？

  • 预处理：展开头文件/宏替换/去注释/条件编译
  • 编译：检查语法，生成汇编代码
  • 汇编：将汇编代码转化为二进制的机器码
  • 链接合成可执行的程序，并在声明的文件找相关的定义

1.2 宏定义和typedef关键字

define

  • 使用一个标识符来表示一个字符串，这就是宏，标识符叫宏名
  • 替换文本可以是常数、表达式、字符串、基本数据类型、简单函数

typedef

  • 为数据类型定义新名称，常用于结构体（对struct {} 起一个别名）

1.3 变量和常量

  • 全局和局部变量的名称允许相同，但函数内优先使用局部变量（建议用全局变量）
    • 函数的参数，形参是局部变量

注：系统不会对局部变量初始化，要手动初始化。全局变量会自动初始化（char–>’\0’，指针–> NULL，默认0）

  • 常量是固定值，不会改变，使用define或者const声明

1.4 输入输出

  • scanf输入，printf输出

注：在读入字符时，scanf要注意getchar()将回车读掉

1.5 运算符

  • /整除，++自增，–自减，三目运算符
  • 位运算符
    • <<左移，左移一位相当于乘2，右移同理
    • ~取反 ，|或，&按位与，^按位异或 不同为1
      • 举例：a 的高八位清 0 ， 保留低八位， 可作 a&255 运算 ( 255 的二进制数为0000000011111111)

例如交换两个整数a=10100001，b=00000110的值，可通过下列语句实现：

　　　　a = a^b； 　　//a=10100111

　　　　b = b^a； 　　//b=10100001

　　　　a = a^b； 　　//a=00000110

二、C语言指针

2.1 存储器的基本概念

首先明确一个计算机基本的冯.诺依曼架构

  • 存储器分为内存和外存
  • 都是以二进制的形式保存的

  • 我们coding的实际叫做高级语言，硬件执行的是机器语言（底层、特定的指令集）
    • 高级语言：C++ Java等可移植语言，由编译器转换为汇编语言
    • 汇编语言：以助记符表示的机器指令
    • 机器语言：将二进制的指令集存储在内存中，并可执行
  • 指令和数据没有本质区别，都是以二进制为基本单位存储

存储容量表示

  • 1bit 1byte（1B） 1KB 1MB 1GB 1TB
  • 一个二进制位称为1bit，1B=8bit
  • 1B 1KB 1MB 1GB 1TB 由小到大，换算单位是2^10

而为了让编译器转换成对应的指令操作内存空间，也需要通过数据类型来表示内存大小

以64位系统为例：

  • char——1B
  • int—— 4B，float——4B
  • long long——8B，double——8B

引出特定的数据类型——>指针，用于存放数据的内存单元地址，占用8B（通常使用十六进制表示）

2.2 为什么要指针

  • 计算机内存拥有大量的存储单元，如果按字节编址，则一个单元大小就是1B
  • 指针就是为每个存储单元编号，来管理这些存储单元，64位系统使用8位，通常使用16进制表示
    • 指针指向的本质就是地址赋值
    • 指针的操作：&引用（取地址），*解引用（取值）
  • 指针不止可以存放变量地址，结构体、数组、函数的地址也可以存放

为什么要指针？

  • 比如结构体，可能包含大量数据，而指针只需8B就可以表示该内存地址，节省访问时间和内存
  • 指针的字节数是固定的，不管指向的是什么数据类型（邮递单号的例子）
  • 一些分配内存malloc等操作必须通过指针来进行
    • int *p = (int*)malloc(sizeof(int));
  • 我们后续学DS复杂的数据结构的时候，多级指针，链表、树、图，都需要指针
    • 举例：连续的地址不够用时（如定义数组），可以通过指针来解决

2.3 数组指针 vs 指针数组

  • 数组是一个复合数据类型，存储相同类型的元素，在内存空间中连续存放
  • 数组名有两个含义：
    • 数组名代指整个数组空间，如&a，表示对整个数组空间取地址符，即数组指针int (*p)[5]
    • 做指针常量时，保存数组的第一个地址，如*a, 表示得到a第一个元素对应的值

数组指针：指向数组的指针 int (*p)[5]：表示一个指针

  • 执行p+1操作时，需跨过n个数据的长度

指针数组：存放指针的数组，int *p[5]，有n个指针，分别指向对应元素

2.4 高维数组

首先说二维数组

  • 明确：二维数组在内存中不是表格存放，而是连续成行放的（本质就是一维数组）
  • 多维数组的构成实质上就是n-1维数组嵌套形成的更高维数组，如二维数组的每个元素是一维数组

高维数组：以二维数组为例

注：二维数组名不是二级指针。二维指针是变量，而二维数组名是二维数组的首地址，也就是一段连续内存空间的首地址，本质上是一个指针常量。

指针的类型取决于地址空间中保存的数据类型

  • 二维数组名a的终极理解：是指向二维数组首元素的地址（即指向一维数组a[0]的指针常量p2）

    • 做取值操作后，他就相当于对一维数组a[0]的整块地址空间取值的操作（即得到指向一维数组a[0]的指针 b2）

例子：通过二维数组模拟开数组（没有连续的空间开二维数组的应用）

  • 假设二维数组开4行，分别开4次空间，得到4个指向每行空间的指针（总大小4*8字节）
  • 将4个指针，对应再申请5个连续的空间，每个元素用int表示（总大小5*4字节）

3.3 指针和函数

函数实际上也对应一个地址，函数名是函数的入口，指向入口的指针就是函数指针（了解）

主要说明指针在函数中的传参，而函数传参一共有三种方式

  • 值传递
  • 地址传递（指针传递）
  • 引用传递（C++特性）

注：传参的本质是创建副本，指针传递的本质就是值传递

  • 创建副本的理解：在调用函数的时候，会在另外一个空间中，拷贝一份变量，函数会对拷贝的变量进行操作
  • 指针传递本质是值传递的理解：传参关键看相对于谁来说，相对于&b来说就是值传递，相对于b来说就是引用传递
  • 后续在递归等函数调用的过程中，会用到创建副本的概念，每个函数调用之间互不影响
