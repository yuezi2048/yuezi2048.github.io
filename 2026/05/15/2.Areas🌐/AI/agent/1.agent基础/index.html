hello-agent-基础

智能体是什么

智能体是什么：对环境感知，通过执行器进行自主性行动的闭环

  • 智能体的发展历程： 基于反射（世界模型的记忆雏形）——基于目标（Planing概念）——基于效用（多目标权衡）——强化学习（通过状态S和奖励R自我修正的行为主义）

    • 目前到达了基于LLM（涌现能力 few-shots上下文/COT），当收到指令后，agent在执行前分解子任务，执行时工具调用，执行后反思修正

  • 智能体权衡点：速度的反应性 vs 最优解的规划性，衍生出混合式智能体（ReAct）

  • 知识表示的历程：

    • 符号主义：全盘使用已有符号描述，解释性强，但存在知识瓶颈

      • PSSH假说：认为智能本质是可以靠物理符号计算和推理得出的
      • 专家系统（MYCIN系统）：用户界面+知识库（产生式规则IF-THEN描述）+推理机（正向链 从IF到THEN ，反向链 从假设目标THEN出发，将IF作为子目标递归），MYCIN系统中提出了置信因子的概念描述诊断结果可靠性
      • SHRDLU积木世界：首次将多个独立的人工智能模块（语言识别 规划 记忆）整合并协同工作（贡献出整合的思想和微观世界的可行性）
      • ELIZA基于模式匹配/文本替换，进行聊天（I - > You）—— 上下文记忆、语义理解、规则扩展的问题，残忍地揭露出符号主义这种穷举的方法，系统永远不理解，注定难以拓展，非常脆弱。
      • 那么针对以上问题，当时马文·明斯基提出了心智社会：一个搭建积木的BUILD机构，底下有SEE、PUT等下层智能体（当时还是认为智能体是僵化的），他们之间通过 去中心化的激活/抑制，形成动态的控制流。每个无心的组件不知道全局任务是什么，但在这种激活相互作用下，搭建金字塔的行为就涌现了出来

    • 亚符号主义（联结主义）：通过神经网络训练出一个黑箱，可以识别非结构化数据，可解释性弱，相比于符号主义的自上而下，他是自下而上的。

    • 神经符号主义：融合两者，弥合感知与认知、直觉与理性之间的鸿沟

智能体运行机制

  • 首先要明确 任务环境（PEAS模型）：Performance Environment Actuators Sensors
    • 旅行AI助手中，明确在有限时间/预算内，行程要求最大化用户满意度和结果合理度，环境提供了各种地图、天气API等服务，执行器通过fastAPI接口调用执行生成格式化文本，传感器则读入用户的输入，分析转换后的JSON
    • 在任务环境中，我们需要考虑
      • 环境的部分可观测性（引伸出记忆、主动探索能力）、连贯/动态性（感知-行动-观察“循环怎么保证及时输出结果，保证结果的时效性？）
      • 多智能体环境（如何实现多智能体交互？）
      • 行动结果是确定性还是随机性的（对输出的结果需要怎么监测处理？）
  • 运行机制：“感知-思考-行动-观察”的闭环

智能体应用

  • 开发者工具：GitHubCopilot、CC、Trae、Cursor
  • 自主协作者的智能体：早期的 BabyAGI、AutoGPT，到如今更为成熟的 CrewAI、AutoGen、MetaGPT、LangGraph
    • AutoGPT是单智能体“**思考-规划-执行-反思”**自主循环经典范式
    • 多智能体
      • CAMEL通过角色扮演的方式来沟通完成
      • MetaGPT 和 CrewAI模拟一个多智能体团队，每个智能体有一套自己的SOP工作流程，通过层级/顺序工作，交付高质量成果
      • AutoGen 和 AgentScope提供了更灵活的对话模式，允许开发者在智能体之间定义复杂的交互网络
    • 更高级的控制：LangGraph等，将agent执行过程建模成状态图，能更灵活地实现循环、分支、回溯以及人工介入等复杂流程
  • agent区别于workflow：Workflow 是让 AI 按部就班地执行指令，而 Agent 则是赋予 AI 自由度去自主达成目标，他是能基于实时信息动态推理和决策的。

基本LLM模型

N-Gram模型到RNN

预测一个句子的概率$P(w_m \mid w_1,\dots,w_{m-1})$，我们一般会想到链式法则，但我们发现w1 w2..的语料很可能没在训练集出现过，所以是不好直接算条件概率的
$$
P(S) = P(w_1, w_2, \dots, w_m)
= P(w_1) \cdot P(w_2 \mid w_1) \cdot P(w_3 \mid w_1, w_2)
\cdots P(w_m \mid w_1, \dots, w_{m-1})
$$
使用马尔可夫假设。一个词的出现概率近似认为和前面n-1个词相关，这就是N-Gram模型，N是上下文窗口，以N=3为例，第n个词只和前两个词相关
$$
P(w_i \mid w_1, \dots, w_{i-1}) \approx P(w_i \mid w_{i-2}, w_{i-1})
$$
此时通过最大似然估计MLE（看到次数最多的就是最可能出现的）估计，那计算就只需要记录 语料库中 成对出现的次数 / 前一个词出现的次数即可
$$
P(w_i \mid w_{i-1}) = \frac{\text{Count}(w_{i-1}, w_i)}{\text{Count}(w_{i-1})}
$$
问题：数据稀疏（语料库没出现的概率就是0，即使平滑化处理也无法根除），泛化能力弱（无法理解agent和robot的相似性）

此时2003年提出了前馈神经网络语言模型：通过神经网络拟合能力来学习前n-1个词向量，来预测第n个词

  • 词向量：每个词在高维空间中对应一个词向量Embedding，这样就能用余弦相似度等表示语义相近了
    • 经典例子：vector(‘King’) - vector(‘Man’) + vector(‘Woman’) = vector(‘Queen’)

尽管解决了泛化问题，但上下文长度是固定的，引伸出循环神经网络RNN，他的思想就是给网络增加记忆能力。

RNN设计中引入了隐藏状态向量h，理解为网络的短期记忆，RNN会读取当前输入词，结合上一步的$h_{t-1}$，生成当前步的$h_{t}$，这样就可以将信息向后传递。

​

问题：LongTerm的长期依赖问题，序列长了，反向传播（根据输出的误差调整权重）的时候梯度连乘就会出现趋于0（梯度消失）或者趋于无限大（梯度爆炸）的情况，梯度消失后，无法学习到早期信息对后续输出影响，难以捕捉长距离依赖关系。

引伸出长短时记忆网络 LSTM，创新点是引入了细胞状态（独立于隐藏状态的信息通路便于在时间步中传递）和门控机制（小型神经网络，选择性的让信息通过来控制细胞的增加/移除——遗忘门-从上一细胞状态中丢弃/输入门-当前新输入信息存入细胞状态/输出门-输出到隐藏状态）。

Transformer

Encoding-Decoding

Transformer我们看成两个部分：

  1. 编码器 (Encoder) ：负责“理解”输入的整个句子。它会读取所有输入词元，最终为每个词元生成一个富含上下文信息的向量表示。
  2. 解码器 (Decoder) ：负责“生成”目标句子。它会参考自己已经生成的前文，并“咨询”编码器的理解结果，来生成下一个词。

自注意力到多头注意力

自注意力

The agent learns because it is intelligent. 读到it时，注意力应当在agent上，这就是自注意力的数学建模：

  • 序列中每一个词都会兼顾其他所有词赋予不同的注意力权重，越大的就表示关联性越高
  • 自注意力学习的三个核心角色向量
    • Q查询：负责主动查询其他词元
    • K：表示被查询词元的key标签索引
    • V：词元的value内容信息
  • 学习
    • 预料：通过原始词嵌入乘以可学习权重矩阵$W_Q, W_K, W_V$得到Q K V向量
    • 计算相关性：当计算词A的新表示时，通过A的Q向量去和所有词（包括A）的K向量点积运算，得到与A的相关性
    • 稳定化+归一化：除以缩放因子$\sqrt{d_k}$（$d_k$是K的向量维度）套上softmax归一化
    • 加权求和：上一步得到的权重乘以每个词的V向量，将所有结果相加，得到融合了全局上下文信息的最终A向量

$$
\text{Attention}(Q,K,V) = \text{softmax}\left( \frac{QK^\mathrm{T}}{\sqrt{d_k}} \right)V
$$

自注意力问题：当处理it时，可能只会关注到主语，但时态、从属等关系我们在语义理解中同样需要。引入了多头注意力机制（Q K V向量在维度上切成了h份，对应h头），可以有多个专家捕捉不同的特征关系，最终得到h个专家的输出向量后，通过线性变换整合得到最终输出。

前馈神经网络

我们发现每个Encoder和Decoder层在多头注意力子层后跟着一个逐位置前馈网络FNN，如果说注意力是为了在整个序列中动态聚合信息，FNN则是从这个聚合信息中提取更高阶特征。

  • FNN的关键在于逐位置，seq_len的序列意味着会被FNN调用seq_len次，每次处理一个词元，保证了对每个词元加工的能力
  • FNN的网络共享权重变量，极大减少了参数量
  • 第一个线性层的输出维度 d_ff 会远大于输入的维度 d_model，经过 ReLU 激活后再通过第二个线性层映射回 d_model 维度。是“先扩大再缩小”的模式

$$
\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2
$$

残差连接与层归一化

所有子模块中都会附带Add & Norm的操作，目的是保证Transformer的稳定训练

  • 残差连接 Add：输入直连输出，解决梯度消失，深层网络易训练
    • 公式：(Output = x + \text{Sublayer}(x))
  • 层归一化 Norm：解决内部协变量偏移，稳定分布、加快收敛
    • 单样本特征归一化，均值 0 方差 1

位置编码

在使用注意力机制后，虽然学习到了任意两个词之间的关系，但是不包含词元顺序/位置信息。引入位置编码

  • 思想：每个词元嵌入向量额外加上表示绝对位置和相对位置的位置向量，通过以下公式得到：pos是位置，i是向量的维度索引[0, $d_{model}$ / 2]，$d_{model}$是向量维度

$$
\begin{aligned}
\text{PE}{(pos,2i)} &= \sin\left(pos \cdot 10000^{-\frac{2i}{d{\text{model}}}}\right) \
\text{PE}{(pos,2i+1)} &= \cos\left(pos \cdot 10000^{-\frac{2i}{d{\text{model}}}}\right)
\end{aligned}
$$

Decoder-Only

OpenAI 在开发 GPT (Generative Pre-trained Transformer) 时，提出了一个更简单的思想：语言的核心任务，不就是预测下一个最有可能出现的词吗？

引伸出：抛弃编码器，只有解码器的部分，这就是Decoder-Only，结构更简单，目标更单一（预测下一个词），非常适合生成任务，工作方式是自回归：

  • 给定初始文本，预测下一个可能出现的词
  • 将刚刚生成的词添加到末尾，作为新的输入，如此循环，直到完整的句子/停止条件

而解码器为了保证不看到后面的内容，使用到掩码自注意力机制

得到注意力矩阵后，在softmax计算前，将其替换成一个非常大的负数，此时经过softmax变换后就是0，这样就截断了其信息，保证预测下一个词之后，只能依靠已经见过的，当前位置之前的信息，保证公平性和语义连贯性

LLM交互基础

提示词工程

模型采样参数：

  • Temperature：控制随机性和确定性，令T>0，softmax计算公式为$p_i(T) = \frac{e^{z_i/T}}{\sum\limits_{j=1}^{k} e^{z_j/T}}$，T越小，高概率权重会被放大，生成更保守重复率小的文本，反之则是创造性高但随机性大的文本
  • Top-k：将所有token概率从高到低排序，对筛选出来的前k个概率归一化$\hat{p}i = \frac{p_i}{\sum\limits{j\in\text{候选集}} p_j}$
  • Top-p：token概率从高到底概率排序，从排序后的第一个token开始，逐步累加概率，直到到达p，此时所有token组成核集合，进行归一化
    • 更能适应长尾特性，更适应概率分布不均匀场景

样本：零样本提示、单样本提示、**少样本提示 **

指令调优：

  • 少样本提示、角色扮演、上下文示例 、思维链

文本分词

分词 (Tokenization) 、**分词器 (Tokenizer)**的作用：文本序列转换成模型能处理的数字形式，划分成的最小单元，就是词元（token）

分词策略：

  • 早期：按词分词（无法处理未登录词，无语义关系），按字符分词（单个字符没有独立语义）
  • 现代：子词分词，常见词保留完整词元，不常见词划分成有意义的子词片段（Token” 和 “ization”），平衡了词表大小，又能兼顾新词的表示

目前常用子词分词方式：字节对编码 (Byte-Pair Encoding, BPE) ，GPT就是这种算法，思想是贪心：

  • 初始化：词表初始化语料库中出现的基本字符
  • 迭代合并：统计每个词元成对出现概率，成对频率最高的一对合并为新的词元，加入词表
  • 重复上一步，直到到达阈值

很多算法就是在BPE思想基础上优化：

  • WordPiece：Google BERT 模型采用的算法，与BPE很类似，但合并词元的标准不是“最高频率”，而是“能最大化提升语料库的语言模型概率”。它会优先合并那些能让整个语料库的“通顺度”提升最大的词元对。
  • SentencePiece：Google 开源的一款分词工具，Llama 系列模型采用了此算法。它最大的特点是，将空格也视作一个普通字符（通常用下划线 _ 表示）。这使得分词和解码过程完全可逆，且不依赖于特定的语言（它不需要知道中文不使用空格分词）。

模型选择

  • 性能能力：核心！不同模型擅长不同任务（推理/代码/创作），参考公开基准排行榜
  • 成本：闭源按Token计费（API），开源看硬件+运维（本地部署），结合预算选
  • 速度（延迟）：实时交互（客服/NPC）选轻量优化款（如GPT-3.5 Turbo）
  • 上下文窗口：处理长文档/长期对话，选大窗口（≥128K Token）
  • 部署方式：API便捷但数据外发，本地部署保隐私但需技术+硬件
  • 生态工具：选主流模型，配套工具（LangChain等）多、社区活跃，开发更高效
  • 可微调性：特定领域/任务，选支持微调的（开源更灵活），提升场景适配性
  • 安全伦理：面向公众/敏感场景，关注模型偏见、幻觉，优先合规款

LLM特性

缩放法则

缩放法则：模型性能与模型参数量、训练数据量以及计算资源之间存在着可预测的幂律关系。

  • 该定律指出，在给定的计算预算下，为了达到最优性能，模型参数量和训练数据量之间存在一个最优配比。如模型参数量少，但训练数据量大性能反而会好
  • 能力涌现：当模型规模达到一定阈值后，会突然展现出在小规模模型中完全不存在或表现不佳的全新能力，如COT，指令遵循等，这是模型参数到达百亿甚至千亿显著涌现的

只要我们持续、按比例地增加这三个要素，模型的性能就会可预测地、平滑地提升，而不会出现明显的瓶颈

模型幻觉

模型幻觉：大语言模型生成的内容与客观事实、用户输入或上下文信息相矛盾，或者生成了不存在的事实。

原因：

  • 训练语料（RAG，外部工具）

    • 训练数据中可能包含错误或矛盾的信息；
    • 知识时效性不足和训练数据中存在的偏见等挑战

  • 训练过程（自我纠正）

    • 自回归生成机制决定了它只是在预测下一个最可能的词元，没有内置的事实核查模块；

    • 面对需要复杂推理的任务时，模型可能会在逻辑链条中出错；

解决：

  • 检索增强生成（RAG）：生成前从外部知识库检索相关信息，引导模型生成贴合事实的回答
  • 多步推理与验证：引导模型分步推理，每一步进行自我检查或外部验证，提升准确性
  • 引入外部工具：允许模型调用搜索引擎、计算器等工具，获取实时信息或完成精确计算

总结

我们的agnet学习从最基本的问题开始：

  • 什么是大语言模型驱动的智能体？ 我们首先明确了其定义，理解了现代智能体是具备了能力的实体。它不再仅仅是执行预设程序的脚本，而是能够自主推理和使用工具的决策者。
  • 智能体如何工作？ 我们深入探讨了智能体与环境交互的运行机制。我们了解到，这个持续的闭环是智能体处理信息、做出决策、影响环境并根据反馈调整自身行为的基础。
  • 如何构建智能体？ 这是本章的实践核心。我们以“智能旅行助手”为例，亲手构建了一个完整的、由真实 LLM 驱动的智能体。
  • 智能体有哪些主流的应用范式？ 最后，我们将视野投向了更广阔的应用领域。我们探讨了两种主流的智能体交互模式：一是以 GitHub Copilot 和 Cursor 等为代表的、增强人类工作流的“开发者工具”；二是以 CrewAI、MetaGPT 和 AgentScope 等框架为代表的、能够独立完成高层级目标的“自主协作者”。同时讲解了 Workflow 与 Agent 的差异。

并回顾了智能体发展的历史脉络，探索了其核心思想从诞生到演进的过程，内容涵盖了人工智能领域几次关键的范式革命：

  • 符号主义的探索与局限：从人工智能的古典时代出发，本章阐述了以专家系统为代表的早期智能体是如何尝试通过“知识+推理”来模拟智能的。通过亲手构建一个基于规则的聊天机器人，我们深刻体会到这一范式的能力边界及其面临的根本性挑战。
  • 分布式智能思想的萌芽：探讨了马文·明斯基的“心智社会”理论。这一革命性的思想揭示了复杂的整体智能可以从简单的局部单元的交互中涌现，为后续的多智能体系统研究提供了重要的哲学启发。
  • 学习范式的演进：见证了智能体获取能力方式的根本性变革。从联结主义赋予智能体感知世界的能力，到强化学习使其学会在与环境的交互中进行最优决策，再到基于大规模数据预训练的大型语言模型（LLM）为其提供了前所未有的世界知识和通用推理能力。
  • 现代智能体的诞生：最后，我们对LLM驱动智能体进行分析。通过对其核心组件（模型、记忆、规划、工具等）和工作原理的分析，我们理解了历史上的各种技术思想是如何在现代Agent的架构中实现技术融合的。

我们不仅理解了第一章所介绍的现代智能体从何而来，更能建立了一个关于智能体技术演进的宏观认知框架。可以发现，智能体的发展并非简单的技术迭代，而是一场关于如何定义“智能”、获取“知识”、进行“决策”的思想变革。

既然现代智能体的核心是大型语言模型，那么深入理解其底层原理便至关重要。重点围绕作为其核心组件的大语言模型 (LLM) 展开。内容从语言模型的早期发展开始，详细讲解了 Transformer 架构，并介绍了与 LLM 进行交互的方法。最后，本章对当前主流的模型生态、发展规律及其固有局限性进行了梳理。

核心知识点回顾：

  • 模型演进与核心架构：本章追溯了从统计语言模型 (N-gram) 到神经网络模型 (RNN, LSTM)，再到奠定现代 LLM 基础的 Transformer 架构。通过“自顶向下”的代码实现，本章拆解了 Transformer 的核心组件，并阐述了自注意力机制在并行计算和捕捉长距离依赖中的关键作用。
  • 与模型的交互方式：本章介绍了与 LLM 交互的两个核心环节：提示工程 (Prompt Engineering) 和文本分词 (Tokenization)。前者用于指导模型的行为，后者是理解模型输入处理的基础。通过本地部署并运行开源模型的实践，将理论知识应用于实际操作。
  • 模型生态与选型：本章系统地梳理了为智能体选择模型时需要权衡的关键因素，并概览了以 OpenAI GPT、Google Gemini 为代表的闭源模型和以 Llama、Mistral 为代表的开源模型的特点与定位。
  • 法则与局限：本章探讨了驱动 LLM 能力提升的缩放法则，阐述了其背后的基本原理。同时，本章也分析了模型存在的如事实幻觉、知识过时等固有局限性，这对于构建可靠、鲁棒的智能体至关重要。

从 LLM 基础到构建智能体：

这一章的LLM基础主要是为了帮助大家更好的理解大模型的诞生以及发展过程，其中也蕴含了智能体设计的部分思考。例如，如何设计有效的提示词来引导 Agent 的规划与决策，如何根据任务需求选择合适的模型，以及如何在 Agent 的工作流中加入验证机制以规避模型的幻觉等问题，其解决方案均建立在本章的基础之上。我们现在已经准备好从理论转向实践。
