第一次面试

发表于2026-03-27|更新于2026-03-27|2.Areas🌐02.面经记录实习面试总结ai面试
|总字数:1.9k|阅读时长:5分钟|浏览量:

模块 1：Java 集合（基础必背，不能出错）

【必背核心考点】

  1. ArrayList & LinkedList 核心对比

    • 底层结构
    • 不同场景下随机访问 / 插入删除的时间复杂度
    • ArrayList 完整扩容机制（初始容量、触发时机、扩容倍数、流程）
    • 两个集合的适用场景

  2. HashMap 全考点

    • JDK1.7 vs 1.8 核心区别（数据结构、哈希算法、扩容逻辑、死循环问题）
    • 1.8红黑树的进化 & 退化条件、解决的核心问题
    • 哈希冲突的解决方式、负载因子的作用

  3. 线程安全的 Map 实现

    • HashMap 多线程场景的具体问题
    • Hashtable、ConcurrentHashMap 的核心区别与底层实现

【本次待补短板】

  1. 明确 LinkedList 插入的时间复杂度：仅已知目标节点时，插入 / 删除是 O (1)；指定下标插入需遍历链表，时间复杂度为 O (n)
  2. 记牢 ArrayList 扩容细节：无参构造默认初始容量为 0，首次添加元素初始化为 10；元素个数超过数组长度时触发扩容，扩容后为原容量的 1.5 倍
  3. 补充 HashMap 多线程问题的具体表现，以及 ConcurrentHashMap 的底层实现原理(分段锁Segment、or CAS LongAdder)

模块 2：MySQL（面试核心，必须吃透原理）

【必背核心考点】

  1. InnoDB 索引体系

    • 聚簇索引的定义、结构、生成规则
    • 非聚簇索引（二级索引）的定义、结构、回表完整流程
    • 覆盖索引的定义、优势、使用场景

  2. 最左前缀原则

    • 核心规则、生效范围、MySQL8.0 的索引跳跃扫描优化
    • 联合索引的命中场景判断

  3. 慢查询全流程处理

    • 完整排查链路、explain 关键字段解读
    • 常见索引失效场景、慢 SQL 优化手段

  4. 事务 ACID、隔离级别、MVCC 核心原理（面试必背）

【本次待补短板】

  1. 纠正索引表述：聚簇索引是按主键组织的索引结构，一张表仅一个；二级索引叶子节点存的是主键值，不是指针
  2. 补充最左前缀原则的例外场景：MySQL8.0 索引跳跃扫描的生效条件
  3. 完善慢查询的完整排查链路，补充深分页、order by 无索引等高频慢场景的优化方案

模块 3：Redis（面试高频，必须覆盖全场景）

【必背核心考点】

  1. Redis 高性能核心原理

    • 纯内存操作、单线程模型、IO 多路复用机制、优化的底层数据结构、无多线程锁竞争开销
    • Redis6.0 的多线程优化（网络 IO 多线程，核心命令仍单线程）

  2. 三大缓存问题全场景

    • 每个问题的核心定义、触发场景、完整解决方案

  3. Redis 分布式锁

    • 正确基础实现、核心特性、常见坑的原因 + 解决方案

  4. 核心数据结构、持久化机制、主从集群原理（面试必背）

【本次待补短板】

  1. 补全 Redis 单线程快的 5 大核心原因，纠正 epoll 的表述（服务端用 epoll 监听事件，不是客户端发送 epoll 指令）

  2. 完善三大缓存问题的核心前提和完整解决方案

    • 缓存穿透：核心是请求的 key 在缓存和数据库都不存在，解决方案：布隆过滤器、空值缓存 + 短过期时间、参数校验
    • 缓存击穿：核心是高并发热点 key 突然过期，解决方案：热点 key 永不过期、分布式锁互斥
    • 缓存雪崩：两大场景（大量 key 同时过期、Redis 集群宕机），对应解决方案：随机过期时间、高可用集群、多级缓存、熔断降级

  3. 补全分布式锁的完整实现和坑的解决方案

    • 原子加锁命令：SET key 唯一标识 NX EX 超时时间
    • 防误释放：Lua 脚本实现 “判断标识一致再删除” 的原子操作
    • 锁超时：Ridssion看门狗续期机制的原理和生效条件
    • 可重入锁实现、主从集群锁、时钟漂移失效问题的解决方案

模块 4：Java 并发编程（面试分水岭，必须吃透底层）

【必背核心考点】

  1. Java 内存模型（JMM）

    • 核心定义、解决的问题、主内存与工作内存的抽象关系
    • 三大核心特性：原子性、可见性、有序性的完整定义
    • volatile 关键字的完整作用、使用场景

  2. 锁机制

    • synchronized 的底层原理、锁升级流程、可重入性、适用场景
    • ReentrantLock 的底层原理、核心特性、适用场景
    • 两者的完整对比

  3. AQS（抽象队列同步器）

    • 核心原理、独占 / 共享模式的实现逻辑
    • 基于 AQS 实现的常用工具类

【本次待补短板】

  1. 背熟 JMM 的完整定义、三大特性（补齐有序性）
  2. 补全 volatile 的完整作用：保证可见性 + 有序性，不保证原子性，以及禁止指令重排序的核心作用
  3. 纠正 synchronized 可重入性的错误，补全 synchronized 和 ReentrantLock 的完整区别与适用场景
  4. 吃透 AQS 的核心原理，以及基于 AQS 实现的常用并发工具类

模块 5：Spring 核心（后端必备，必须搞懂设计逻辑）

【必背核心考点】

  1. Spring Bean 完整生命周期

    • 完整 7 步流程：实例化 → 属性填充（依赖注入） → 初始化前 → 初始化 → 初始化后 → 正常使用 → 销毁

  2. 循环依赖解决方案

    • 解决范围：仅支持单例 Bean 的 setter 注入循环依赖
    • 三级缓存各自的作用、完整处理流程
    • 必须用三级缓存的核心原因

  3. Spring 事务

    • 事务传播行为、隔离级别（两个核心概念不能混淆）
    • @Transactional 的底层原理、高频失效场景与根本原因

【本次待补短板】

  1. 记牢 Spring Bean 的完整生命周期，补齐属性填充、BeanPostProcessor 处理环节，规范术语
  2. 纠正循环依赖的流程错误，补全三级缓存的作用，以及 “为什么不能用二级缓存” 的核心原因
  3. 彻底区分事务的「隔离级别」和「传播行为」，补全 @Transactional 所有高频失效场景与根本原因

模块 6：项目梳理（应届生面试核心，决定上限）

【必背项目梳理框架（STAR 法则）】

每个项目必须提前梳理清楚：

  • S（场景）：项目背景、解决的核心问题
  • T（任务）：你负责的核心模块、核心任务
  • A（行动）：遇到的核心技术难点，你的分析思路、解决方案、具体执行步骤
  • R（结果）：最终效果，可量化的指标（如支持的用户量、响应时间、错误率等）

【本次待补短板】

  1. 重新梳理项目核心难点，不要规避核心问题：如果用单用户排他锁方案，就明确为 “多用户在线查看、单用户独占编辑”，避免被协同编辑的核心问题追问
  2. 搞懂项目中所有技术组件的底层原理：Disruptor 的核心结构、性能优势的核心原因，WebSocket 消息可靠性保障方案（ACK 确认、断线重连补发、持久化兜底）
  3. 补充项目可量化的压测数据，明确压测环境、核心指标（在线连接数、TPS、响应时间）
  4. 删掉项目中自己不懂、说不清楚的技术名词，避免给自己挖坑
