1.智能体初识

发表于2026-04-17|更新于2026-04-17|2.Areas🌐AIHello-Agents
|总字数:6.6k|阅读时长:19分钟|浏览量:

refer：https://hello-agents.datawhale.cc/#/./chapter1/%E7%AC%AC%E4%B8%80%E7%AB%A0%20%E5%88%9D%E8%AF%86%E6%99%BA%E8%83%BD%E4%BD%93

在本章中，我们共同踏上了探索智能体的初识之旅。我们的旅程从最基本的问题开始：

  • 什么是大语言模型驱动的智能体？ 我们首先明确了其定义，理解了现代智能体是具备了能力的实体。它不再仅仅是执行预设程序的脚本，而是能够自主推理和使用工具的决策者。
  • 智能体如何工作？ 我们深入探讨了智能体与环境交互的运行机制。我们了解到，这个持续的闭环是智能体处理信息、做出决策、影响环境并根据反馈调整自身行为的基础。
  • 如何构建智能体？ 这是本章的实践核心。我们以“智能旅行助手”为例，亲手构建了一个完整的、由真实 LLM 驱动的智能体。
  • 智能体有哪些主流的应用范式？ 最后，我们将视野投向了更广阔的应用领域。我们探讨了两种主流的智能体交互模式：一是以 GitHub Copilot 和 Cursor 等为代表的、增强人类工作流的“开发者工具”；二是以 CrewAI、MetaGPT 和 AgentScope 等框架为代表的、能够独立完成高层级目标的“自主协作者”。同时讲解了 Workflow 与 Agent 的差异。

什么是智能体？

宏观解释

  • 输入：通过【Sesnsors】感知【Environment】
  • 输出：自主 通过【Actuators 】采取 【Action】

智能是因为他的自主性（Autonomy），这意味着他能自主决策，而不是只是被动响应外部刺激或者预设指令的小呆瓜

有哪些类型？

基于内部决策架构分类

第一种分类维度是依据智能体内部决策架构的复杂程度，传统智能体，他的演进路径本身就构成了最经典的分类阶梯，

它涵盖了从简单的反应式智能体，到引入内部模型的模型式智能体，再到更具前瞻性的基于目标和基于效用的智能体。此外，学习能力则是一种可赋予上述所有类型的元能力，能通过经验自我改进。

  • 起点：最简单的反射智能体（Simple Reflex Agent），只有最简单的“条件-动作”，如恒温器，一旦高于设定值就启动制冷
    • 问题：仅依赖当前感知输入，没有记忆 / 预测能力，如果仅依靠预设条件不足以决策，怎么办？
  • 基于模型的反射智能体（Model-Based Reflex Agent）：内部有个世界模型（World Model），可以试图追踪无法被直接感知的内容
    • 它试图回答：“世界现在是什么样子的？”。这就发展出了简单的记忆，而不是依赖瞬时感知，有一个更完整的状态理解
    • 如自动驾驶汽车摄像头捕捉不到前方车辆，但他内部模型仍然会维持车存在、速度和预估位置的推断
  • 基于目标的智能体（Goal-Based Agent）：他不是被动对环境反应，而是针对target主动地、可预见地朝未来某个特定状态移动
    • 回答的问题是：“我应该做什么才能达成目标？”，这就体现了未来的规划。
    • 如GPS会基于地图数据（世界模型），目的是去公司，会主动用A*等搜索算法规划（Planning）最优路径
    • 问题：我们不仅希望只是去公司一个目标，往往有同时多个目标，最省油 、最省时间等。
  • 基于效用的智能体（Utility-Based Agent）：为每一个可能的世界状态都赋予一个效用值，这个值代表了满意度的高低，目标是最大化期望效用
    • 回答的是：哪种行为能为我带来最满意的结果？，这就是在互相冲突的目标相互权衡，其决策很接近人的理性选择了
    • 问题：所有决策依赖人类的先验知识，能否自主在与环境交互中学习？
  • 学习型智能体（Learning Agent）：**强化学习（Reinforcement Learning, RL）**是实现这一思想最具代表性的路径。在原先的性能元件基础上加入学习元件，通过不断观察行动带来的结果不断修正性能元件的决策策略。
    • 如AlphaGo一开始可能随机落子，但最终赢下来时会给予正向奖励，通过大量自我对弈，学习元件会探索出最佳的棋路。

在**GPT（Generative Pre-trained Transformer）**等LLM大语言模型驱动下，Agent产生了新的范式，我们的核心是怎么引导这个大脑去规划、行动、学习，而不是开发专用工具本身。

  • 主要的优势是：LLM通过海量数据预训练，获取到了隐式的世界模型和强大的涌现能力，能更加泛化地应对复杂任务，这意味着他能直接处理模糊的、高层级的、充满上下文信息的自然语言指令。
  • 如旅行助手在LLM出来前，要手动切换不同的应用，最终由用户来整合与决策，而在LLM出来后，只需要接收到规划厦门的路线后，他会将整个流程整合：
    • 规划和推理：分解子任务，如确认偏好–查询目的地信息–指定行程草案–预订住宿
    • 工具使用：识别到信息缺口，主动调用工具补全，比如天气工具有雨，那么就会在后面活动推荐尽量室内活动
    • 动态修正：根据用户反馈（预算等）会作为后续行动的约束，重新搜索并推荐新的选项，展现出根据上下文修正的能力

基于时间与反应性分类（ReAct）

这里关注的是智能体在接收到信息后，是立即行动还是规划后再行动。是追求速度的**反应性（Reactivity）与追求最优解的规划性（Deliberation）**之间的平衡，衍生出三种不同的智能体

  • 反应式：简单反应式和基于模型的智能体都属于此类别。速度快，计算开销低
    • 快速决策的场景：安全气囊必须在发生事故毫秒级做出反应，但代价是缺乏长远规划
  • 规划式：基于目标和基于效用的智能体很典型，他的决策就像是一个棋手深思熟虑评估每一个不同序列的结果来找到最佳路径
    • 深思熟虑：指定商业计划或者旅行计划，但代价是可能错过执行最佳时机
  • 混合式：而现实中我们往往既要及时反应又要长远规划，比如旅行助手规划好行程后又希望根据用户行为实时推荐，衍生出ReAct的行为模式：“思考-行动-观察”的循环，这样既能通过反馈应对环境实时变化，又有连贯的步骤可以完成复杂的长期目标。
    • 规划(Reasoning) ：LLM分析当前状况，规划出下一步的合理行动。这是一个审议过程。
    • 反应(Acting & Observing) ：在“行动”和“观察”阶段，智能体与外部工具或环境交互，并立即获得反馈。这是一个反应过程。

基于知识表示的分类（逻辑推理-神经网络）

这是更根本地表示知识是以何种形式存储在AI脑袋中

  • 符号主义 AI（Symbolic AI）：

    • 理念是智能源于对于符号的逻辑操作

      • 好比是一个图书管理员，将世界整理为一个规则的知识图谱。

    • 优势：透明和可解释，可追溯决策过程，适合金融医疗这种高风险。

    • 劣势：但依赖一个规则体系，有“知识获取瓶颈”，任何未被覆盖的新情况都会导致系统失灵

  • 亚符号主义 AI（Sub-symbolic AI）

    • 隐式地作为神经元分布在神经网络中，是一种海量数据中的统计模式
      • 如果说符号主义是图书管理员，那他就是一个小孩，不是学习猫会叫的规则，而是看过成千上万个猫后，神经网络中辨认出猫的视觉模式
    • 优势：模式识别能力和噪声的鲁棒性，能处理图像等非结构化数据
    • 劣势：但通常被视为一个黑箱（Black Box），在纯推理上表现差，会有幻觉问题

  • 神经符号主义 AI（Neuro-Symbolic AI）

    • 这是针对讲规则的符号主义和讲经验的亚符号主义 统筹后的神经符号混合主义，思路就是混合这两个范式的优点，我既要能从神经网络中从数据学习，又能像符号系统一样进行逻辑推理，试图弥合感知和认知，直觉和理性的鸿沟，

下图中，是诺贝尔经济学奖得主丹尼尔·卡尼曼（Daniel Kahneman）在其著作《思考，快与慢》（Thinking, Fast and Slow）中提出的双系统理论，为我们理解神经符号主义提供了一个绝佳的类比

  • 系统 1缓慢、有条理、基于逻辑的审慎思维，恰如符号主义 AI 的推理过程。
  • 系统 2是快速、凭直觉、并行的思维模式，类似于亚符号主义 AI 强大的模式识别能力。

怎么工作的？

我们首先要理解任务是什么，用PEAS模型来描述。

值得说明：

  • 环境是部分可观察的，他一般获取的是部分航班数据，所以一般需要记忆（记住之前的路线）和探索（尝试查询不同日期）的能力
  • 环境有确定性和随机性，搜索票价价格可能不一样，所以需要具备对于不确定性的处理能力
  • 环境可能是在多智能体情况下，那么其他用户的自动化脚本等就是该智能体的环境中的其他智能体，他们的行动会影响当前智能体的决策
  • 任务是在序贯、动态的环境中。序贯是当前动作会影响未来，而动态说明环境自身也在随时变化，所以感知-思考-行动-观察循环必须迅速，灵活适应环境

那么确定任务环境后，看一下具体的运行机制。核心是Agent Loop

  • 感知：即观察，是循环的起点，如传感器接收的信息，当然也可以是上一步的反馈
  • 思考：进入决策，通常是LLM驱动内部推理，内部有两个关键环节
    • 规划：拆解子任务
    • 工具选择：从可用工具库中选择适合下一步骤的工具，并确定参数
  • 行动：通过执行器执行具体行动，意图改变环境状态

我们需要一套交互协议来有效规范Agent和环境的信息交换，遵循特定格式文本，一般包含思考+行为

外部解析器捕捉到这个行为后，就会调用对应函数。

但LLM一般不希望JSON这种特殊格式，会引入冗余信息，所以感知系统会扮演传感器的角色，将原始输出改写为一个清晰的自然语言

这段Observation则会给智能体，并作为下一轮的Thought和Action

实操1：开始第一个智能体

集中演示了基于Thought-Action-Observation范式的智能体所具备的四项基本能力：任务分解、工具调用、上下文理解和结果合成。正是通过这个循环的不断迭代，智能体才得以将一个模糊的用户意图，转化为一系列具体、可执行的步骤，并最终达成目标。

智能体协作

其协作模式主要分为两种：

  • 一种是作为高效工具，深度融入我们的工作流；
    • 像GitHubCopilot、Claude Code、Trae、Cursor增强了开发，帮助自动化处理繁琐、重复的任务，讲究人机协作，AI正在融入软件开发的生命周期。
  • 另一种则是作为自主的协作者，与其他智能体协作完成复杂目标。

在第二种模式下，智能体独立地进行规划、推理、执行和反思，直到最终交付成果，从“命令-执行”演变为“目标-委托”，他不是被动工具，而是主动的目标执行

  • 单智能体自主循环： AgentGPT 所代表的模式。其核心是一个通用智能体通过“思考-规划-执行-反思”的闭环
  • 多智能体协作：模拟人类团队的协作模式来解决复杂问题
    • 角色扮演式对话：如 CAMEL 框架，通过为两个智能体（例如，“程序员”和“产品经理”）设定明确的角色和沟通协议，让它们在一个结构化的对话中协同完成任务
    • 组织化工作流：如 MetaGPT 和 CrewAI，它们模拟一个分工明确的“虚拟团队”（如软件公司或咨询小组）。每个智能体都有预设的职责和工作流程（SOP），通过层级化或顺序化的方式协作，产出高质量的复杂成果（如完整的代码库或研究报告）。
    • AutoGen 和 AgentScope 则提供了更灵活的对话模式，允许开发者自定义智能体间的复杂交互网络。
  • 高级控制流架构：诸如 LangGraph 等框架，则更侧重于为智能体提供更强大的底层工程基础。它将智能体的执行过程建模为状态图（State Graph），从而能更灵活、更可靠地实现循环、分支、回溯以及人工介入等复杂流程。

有必要区分work-flow和agent

Workflow 是让 AI 按部就班地执行指令，而 Agent 则是赋予 AI 自由度去自主达成目标。

  • 工作流是一种传统的自动化范式，其核心是对一系列任务或步骤进行预先定义的、结构化的编排
  • 基于大型语言模型的智能体是一个具备自主性的、以目标为导向的系统

例子说明：智能旅行助手。当我们向它下达一个新指令，例如：“你好，请帮我查询一下今天北京的天气，然后根据天气推荐一个合适的旅游景点。” 它的处理过程充分展现了其自主性：

  1. 规划与工具调用： Agent 首先会把任务拆解为两个步骤：① 查询天气；② 基于天气推荐景点。随即，它会自主选择并调用“天气查询 API”，并将“北京”作为参数传入。
  2. 推理与决策： 假设 API 返回结果为“晴朗，微风”。Agent 的 LLM 大脑会基于这个信息进行推理：“晴天适合户外活动”。接着，它会根据这个判断，在它的知识库或通过搜索引擎这个工具中，筛选出北京的户外景点，如故宫、颐和园、天坛公园等。
  3. 生成结果： 最后，Agent 会综合信息，给出一个完整的、人性化的回答：“今天北京天气晴朗，微风，非常适合户外游玩。为您推荐前往【颐和园】，您可以在昆明湖上泛舟，欣赏美丽的皇家园林景色。”

在这个过程中，没有任何写死的if天气=晴天 then 推荐颐和园的规则。如果天气是“雨天”，Agent 会自主推理并推荐国家博物馆、首都博物馆等室内场所。这种基于实时信息进行动态推理和决策的能力，正是 Agent 的核心价值所在。

课后思考

  1. 请分析以下四个 case 中的主体是否属于智能体，如果是，那么属于哪种类型的智能体（可以从多个分类维度进行分析），并说明理由：

    case A：一台符合冯·诺依曼结构的超级计算机，拥有高达每秒 2EFlop 的峰值算力

    他不是智能体，只能被动执行指令，完全由外部程序定义，只能作为计算工具，而不是agent本身

    case B：特斯拉自动驾驶系统在高速公路上行驶时，突然检测到前方有障碍物，需要在毫秒级做出刹车或变道决策

    从决策类型上，他是模型式智能体，有一定记忆了，能对潜在的情况进行分析

    时间反应上，仍是属于反应式

    知识表示上，神经符号混合架构，神经网络处理摄像头，上层通过符号化来处理刹车等安全决策

    case C：AlphaGo在与人类棋手对弈时，需要评估当前局面并规划未来数十步的最优策略

    决策模型上，是学习型了，有强化学习的概念，可以根据不断的试错和补偿机制调整

    时间反应上，是规划式，有一个目标在，根据当前目标的收益会懂得分析下一步的目标了

    知识表示上，神经网络的直觉判断 + 符号系统的逻辑推理的结合

    case D：ChatGPT 扮演的智能客服在处理用户投诉时，需要查询订单信息、分析问题原因、提供解决方案并安抚用户情绪

    决策模型上，是基于LLM的，用预处理模型能够更加泛化地处理自然语言、多模态数据

    时间反应上，是结合了反应式和规则式：“思考-行动-观察”，先拆解任务，再逐个执行，接着再根据执行结果继续分析，然后循环

    知识表示上，是结合了符号主义和亚符号主义，既能从神经网络中从数据学习，又能像符号系统一样进行逻辑推理，试图弥合感知和认知，直觉和理性的鸿沟

  2. 假设你需要为一个”智能健身教练”设计任务环境。这个智能体能够：

    • 通过可穿戴设备监测用户的心率、运动强度等生理数据
    • 根据用户的健身目标（减脂/增肌/提升耐力）动态调整训练计划
    • 在用户运动过程中提供实时语音指导和动作纠正
    • 评估训练效果并给出饮食建议

    请使用 PEAS 模型完整描述这个智能体的任务环境，并分析该环境具有哪些特性（如部分可观察、随机性、动态性等）。

    Performance：客户的健身目标达成率，用户的执行成功率，执行后的效果反馈

    Environment：用户状态、物理环境、穿戴设备、用户输入、时间

    Actuators：语音播报、计划文本生成、穿戴设备控制

    sensors：设备的生理数据，运动过程状态、用户输入

    部分可观察：只能观测到用户部分状态，主观感受、不能完全捕捉到用户所有的饮食，这就需要对已观察的推断，并通过反馈不断修正

    随机性：相同的输入会有不同结果，比如同样的用户输入，会有不同的训练计划，根据当前用户状态，动态调整策略

    序贯和动态性：心率、动作动态变化，要求Agent要实时响应，要在ReAct中快速响应，而不能规划好了才思考，当前决策会对未来的影响，今天训练会对未来的训练有影响，错误的指导导致用户出现损伤，也不能只看短期

    单智能体环境：多人训练场景下，可能需要考虑多智能体

  3. 某电商公司正在考虑两种方案来处理售后退款申请：

    方案 A（Workflow）：设计一套固定流程，例如：

    A.1 对于一般商品且在 7 天之内，金额 < 100RMB 自动通过；100-500RMB 由客服审核；>500RMB 需主管审批；而特殊商品（如定制品）一律拒绝退款

    A.2 对于超过 7 天的商品，无论金额，只能由客服审核或主管审批；

    方案 B（Agent）：搭建一个智能体系统，让它理解退款政策、分析用户历史行为、评估商品状况，并自主决策是否批准退款

    请分析：

    • 这两种方案各自的优缺点是什么？
      • 前者：实时性高，确定性高，成本低，可解释性高，抗风险强；但对应的就是僵化死板，复杂业务难处理，恶意用户薅羊毛行为，扩展性等
      • 后者：灵活性，上下文理解，自学习，用户体验，人工替代，适应性；但决策不确定（幻觉），维护高，可解释性差，prompt注入攻击等
    • 在什么情况下 Workflow 更合适？什么情况下 Agent 更有优势？如果你是该电商公司的负责人，你更倾向于采用哪种方案？
      • workflow确定性高，适合确定的任务，7天无理由，并且时间要求很高，以及金融这种合规性要求很高，以及在流量大的场景下需要控制成本
      • agent自主规划性高，适合复杂任务，比如问题边界模糊，需要多维度决策（历史信用 金额等），业务变化快，自然语言处理需求，减少人工工作量等
      • 我作为负责人，单独使用A或者B显然不好，前者20%复杂情况堆积到人工处理，后者所有用大模型成本又太高，以及错误决策风险
    • 是否存在一个方案 C，能够结合两种方案，达到扬长避短的效果？
      • 可以，先由 Workflow 处理标准化小额退款，复杂场景交由 Agent 决策，大额 / 高风险订单由人工兜底

  4. 在 1.3 节的智能旅行助手基础上，请思考如何添加以下功能（可以只描述设计思路，也可以进一步尝试代码实现）：

    提示：思考如何修改 Thought-Action-Observation 循环来实现这些功能。

    • 添加一个”记忆”功能，让智能体记住用户的偏好（如喜欢历史文化景点、预算范围等）
      • 思路：在原先的临时列表改进，分为短期记忆（近期会话内容）和长期记忆（结构化用户画像），用JSON文件存储该会话ID下的历史会话，会话结束后，LLM更新用户画像，后续调用的时候会将历史会话记忆作为信息上下文即可
    • 当推荐的景点门票已售罄时，智能体能够自动推荐备选方案
      • 针对该需求，需要提供一个查询库存的工具
      • 依旧使用ReAct的方式，在这个过程中，就会思考先用这个查询库存先检查库存
      • 如果库存没有了，会在短期记忆里加入售罄的标识，在后续的思考阶段主动排除 短期记忆中的排除节点
    • 如果用户连续拒绝了 3 个推荐，智能体能够反思并调整推荐策略
      • 短期记忆记录一个拒绝数量和拒绝原因列表，当拒绝的时候添加，如果等于3之后，就把原因也加进来，LLM生成一个反思报告，推断出新的偏好，更新到用户长期记忆中

  5. 卡尼曼的”系统 1”（快速直觉）和”系统 2”（慢速推理）理论[2]为神经符号主义 AI 提供了很好的类比。请首先构思一个具体的智能体的落地应用场景，然后说明场景中的：

    提示：医疗诊断助手、法律咨询机器人、金融风控系统等都是常见的应用场景

    • 哪些任务应该由”系统 1”处理？
      • 毫秒级实时感知与应急响应，如路况视觉识别、障碍物检测、紧急制动、车道偏离预警、突发碰撞预判，依赖神经网络快速直觉式判断。
    • 哪些任务应该由”系统 2”处理？
      • 慢速、逻辑化的复杂决策与规划，如长途导航路径优化、复杂路况权衡决策、驾驶风险综合评估、事故原因逻辑推演。
    • 这两个系统如何协同工作以达成最终目标？
      • 系统 1 先做实时安全兜底，快速处理突发状况；系统 2 基于系统 1 的感知数据做长期理性规划，同时系统 2 的决策指令约束系统 1 的执行，二者配合既保证驾驶安全实时性，又实现行程决策合理性。

  6. 尽管大语言模型驱动的智能体系统展现出了强大的能力，但它们仍然存在诸多局限。请分析以下问题：

    • 为什么智能体或智能体系统有时会产生”幻觉”（生成看似合理但实际错误的信息）？
      • 因为LLM本质是个文本概率生成模型，基于预训练词汇预测后续关联输出内容，没有事实校验的能力，为了文本通顺而加入非严格事实就是幻觉
    • 在 1.3 节的案例中，我们设置了最大循环次数为 5 次。如果没有这个限制，智能体可能会陷入什么问题？
      • 可能会死循环，卡死并且浪费token
    • 如何评估一个智能体的”智能”程度？仅使用准确率指标是否足够？
      • 不只是准确率，还有工具调用效率、资源消耗、反思纠错能力、逻辑规划能力、可解释性等
