📊 当前可用资源总结

🔌 Plugins（插件）

Plugin 状态 用途
telegram ✅ 可用 Telegram 消息
openclaw-lark ✅ 可用 飞书消息
lossless-claw ✅ 可用 无损上下文（缺依赖）

📦 Skills（技能）

Skill 状态 用途
agent-browser ✅ 可用 浏览器自动化（点击、填表、截图）
bb-browser ✅ 可用 信息获取（爬虫、搜索）
opencli ✅ 可用 社交网站 CLI（B 站、知乎、推特）
openclaw-backup ✅ 可用 加密备份/恢复
self-improving-agent ✅ 可用 自改进 Agent
skill-vetter ✅ 可用 安全审核

🔧 Tools（工具）

• 飞书全栈工具（文档、表格、日历、任务） • 浏览器控制 • 消息发送 • 节点管理 • TTS 语音

───

🎯 你能用它们干什么

场景 可用技能
刷招聘网站 bb-browser + agent-browser
自动投简历 opencli（知乎、B 站）
备份数据 openclaw-backup
监听技术博客 bb-browser（手动）
管理知识库 obsidian（需配置）
语音播报天气 天气 + TTS

───

📖 Tool vs Skill 区别

Tool（工具） Skill（技能）
定义 可直接调用的函数 封装好的工作流/技能包
调用方式 我直接执行 触发条件后自动加载
例子 feishu_fetch_doc 读文档 blogwatcher 监控博客
关系 Skill 调用多个 Tools Skill 是更高级的封装

简单说：Tool = 我手上的工具，Skill = 我学会的技能～ 🐱
