word转markdown（latex公式）+图片，使用pandoc，方便把导师/老板喜欢的word，转换成AI喜欢的markdown，避免直接导入给AI把一些公式识别不出

  • 关键参数：--extract-media 提取图片，公式会自动转为 $...$（行内）/ $$...$$（块）标准 LaTeX 格式
  • 打开输出的 输出.md 到 Typora，所有公式都是可编辑的 LaTeX 代码，不是图片

清除掉git误提交的密钥或配置等文件，使用git-filter-repo的python库组件
使用的坑点

  • 他为了保证远端仓库安全，本地清除后会删除掉远端地址，需要本地没问题后，手动进行添加
  • 注意目录路径的变动，要用历史提交中的工作区路径，而不是当前工作区路径
    顺手的事：
  • 顺便用到了git reset的命令：soft是版本后的内容帮我add到暂存区，mixed是版本后的内容不add，全部放到工作区里，hard是丢失后面全部提交的，keep是安全版的hard，本地修改不覆盖
  • git stash作用是要切换分支或者拉最新的代码（可能有冲突），但不想把我的垃圾代码提交上去 / 不想删掉，那就可以先贮存起来，后面用git stash pop就能pop回来

带着眼镜晚上看发光源会出现比较严重的眩光，可能是眼镜有油，可以尝试用洗洁精搓两面，清水冲掉，纸巾从中心向四周单向擦开，看发光源就会舒服很多。

windows一键安装激活office全件套（Office Tool Plus v11.2.19.0 -office安装激活一条龙 - 果核剥壳）

1925 年的 “柯立芝繁荣” 是不是当代的AI兴起的旧影？看似AI发展的很猛，是不是要警惕AI泡沫 / 虚假繁荣的陷阱？有钱人先享受技术会不会加剧贫富差距？ “柯立芝繁荣” 是奇迹，但后面1929经济大萧条的黑色星期四更值得反思。1928年那些在股市死的很惨的那些人做错了什么？他们是靠时代红利获利的人（把时代 β 当个人 α），自然也会被时代所抛弃，那映射到我现在找的公司，他是不是真的用AI实现了降本增效，还是说是画饼炒概念的泡沫经济？穷人用不上科技，就是被降本，被取代了。对应到现在，不用AI的话，一定会被取代，驾驭住AI才能保住饭碗（我有机会能做到的），而那些掌握算力/资本的才是真正收割技术红利的
当时的聪明人在做什么？他们把现金攥在手里，不加杠杆，布局的是刚需行业，而不是跟风，在崩盘的那一刻也是他们抄底资产的机会。那在当前值得思考，当今AI风口这种很虚的概念到底是什么？~~那说到底的话，最重要的还是客户刚需的业务，还有积累的客户，能带来现金流的业务，而不是把所有杠杆赌在一个模型上的，所以对我目前的话，基本的客户真实业务盘要稳住，不能把精力全部梭哈到AI上~
